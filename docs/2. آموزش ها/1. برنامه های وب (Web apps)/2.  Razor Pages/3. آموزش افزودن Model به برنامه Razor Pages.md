
در این بخش، با نحوه ایجاد **مدل داده (Model)** در یک پروژه **ASP.NET Core Razor Pages** آشنا می‌شویم. همچنین یاد می‌گیریم چگونه با استفاده از **Entity Framework Core (EF Core)** پایگاه داده را ایجاد کرده و صفحات مدیریت اطلاعات را به‌صورت خودکار تولید کنیم.

در پایان این بخش، برنامه قادر خواهد بود اطلاعات فیلم‌ها را در پایگاه داده ذخیره، نمایش، ویرایش و حذف کند.

---

# Entity Framework Core چیست؟

**Entity Framework Core (EF Core)** یک **ORM (Object-Relational Mapper)** برای .NET است.

ORM لایه‌ای بین برنامه و پایگاه داده ایجاد می‌کند تا به‌جای نوشتن دستورات SQL، بتوانید با استفاده از کلاس‌های C# با داده‌ها کار کنید.

به کمک EF Core می‌توانید:

- اطلاعات را ذخیره کنید.
    
- اطلاعات را بازیابی کنید.
    
- داده‌ها را ویرایش کنید.
    
- داده‌ها را حذف کنید.
    
- ساختار پایگاه داده را از روی کلاس‌های C# ایجاد کنید.
    

---

# ایجاد مدل داده (Model)

ابتدا پوشه‌ای با نام **Models** در پروژه ایجاد کنید.

سپس داخل این پوشه یک کلاس با نام **Movie** بسازید.

نمونه کد:

```csharp
using System.ComponentModel.DataAnnotations;

namespace RazorPagesMovie.Models;

public class Movie
{
    public int Id { get; set; }

    public string? Title { get; set; }

    [DataType(DataType.Date)]
    public DateTime ReleaseDate { get; set; }

    public string? Genre { get; set; }

    public decimal Price { get; set; }
}
```

---

# بررسی کلاس Movie

این کلاس مشخص می‌کند که هر فیلم چه اطلاعاتی خواهد داشت.

## شناسه فیلم

```csharp
public int Id { get; set; }
```

این فیلد **کلید اصلی (Primary Key)** جدول است.

EF Core به‌صورت خودکار این فیلد را به عنوان شناسه یکتا در نظر می‌گیرد.

---

## عنوان فیلم

```csharp
public string? Title { get; set; }
```

عنوان فیلم در این ویژگی ذخیره می‌شود.

علامت **?** نشان می‌دهد مقدار این ویژگی می‌تواند تهی (Null) باشد.

---

## تاریخ انتشار

```csharp
[DataType(DataType.Date)]
public DateTime ReleaseDate { get; set; }
```

ویژگی **DataType** مشخص می‌کند که این مقدار فقط یک تاریخ است.

در نتیجه:

- فقط تاریخ نمایش داده می‌شود.
    
- نیازی به وارد کردن ساعت و دقیقه نیست.
    

---

## ژانر

```csharp
public string? Genre { get; set; }
```

نوع فیلم مانند:

- اکشن
    
- کمدی
    
- ترسناک
    
- علمی تخیلی
    

در این قسمت ذخیره می‌شود.

---

## قیمت

```csharp
public decimal Price { get; set; }
```

برای ذخیره قیمت از نوع **decimal** استفاده می‌شود.

این نوع داده برای مقادیر مالی مناسب‌تر از **float** یا **double** است، زیرا دقت بیشتری دارد.

# کلاس‌های POCO

کلاس **Movie** یک کلاس **POCO (Plain Old CLR Object)** محسوب می‌شود.

کلاس‌های POCO وابستگی مستقیمی به EF Core ندارند و فقط ساختار داده‌های برنامه را تعریف می‌کنند. همین موضوع باعث می‌شود بتوان از آن‌ها در بخش‌های مختلف برنامه استفاده کرد و نگهداری کد ساده‌تر شود.

# تولید خودکار صفحات (Scaffolding)

یکی از امکانات مهم ASP.NET Core، قابلیت **Scaffolding** است.

Scaffolding بر اساس مدل داده، صفحات موردنیاز برای عملیات **CRUD** را به‌صورت خودکار ایجاد می‌کند.

CRUD مخفف چهار عملیات اصلی روی داده‌ها است:

- **Create** : ایجاد اطلاعات جدید
    
- **Read** : نمایش اطلاعات
    
- **Update** : ویرایش اطلاعات
    
- **Delete** : حذف اطلاعات

# تغییر فایل Program.cs

Scaffolding چند خط کد نیز به فایل **Program.cs** اضافه می‌کند.

مهم‌ترین قسمت آن ثبت **DbContext** در سیستم Dependency Injection است.

نمونه کد:

```csharp
builder.Services.AddDbContext<RazorPagesMovieContext>(options =>
    options.UseSqlServer(
        builder.Configuration.GetConnectionString("RazorPagesMovieContext")));
```

این کد به برنامه اعلام می‌کند که برای ارتباط با پایگاه داده از کلاس **RazorPagesMovieContext** استفاده کند.

---

# ایجاد پایگاه داده با Migration

یکی از قابلیت‌های مهم EF Core، **Migration** است.

Migration امکان ایجاد و به‌روزرسانی ساختار پایگاه داده را بدون از بین رفتن اطلاعات فراهم می‌کند.

ابتدا دستور زیر را اجرا کنید:

```powershell
Add-Migration InitialCreate
```

این دستور اولین Migration پروژه را ایجاد می‌کند.

---

سپس پایگاه داده را ایجاد کنید:

```powershell
Update-Database
```

این دستور:

- پایگاه داده را ایجاد می‌کند.
    
- جدول Movie را می‌سازد.
    
- Migration را روی پایگاه داده اعمال می‌کند.
    

---

# کلاس RazorPagesMovieContext

فایل زیر ایجاد می‌شود:

```text
Data/RazorPagesMovieContext.cs
```

نمونه کد:

```csharp
public class RazorPagesMovieContext : DbContext
{
    public RazorPagesMovieContext(
        DbContextOptions<RazorPagesMovieContext> options)
        : base(options)
    {
    }

    public DbSet<Movie> Movie { get; set; } = default!;
}
```

---

## DbContext چیست؟

کلاس **DbContext** پل ارتباطی بین برنامه و پایگاه داده است.

تمام عملیات EF Core از طریق این کلاس انجام می‌شود.

---

## DbSet

```csharp
public DbSet<Movie> Movie { get; set; }
```

هر **DbSet** نماینده یک جدول در پایگاه داده است.

در این مثال:

کلاس **Movie** ⇔ جدول **Movie**

هر شیء Movie نیز معادل یک **رکورد (Row)** در جدول خواهد بود.

# Dependency Injection

ASP.NET Core از ابتدا بر پایه **Dependency Injection (DI)** طراحی شده است.

به همین دلیل، به‌جای ساخت مستقیم اشیاء، سرویس‌ها در Program.cs ثبت شده و در زمان نیاز به کلاس‌ها تزریق می‌شوند.

ثبت DbContext نیز به همین روش انجام می‌شود:

```csharp
builder.Services.AddDbContext<RazorPagesMovieContext>(...);
```

در ادامه آموزش، صفحات Razor به‌صورت خودکار نمونه‌ای از **RazorPagesMovieContext** را از DI دریافت کرده و برای ارتباط با پایگاه داده از آن استفاده خواهند کرد.

# جمع‌بندی

در این بخش با یکی از مهم‌ترین مراحل توسعه برنامه‌های مبتنی بر **ASP.NET Core Razor Pages** آشنا شدیم. ابتدا یک مدل داده برای ذخیره اطلاعات فیلم‌ها ایجاد کردیم و سپس با استفاده از **Entity Framework Core** و قابلیت **Scaffolding**، صفحات مدیریت اطلاعات را به‌صورت خودکار تولید کردیم.

همچنین یاد گرفتیم که:

- **Model** ساختار داده‌های برنامه را مشخص می‌کند.
    
- **Entity Framework Core** ارتباط بین برنامه و پایگاه داده را مدیریت می‌کند.
    
- **DbContext** واسطه اصلی برای انجام عملیات روی پایگاه داده است.
    
- هر **DbSet** معمولاً معادل یک جدول در پایگاه داده است.
    
- با استفاده از **Migration** می‌توان ساختار پایگاه داده را ایجاد و در آینده به‌روزرسانی کرد.
    
- قابلیت **Scaffolding** صفحات CRUD را به‌صورت خودکار ایجاد می‌کند و سرعت توسعه را به‌طور چشمگیری افزایش می‌دهد.
    

این مفاهیم پایه‌ی کار با داده‌ها در ASP.NET Core هستند و در ادامه آموزش، از همین ساختار برای توسعه و گسترش برنامه استفاده خواهیم کرد.
