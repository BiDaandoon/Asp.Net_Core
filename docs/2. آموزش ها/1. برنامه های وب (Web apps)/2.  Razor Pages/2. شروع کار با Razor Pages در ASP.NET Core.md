
در این آموزش با نحوه ایجاد اولین برنامه وب با استفاده از **ASP.NET Core Razor Pages** آشنا می‌شوید. هدف این آموزش، یادگیری مفاهیم پایه و آشنایی با ساختار پروژه‌های Razor Pages است.

در پایان این آموزش، برنامه‌ای خواهید داشت که اطلاعات فیلم‌ها را در یک پایگاه داده مدیریت می‌کند و قابلیت‌هایی مانند نمایش، افزودن، ویرایش و حذف اطلاعات را در اختیار کاربر قرار می‌دهد.

---

# پیش‌نیازها

برای توسعه برنامه‌های Razor Pages به ابزارهای زیر نیاز دارید:

- **Visual Studio** (نسخه‌ای که از .NET و ASP.NET Core پشتیبانی کند)
    
- **Visual Studio Code** به همراه .NET SDK
    

همچنین باید آخرین نسخه **.NET SDK** روی سیستم نصب شده باشد.

---

# ایجاد پروژه Razor Pages

پس از اجرای Visual Studio مراحل زیر را انجام دهید:

1. گزینه **Create a new project** را انتخاب کنید.
    
2. از میان قالب‌های موجود، **ASP.NET Core Web App (Razor Pages)** را انتخاب کنید.
    
3. روی **Next** کلیک کنید.
    
4. برای پروژه نام **RazorPagesMovie** را وارد کنید.
    
5. مجدداً روی **Next** کلیک کنید.
    
6. نسخه **.NET 10** (یا آخرین نسخه موجود) را انتخاب کنید.
    
7. گزینه **Do not use top-level statements** را در حالت غیرفعال (Unchecked) قرار دهید.
    
8. روی **Create** کلیک کنید.
    

پس از انجام این مراحل، Visual Studio ساختار اولیه پروژه را به‌صورت خودکار ایجاد می‌کند.

---

# اجرای پروژه

برای اجرای برنامه، پروژه را انتخاب کرده و کلیدهای زیر را فشار دهید:

```text
Ctrl + F5
```

این دستور برنامه را **بدون اجرای حالت اشکال‌زدایی (Debug)** اجرا می‌کند.

در اولین اجرا ممکن است از شما خواسته شود که گواهی امنیتی (SSL Certificate) مربوط به محیط توسعه را تأیید کنید.

در صورت نمایش پیام، گزینه **Yes** را انتخاب کنید.

پس از آن:

- وب‌سرور **Kestrel** اجرا می‌شود.
    
- مرورگر به‌صورت خودکار باز خواهد شد.
    
- صفحه اصلی برنامه در آدرس زیر نمایش داده می‌شود:
    

```text
https://localhost:xxxx
```

عدد انتهای آدرس (**Port**) در هر پروژه ممکن است متفاوت باشد.

---

# آشنایی با ساختار پروژه

پس از ایجاد پروژه، پوشه‌ها و فایل‌های مختلفی مشاهده خواهید کرد. مهم‌ترین آن‌ها عبارت‌اند از:

---

## پوشه Pages

این پوشه مهم‌ترین بخش پروژه‌های Razor Pages است.

تمام صفحات برنامه داخل این پوشه قرار می‌گیرند.

هر صفحه معمولاً از دو فایل تشکیل شده است.

### فایل `.cshtml`

این فایل شامل موارد زیر است:

- HTML
    
- دستورات Razor
    
- نمایش اطلاعات صفحه
    

نمونه:

```text
Index.cshtml
```

---

### فایل `.cshtml.cs`

این فایل شامل کدهای C# مربوط به همان صفحه است.

در این فایل معمولاً رویدادها، پردازش داده‌ها و منطق برنامه نوشته می‌شود.

نمونه:

```text
Index.cshtml.cs
```

بنابراین هر صفحه Razor از دو بخش تشکیل می‌شود:

- رابط کاربری (UI)
    
- منطق برنامه (Code Behind)
    

این ساختار باعث خوانایی بیشتر و نگهداری آسان‌تر پروژه می‌شود.

---

## فایل‌های کمکی (Supporting Files)

برخی فایل‌ها با علامت **_** آغاز می‌شوند.

برای مثال:

```text
_Layout.cshtml
```

این فایل قالب کلی صفحات سایت را مشخص می‌کند.

مواردی مانند:

- منوی اصلی
    
- Header
    
- Footer
    
- بخش Copyright
    

در این فایل تعریف می‌شوند.

به همین دلیل، تمام صفحات سایت ظاهر یکسانی خواهند داشت.

---

# پوشه wwwroot

پوشه **wwwroot** محل نگهداری فایل‌های ثابت (Static Files) است.

فایل‌های موجود در این پوشه مستقیماً توسط مرورگر قابل دریافت هستند.

نمونه فایل‌های موجود در این پوشه:

- CSS
    
- JavaScript
    
- تصاویر
    
- فونت‌ها
    
- فایل‌های HTML
    

هر فایل ثابتی که قرار باشد در مرورگر نمایش داده شود، داخل این پوشه قرار می‌گیرد.

---

# فایل appsettings.json

این فایل برای نگهداری تنظیمات برنامه استفاده می‌شود.

به عنوان مثال:

- رشته اتصال به پایگاه داده (Connection String)
    
- تنظیمات امنیتی
    
- تنظیمات برنامه
    
- مقادیر قابل تغییر
    

به جای قرار دادن این اطلاعات داخل کد برنامه، آن‌ها را در این فایل ذخیره می‌کنند.

---

# فایل Program.cs

مهم‌ترین فایل پروژه، **Program.cs** است.

این فایل نقطه شروع اجرای برنامه محسوب می‌شود.

نمونه کد:

```csharp
var builder = WebApplication.CreateBuilder(args);

// ثبت سرویس Razor Pages
builder.Services.AddRazorPages();

var app = builder.Build();
```

---

## ایجاد WebApplicationBuilder

```csharp
var builder = WebApplication.CreateBuilder(args);
```

این دستور شیء **WebApplicationBuilder** را ایجاد می‌کند.

وظیفه آن آماده‌سازی برنامه و ثبت سرویس‌های موردنیاز است.

---

## ثبت سرویس Razor Pages

```csharp
builder.Services.AddRazorPages();
```

این خط، پشتیبانی از Razor Pages را به سیستم **Dependency Injection (DI)** اضافه می‌کند.

اگر این دستور حذف شود، برنامه دیگر قادر به اجرای صفحات Razor نخواهد بود.

---

## ساخت برنامه

```csharp
var app = builder.Build();
```

پس از ثبت تمام سرویس‌ها، برنامه ساخته می‌شود.

---

# پیکربندی برنامه

کد زیر بررسی می‌کند که آیا برنامه در محیط توسعه اجرا می‌شود یا خیر.

```csharp
if (!app.Environment.IsDevelopment())
{
    app.UseExceptionHandler("/Error");
    app.UseHsts();
}
```

اگر برنامه در محیط **Production** یا **Test** اجرا شود:

- خطاها در صفحه Error نمایش داده می‌شوند.
    
- قابلیت امنیتی **HSTS** فعال می‌شود.
    

---

# Middlewareها

پس از ساخت برنامه، تعدادی Middleware فعال می‌شوند.

---

## UseHttpsRedirection

```csharp
app.UseHttpsRedirection();
```

تمام درخواست‌های HTTP را به HTTPS هدایت می‌کند.

این کار باعث افزایش امنیت برنامه می‌شود.

---

## UseRouting

```csharp
app.UseRouting();
```

مسیر صفحات و درخواست‌ها را مدیریت می‌کند.

به کمک این Middleware مشخص می‌شود هر درخواست باید به کدام صفحه ارسال شود.

---

## UseAuthorization

```csharp
app.UseAuthorization();
```

وظیفه کنترل دسترسی کاربران را بر عهده دارد.

اگر پروژه از سیستم ورود کاربران استفاده نکند، این Middleware ضرورتی نخواهد داشت.

---

## MapStaticAssets

```csharp
app.MapStaticAssets();
```

این دستور باعث بهینه‌سازی ارسال فایل‌های ثابت مانند:

- CSS
    
- JavaScript
    
- تصاویر
    
- فونت‌ها
    

می‌شود.

---

## MapRazorPages

```csharp
app.MapRazorPages();
```

این دستور صفحات Razor را به سیستم مسیریابی معرفی می‌کند.

در واقع بدون این دستور هیچ صفحه Razor قابل نمایش نخواهد بود.

---

## WithStaticAssets

```csharp
.WithStaticAssets();
```

این متد باعث می‌شود صفحات Razor نیز از سیستم بهینه‌سازی فایل‌های ثابت استفاده کنند.

---

## Run

```csharp
app.Run();
```

این آخرین دستور برنامه است.

با اجرای آن، وب‌سرور شروع به کار کرده و برنامه آماده دریافت درخواست‌های کاربران می‌شود.

---

# ساختار اجرای برنامه

به‌طور خلاصه، مراحل اجرای برنامه به ترتیب زیر است:

1. ایجاد **WebApplicationBuilder**
    
2. ثبت سرویس‌ها
    
3. ساخت برنامه
    
4. پیکربندی Middlewareها
    
5. ثبت مسیر صفحات Razor
    
6. اجرای وب‌سرور

# جمع‌بندی

در این بخش با اولین پروژه **ASP.NET Core Razor Pages** آشنا شدیم و ساختار آن را بررسی کردیم.

مهم‌ترین نکاتی که یاد گرفتیم عبارت‌اند از:

- **Razor Pages** یک مدل صفحه‌محور برای توسعه برنامه‌های وب است.
    
- هر صفحه از دو فایل تشکیل می‌شود:
    
    - فایل **`.cshtml`** برای رابط کاربری
        
    - فایل **`.cshtml.cs`** برای منطق برنامه
        
- پوشه **Pages** محل قرارگیری صفحات برنامه است.
    
- پوشه **wwwroot** فایل‌های ثابت مانند CSS، JavaScript و تصاویر را نگهداری می‌کند.
    
- فایل **appsettings.json** محل ذخیره تنظیمات برنامه است.
    
- فایل **Program.cs** نقطه شروع اجرای برنامه محسوب می‌شود و مسئول ثبت سرویس‌ها، پیکربندی Middlewareها و راه‌اندازی وب‌سرور است.
    
- Middlewareها نقش مهمی در پردازش درخواست‌ها، مسیریابی، امنیت و مدیریت فایل‌های ثابت دارند.
    

> **نکته آموزشی:** در نسخه‌های جدید ASP.NET Core، فایل **Program.cs** قلب برنامه محسوب می‌شود. تقریباً تمام تنظیمات اولیه، سرویس‌ها و اجزای اصلی برنامه در این فایل پیکربندی می‌شوند؛ بنابراین درک ساختار و عملکرد آن، یکی از مهم‌ترین مهارت‌ها برای هر توسعه‌دهنده ASP.NET Core است.
