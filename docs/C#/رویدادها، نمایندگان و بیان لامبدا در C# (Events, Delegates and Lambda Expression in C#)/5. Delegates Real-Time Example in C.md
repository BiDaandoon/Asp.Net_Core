## **مثال بلادرنگ از Delegates در سی شارپ**

در این مقاله، قصد دارم در مورد **Delegateها در سی شارپ با مثال‌های بلادرنگ** بحث کنم . Delegateها یکی از مهمترین مفاهیمی هستند که شما به عنوان یک توسعه‌دهنده سی شارپ باید آنها را درک کنید. در بسیاری از مصاحبه‌ها، اکثر مصاحبه‌کنندگان از شما می‌خواهند که کاربرد Delegateها را در پروژه بلادرنگی که روی آن کار کرده‌اید توضیح دهید. لطفاً دو مقاله زیر را بخوانید، که در آنها اصول اولیه Delegate و Multicast Delegateها در سی شارپ را با مثال‌هایی مورد بحث قرار داده‌ایم.

نمایندگان (Delegate) در سی شارپ (C#) به طور گسترده توسط توسعه‌دهندگان چارچوب استفاده می‌شوند. اجازه دهید نمایندگان (Delegate) در سی شارپ را با دو مثال بلادرنگ (real-time) درک کنیم.

##### **مثال ۱ از تفویض اختیار بلادرنگ در سی شارپ:**

فرض کنید کلاسی به نام Worker داریم و این کلاس متدی به نام DoWork دارد. نیاز کاری ما هنگام فراخوانی متد DoWork این است که باید یک اعلان در مورد درصد کار انجام شده به مصرف‌کننده ارسال کنیم و همچنین پس از اتمام کار، باید اعلان را نیز ارسال کنیم. به عنوان مثال، چقدر کار در ۱ ساعت، چقدر کار در ۲ ساعت، چقدر کار در ۳ ساعت و به همین ترتیب تا زمانی که کار تکمیل شود؟ و متد DoWork هیچ ایده‌ای ندارد که اعلان‌ها را به چه کسی ارسال کند. فراخوانی‌کننده متد DoWork باید تصمیم بگیرد که اعلان را به چه کسی ارسال کند. این بدان معناست که در اینجا باید از توابع فراخوانی برگشتی استفاده کنیم.

این یک سناریوی ایده‌آل است که در آن نیاز به استفاده از delegateها داریم. بنابراین، یک فایل کلاس با نام Worker,cs ایجاد کنید و سپس کد زیر را در آن کپی و جایگذاری کنید. در اینجا، می‌توانید ببینید که ما دو delegate ایجاد کرده‌ایم. می‌توانید delegateها را در هر جایی ایجاد کنید و همچنین می‌توانید delegateها را درون کلاس یا مستقیماً درون فضای نام ایجاد کنید. دلیل این امر این است که در پشت صحنه delegateها فقط کلاس هستند. متد DoWork چهار پارامتر می‌گیرد که از بین این چهار پارامتر، دو پارامتر از نوع delegate هستند. سپس درون متد، از یک حلقه for استفاده می‌کنیم و هر بار که حلقه اجرا می‌شود، مقداری کار را پردازش می‌کنیم و با فراخوانی delegate به کاربر اطلاع می‌دهیم. اجرای حلقه تکمیل می‌شود، به این معنی که کار ما انجام شده است، و سپس delegate تکمیل شده کار را فراخوانی می‌کنیم تا به کاربر اطلاع دهیم که کار انجام شده است.

```csharp
using System.Threading;

namespace DelegatesRealTimeExample
{
    public delegate void WorkPerformedHandler(int hours, string workType);
    public delegate void WorkCompletedHandler(string workTyp);

    public class Worker
    {
        public void DoWork(int hours, string workType, WorkPerformedHandler del1, WorkCompletedHandler del2)
        {
            //Do Work here and notify the consumer that work has been performed
            for (int i = 0; i < hours; i++)
            {
                //Do Some Processing
                Thread.Sleep(1000);
                //Notfiy how much works have been done
                del1(i+1, workType);
            }

            //Notfiy the consumer that work has been completed
            del2(workType);
        }
    }
}
```

حالا، کلاس Program را به صورت زیر تغییر دهید. در اینجا، ما دو متد فراخوانی با امضای مشابه delegate های خود ایجاد می‌کنیم. سپس نمونه‌هایی از delegate و همچنین یک نمونه از کلاس Worker ایجاد می‌کنیم و با ارسال مقادیر مورد نیاز به همراه دو نمونه delegate، متد DoWork را فراخوانی می‌کنیم.

```csharp
using System;

namespace DelegatesRealTimeExample
{
    class Program
    {
        static void Main(string[] args)
        {
            WorkPerformedHandler del1 = new WorkPerformedHandler(Worker_WorkPerformed);
            WorkCompletedHandler del2 = new WorkCompletedHandler(Worker_WorkCompleted);

            Worker worker = new Worker();
            worker.DoWork(5, "Generating Report", del1, del2);
            Console.ReadKey();
        }

        //Delegate Signature must match with the method signature
        static void Worker_WorkPerformed(int hours, string workType)
        {
            Console.WriteLine($"{hours} Hours completed for {workType}");
        }

        static void Worker_WorkCompleted(string workType)
        {
            Console.WriteLine($"{workType} work completed");
        }
    }
}
```

حالا برنامه را اجرا کنید و خواهید دید که هر بار که حلقه اجرا می‌شود، اعلان را ارسال می‌کند و پس از انجام کار، همانطور که در تصویر زیر نشان داده شده است، اعلان دیگری ارسال می‌کند.

این به خوبی کار می‌کند. حال، اگر به کلاس Worker و درون متد DoWork بروید، به شما این امکان را می‌دهد که نمونه delegate را null کنید.

```csharp
using System.Threading;

namespace DelegatesRealTimeExample
{
    public delegate void WorkPerformedHandler(int hours, string workType);
    public delegate void WorkCompletedHandler(string workTyp);

    public class Worker
    {
        public void DoWork(int hours, string workType, WorkPerformedHandler del1, WorkCompletedHandler del2)
        {
            del1 = null; //Allowing to set null
            del2 = null; //Allowing to set null

            //Do Work here and notify the consumer that work has been performed
            for (int i = 0; i < hours; i++)
            {
                //Do Some Processing
                Thread.Sleep(1000);
                //Notfiy how much works have been done
                del1(i + 1, workType);
            }

            //Notfiy the consumer that work has been completed
            del2(workType);
        }
    }
}
```

بنابراین، در اینجا، اگر کلاس Worker نمونه‌ای ایجاد نکند و اگر کلاس Worker هیچ ایده‌ای در مورد متدهایی که اعلان‌ها را به آنها ارسال می‌کند نداشته باشد، نباید اجازه دهد نمونه delegate تهی شود. چگونه می‌توانیم این را محدود کنیم؟ با استفاده از [**رویدادها در C#**](https://dotnettutorials.net/lesson/events-in-csharp/) که در مقاله بعدی خود در مورد آن بحث خواهیم کرد.

##### **مثال ۲ از Delegates در زبان سی شارپ:**

حالا، یک مثال بلادرنگ دیگر از delegate ها در سی شارپ خواهیم دید. فرض کنید کلاسی به نام Employee داریم که در زیر نشان داده شده است.

```csharp
namespace DelegateRealtimeExample
{
    public class Employee
    {
        public int ID { get; set; }
        public string Name { get; set; }
        public string Gender { get; set; }
        public int Experience { get; set; }
        public int Salary { get; set; }
    }
}
```

کلاس Employee فوق دارای ویژگی‌های زیر است.  

1. **شناسه**
2. **نام**
3. **جنسیت**
4. **تجربه**
5. **حقوق**

حالا می‌خواهم متدی در کلاس Employee بنویسم که بتواند برای ارتقای کارمندان استفاده شود. متدی که قرار است بنویسیم، لیستی از اشیاء Employee را به عنوان پارامتر می‌گیرد و سپس نام تمام کارمندانی را که واجد شرایط ارتقای شغلی هستند، چاپ می‌کند.

اما منطقی که بر اساس آن کارمند ارتقا می‌یابد نباید به صورت کدنویسی شده باشد. گاهی اوقات ممکن است کارمندان را بر اساس تجربه‌شان ارتقا دهیم و گاهی اوقات ممکن است کارمندان را بر اساس حقوقشان یا شاید بر اساس دلایل دیگری مانند عملکرد کارمند ارتقا دهیم. بنابراین، منطق ارتقا کارمندان نباید در داخل متد به صورت کدنویسی شده باشد.

چگونه می‌توانیم به این هدف دست یابیم؟ این کار را می‌توان به راحتی با کمک delegateها انجام داد. بنابراین، اکنون باید کلاس Employee خود را مطابق شکل زیر طراحی کنم.

```csharp
using System;
using System.Collections.Generic;

namespace DelegateRealtimeExample
{
    public delegate bool EligibleToPromotion(Employee EmployeeToPromotion);

    public class Employee
    {
        public int ID { get; set; }
        public string Name { get; set; }
        public string Gender { get; set; }
        public int Experience { get; set; }
        public int Salary { get; set; }

        public static void PromoteEmployee(List<Employee> lstEmployees, EligibleToPromotion IsEmployeeEligible)
        {
            foreach (Employee employee in lstEmployees)
            {
                if (IsEmployeeEligible(employee))
                {
                    Console.WriteLine($"Employee {employee.Name} Promoted");
                }
            }
        }
    }
}
```

در کد بالا، ما یک delegate به نام EligibleToPromotion ایجاد کرده‌ایم. این delegate شیء Employee را به عنوان پارامتر می‌گیرد و یک مقدار بولی برمی‌گرداند که نشان می‌دهد آیا کارمند ارتقا یافته است یا خیر. در کلاس Employee، ما همچنین متد PromoteEmpoloyee را ایجاد کرده‌ایم و این متد لیستی از Employees و یک Delegate از نوع EligibleToPromotion را به عنوان پارامتر می‌گیرد.

سپس، متد PromoteEmployee شیء هر کارمند را پیمایش کرده و آن را به delegate ارسال می‌کند. اگر delegate مقدار true را برگرداند، Employee ارتقا می‌یابد، در غیر این صورت ارتقا نمی‌یابد. بنابراین، درون متد، هیچ منطقی در مورد نحوه‌ی ارتقای کارمندان تعریف نکرده‌ایم.

اکنون کلاینتی که از کلاس Employee استفاده می‌کند، انعطاف‌پذیری لازم برای تعیین منطق نحوه‌ی ارتقای کارمندان خود را دارد. ابتدا اشیاء کارمند - emp1، emp2 و emp3 - را ایجاد کنید. ویژگی‌های اشیاء مربوطه را پر کنید. سپس یک لیست کارمند ایجاد کنید تا هر 3 کارمند را همانطور که در زیر نشان داده شده است، در خود جای دهد.

```csharp
using System;
using System.Collections.Generic;

namespace DelegateRealtimeExample
{
    public class Program
    {
        static void Main()
        {
            Employee emp1 = new Employee()
            {
                ID = 101,
                Name = "Pranaya",
                Gender = "Male",
                Experience = 5,
                Salary = 10000
            };

            Employee emp2 = new Employee()
            {
                ID = 102,
                Name = "Priyanka",
                Gender = "Female",
                Experience = 10,
                Salary = 20000
            };

            Employee emp3 = new Employee()
            {
                ID = 103,
                Name = "Anurag",
                Experience = 15,
                Salary = 30000
            };

            List<Employee> lstEmployess = new List<Employee>();
            lstEmployess.Add(emp1);
            lstEmployess.Add(emp2);
            lstEmployess.Add(emp3);

            EligibleToPromotion eligibleTopromote = new EligibleToPromotion(Program.Promote);
            Employee.PromoteEmployee(lstEmployess, eligibleTopromote);

            Console.ReadKey();
        }

        public static bool Promote(Employee employee)
        {
            if (employee.Salary > 10000)
            {
                return true;
            }
            else
            {
                return false;
            }
        }
    }
}
```

به متد Promote که ایجاد کرده‌ایم توجه کنید. این متد منطق چگونگی ارتقای کارمندان ما را دارد. سپس این متد به عنوان پارامتر به delegate ارسال می‌شود. همچنین، توجه داشته باشید که این متد امضایی مشابه با delegate مربوط به EligibleToPromotion دارد. این بسیار مهم است زیرا اگر امضا متفاوت باشد، نمی‌توان متد Promote را به عنوان پارامتر به delegate ارسال کرد. به همین دلیل است که delegateها، اشاره‌گرهای تابع type-safe نامیده می‌شوند. اکنون، برنامه را اجرا کنید و خروجی زیر را مشاهده خواهید کرد.

بنابراین، اگر مفهوم نمایندگان (delegates) را نداشتیم، ارسال یک تابع به عنوان پارامتر امکان‌پذیر نبود. از آنجایی که متد Promote در کلاس Employee از نمایندگان (delegates) استفاده می‌کند، می‌توان به صورت پویا منطق چگونگی ارتقای کارمندان را تعیین کرد.

در C# 3.0 عبارات لامبدا معرفی شده‌اند. بنابراین، می‌توانید به جای ایجاد یک تابع و سپس نمونه‌ای از یک delegate و سپس ارسال تابع به عنوان پارامتر به delegate، از عبارات لامبدا استفاده کنید. مثال زیر با استفاده از عبارت لامبدا بازنویسی شده است. اکنون دیگر نیازی به متد Private Promote نیست.

```csharp
using System;
using System.Collections.Generic;

namespace DelegateRealtimeExample
{
    public class Program
    {
        static void Main()
        {
            Employee emp1 = new Employee()
            {
                ID = 101,
                Name = "Pranaya",
                Gender = "Male",
                Experience = 5,
                Salary = 10000
            };

            Employee emp2 = new Employee()
            {
                ID = 102,
                Name = "Priyanka",
                Gender = "Female",
                Experience = 10,
                Salary = 20000
            };

            Employee emp3 = new Employee()
            {
                ID = 103,
                Name = "Anurag",
                Experience = 15,
                Salary = 30000
            };

            List<Employee> lstEmployess = new List<Employee>();
            lstEmployess.Add(emp1);
            lstEmployess.Add(emp2);
            lstEmployess.Add(emp3);

            Employee.PromoteEmployee(lstEmployess, x => x.Experience > 5);
            Console.ReadKey();
        }
    }
}
```
