##### **نماینده (Delegate) در سی شارپ چیست؟**

یک نماینده (Delegate) یک اشاره‌گر تابع از نوع ایمن (Type-Safe) است **.** این بدان معناست که نماینده مرجع یک متد یا تابع را در خود نگه می‌دارد و وقتی نماینده را فراخوانی می‌کنیم، متدی که به آن اشاره می‌کند اجرا خواهد شد. امضای نماینده و متدی که به آن اشاره می‌کند باید امضای یکسانی داشته باشند. هنگام ایجاد نمونه نماینده، باید متد را به عنوان پارامتر به سازنده نماینده ارسال کنیم. باز هم، دو نوع نماینده در سی شارپ وجود دارد. آنها به شرح زیر هستند:

1. **نماینده تک‌ریختی (Single Cast Delegate):** نماینده به یک تابع یا متد واحد اشاره دارد.
2. **نماینده چندپخشی:** نماینده به چندین تابع یا روش اشاره دارد.

 امروز، قصد داریم Multicast Delegate ها را در C# مورد بحث قرار دهیم.

##### **نماینده چندپخشی (Multicast Delegate) در سی شارپ چیست؟**

یک نماینده چندپخشی (Multicast Delegate) در سی شارپ، نماینده‌ای است که ارجاعات بیش از یک تابع کنترل‌کننده را در خود نگه می‌دارد. وقتی نماینده چندپخشی را فراخوانی می‌کنیم، تمام توابعی که توسط نماینده ارجاع داده می‌شوند، فراخوانی می‌شوند. اگر می‌خواهید چندین متد را با استفاده از یک نماینده فراخوانی کنید، امضای همه متدها باید یکسان باشد.

بنابراین، یک نماینده چندپخشی فقط آرایه‌ای از چندین خط لوله یا چندین نماینده است. نمایندگان به همان ترتیبی که در لیست فراخوانی قرار گرفته‌اند، فراخوانی می‌شوند. یک InvocationList چیزی جز آرایه‌ای از نمایندگان یا خطوط لوله نیست که در آن هر خط لوله داده‌ها را در یک متد متفاوت قرار می‌دهد. اگر در حال حاضر این موضوع برایتان واضح نیست، نگران نباشید، سعی خواهیم کرد با مثال‌های متعدد این موضوع را درک کنیم.

##### **مثالی برای درک Multicast Delegate در سی شارپ**

بیایید مثالی برای درک Multicast Delegate در سی شارپ ببینیم. لطفاً به مثال زیر که بدون استفاده از delegates است، نگاهی بیندازید. در مثال زیر، دو متد ایجاد کرده‌ایم و سپس از متد main، نمونه‌ای از کلاس ایجاد می‌کنیم و سپس دو متد را فراخوانی می‌کنیم.

```csharp
using System;

namespace MulticastDelegateDemo
{
    public class Rectangle
    {
        public void GetArea(double Width, double Height)
        {
            Console.WriteLine($"Area is {Width * Height}");
        }
        public void GetPerimeter(double Width, double Height)
        {
            Console.WriteLine($"Perimeter is {2 * (Width + Height)}");
        }
        static void Main(string[] args)
        {
            Rectangle rect = new Rectangle();
            rect.GetArea(23.45, 67.89);
            rect.GetPerimeter(23.45, 67.89);
            Console.ReadKey();
        }
    }
}
```

در مثال بالا، ما یک نمونه از کلاس Rectangle ایجاد کردیم و سپس دو متد را فراخوانی کردیم. حالا می‌خواهم یک delegate واحد ایجاد کنم که قرار است دو متد فوق (یعنی GetArea و GetPerimeter) را فراخوانی کند. این دو متد امضای یکسانی با نام‌های متد متفاوت دارند، بنابراین می‌توانیم یک delegate واحد ایجاد کنیم که مرجع دو متد فوق را در خود نگه دارد. و وقتی delegate را فراخوانی می‌کنیم، قرار است دو متد فوق را فراخوانی کند. و وقتی این کار را انجام می‌دهیم، در C# به آن Multicast Delegate گفته می‌شود.

##### **مثال نماینده چندپخشی در سی شارپ:**

در مثال زیر، ما یک delegate ایجاد کرده‌ایم که امضای آن با دو متد GetArea و GetPerimeter یکسان است. سپس نمونه‌ای از delegate ایجاد کرده‌ایم و دو متد را با استفاده از عملگر += به هم متصل کرده‌ایم. به طور مشابه، می‌توانید از عملگر -= برای حذف یک تابع از delegate استفاده کنید. هنگامی که دو متد را با نمونه delegate متصل می‌کنیم و وقتی delegate را فراخوانی می‌کنیم، هر دو متد اجرا می‌شوند. در این حالت، در پشت صحنه، وقتی چندین متد را به delegate اضافه می‌کنیم، چندین pipeline اضافه می‌شوند. به عبارت دیگر، اکنون می‌توانیم بگوییم که InvocationList اکنون شامل دو delegate یا دو pipeline به همان ترتیبی است که متدها را اضافه می‌کنیم. در این حالت، delegate یا pipeline اول داده‌ها را در متد GetArea و pipeline دوم داده‌ها را در متد GetPerimeter قرار می‌دهد و وقتی برنامه را اجرا می‌کنید، خواهید دید که ابتدا متد GetArea اجرا می‌شود و سپس متد GetPerimeter اجرا می‌شود. و در InvocationList، خواهید دید که در این مثال دو pipeline یا delegate با نام یکسان داریم.

```csharp
using System;

namespace MulticastDelegateDemo
{
    public delegate void RectangleDelegate(double Width, double Height);
    public class Rectangle
    {
        public void GetArea(double Width, double Height)
        {
            Console.WriteLine($"Area is {Width * Height}");
        }
        public void GetPerimeter(double Width, double Height)
        {
            Console.WriteLine($"Perimeter is {2 * (Width + Height)}");
        }
        static void Main(string[] args)
        {
            Rectangle rect = new Rectangle();
            RectangleDelegate rectDelegate = new RectangleDelegate(rect.GetArea);
            // RectangleDelegate rectDelegate = rect.GetArea;

            // binding a method with delegate object
            // In this example rectDelegate is a multicast delegate. 
            // You use += operator to chain delegates together.

            rectDelegate += rect.GetPerimeter;

            Delegate[] InvocationList = rectDelegate.GetInvocationList();
            Console.WriteLine("InvocationList:");
            foreach (var item in InvocationList)
            {
                Console.WriteLine($"  {item}");
            }

            Console.WriteLine();
            Console.WriteLine("Invoking Multicast Delegate:");
            rectDelegate(23.45, 67.89);
            //rectDelegate.Invoke(13.45, 76.89);
            
            Console.WriteLine();
            Console.WriteLine("Invoking Multicast Delegate After Removing one Pipeline:");
            //Removing a method from delegate object
            rectDelegate -= rect.GetPerimeter;
            rectDelegate.Invoke(13.45, 76.89);

            Console.ReadKey();
        }
    }
}
```

##### **رویکرد دیگر برای ایجاد نمایندگان چندپخشی در سی شارپ:**

در مثال زیر، استفاده از متدهای استاتیک و غیراستاتیک را به همراه روش‌های مختلف ایجاد و فراخوانی نماینده‌های چندپخشی در سی‌شارپ به شما نشان خواهم داد. لطفاً به مثال زیر نگاهی بیندازید. در اینجا، ما یک نماینده ایجاد کرده‌ایم که دو پارامتر عدد صحیح می‌گیرد و چیزی برنمی‌گرداند. سپس در کلاس برنامه، چهار متد تعریف می‌کنیم و هر چهار متد دو پارامتر عدد صحیح می‌گیرند و چیزی برنمی‌گردانند، یعنی void. سپس چهار نمونه از نماینده ایجاد می‌کنیم و چهار متد را به آن متصل می‌کنیم. در نهایت، نمونه نماینده پنجم را ایجاد می‌کنیم و به این نمونه، هر چهار نمونه نماینده را با استفاده از عملگر + متصل می‌کنیم. اکنون، نماینده پنجم به یک نماینده چندپخشی تبدیل می‌شود. در این حالت، برای نماینده ۵، InvocationList دارای ۴ نماینده یا می‌توان گفت ۴ خط لوله است که هر خط لوله یا نماینده قرار است داده‌ها را در یک متد متفاوت بریزد. و هنگامی که نمونه نماینده پنجم را فراخوانی می‌کنیم، هر چهار متد اجرا می‌شوند. اگر می‌خواهید اتصال یک متد را حذف کنید، کافیست از عملگر -= استفاده کنید و نمونه نماینده‌ای را که می‌خواهید حذف کنید، مشخص کنید.

```csharp
using System;

namespace MulticastDelegateDemo
{
    public delegate void MathDelegate(int No1, int No2);

    public class Program
    {
        //Static Method
        public static void Add(int x, int y)
        {
            Console.WriteLine($"Addition of {x} and {y} is : {x + y}");
        }
        //Static Method
        public static void Sub(int x, int y)
        {
            Console.WriteLine($"Subtraction of {x} and {y} is : {x - y}");
        }
        //Non-Static Method
        public void Mul(int x, int y)
        {
            Console.WriteLine($"Multiplication of {x} and {y} is : {x * y}");
        }
        //Non-Static Method
        public void Div(int x, int y)
        {
            Console.WriteLine($"Division of {x} and {y} is : {x / y}");
        }

        static void Main(string[] args)
        {
            Program p = new Program();
            //Static Method within the same class can be access directly
            MathDelegate del1 = new MathDelegate(Add);
            //Static Method can be access using class name
            MathDelegate del2 = new MathDelegate(Program.Sub);
            //Non-Static Method must be access through object 
            MathDelegate del3 = new MathDelegate(p.Mul); 
            MathDelegate del4 = new MathDelegate(p.Div); ;

            //In this example del5 is a multicast delegate. 
            //We can use +(plus) operator to chain delegates together and 
            //-(minus) operator to remove a delegate.
            MathDelegate del5 = del1 + del2 + del3 + del4;

            Delegate[] InvocationList = del5.GetInvocationList();
            Console.WriteLine("InvocationList:");
            foreach (var item in InvocationList)
            {
                Console.WriteLine($" {item}");
            }
            Console.WriteLine();

            Console.WriteLine("Invoking Multicast Delegate::");
            del5.Invoke(20, 5);
            Console.WriteLine();

            Console.WriteLine("Invoking Multicast Delegate After Removing one Delegate:");
            del5 -= del2;
            del5(22, 7);

            Console.ReadKey();
        }
    }
}
```

##### **نمایندگان چندپخشی با نوع بازگشتی در سی شارپ:**

یک نماینده چندپخشی، متدها را به همان ترتیبی که به فهرست فراخوانی اضافه شده‌اند و قبلاً دیده‌ایم، فراخوانی می‌کند. تاکنون، مثال‌هایی که با استفاده از نماینده چندپخشی مورد بحث قرار داده‌ایم، چیزی برنمی‌گردانند، یعنی نوع بازگشتی void است. حال، اگر نماینده نوع بازگشتی داشته باشد، چه اتفاقی می‌افتد؟

اگر نماینده نوع بازگشتی غیر از void داشته باشد، و اگر نماینده یک نماینده چندپخشی باشد، فقط مقدار آخرین متد فراخوانی شده بازگردانده می‌شود. به همین ترتیب، اگر نماینده دارای پارامتر out باشد، مقدار پارامتر خروجی، مقداری خواهد بود که توسط آخرین متد فراخوانی شده از فهرست فراخوانی اختصاص داده شده است.

##### **مثالی برای درک Multicast Delegates با نوع بازگشتی در سی شارپ**

بیایید با یک مثال، Delegate های چندپخشی با نوع بازگشتی در سی شارپ را درک کنیم. لطفاً به مثال زیر نگاهی بیندازید. در اینجا، ما یک delegate ایجاد کردیم که هیچ پارامتری نمی‌گیرد اما نوع بازگشتی آن int است. سپس دو متد استاتیک ایجاد کردیم، اولین متد استاتیک مقدار ۱ و دومین متد استاتیک مقدار ۲ را برمی‌گرداند. سپس نمونه delegate را ایجاد کردیم و ابتدا MethodOne و سپس MethodTwo را bind کردیم. و وقتی delegate را فراخوانی می‌کنیم، ابتدا MethodOne اجرا می‌شود و سپس MethodTwo اجرا می‌شود و مقدار ۲ را برمی‌گرداند، زیرا آخرین متد فراخوانی شده در InvocationList، MethodTwo است که مقدار ۲ را برمی‌گرداند.

```csharp
using System;

namespace MulticastDelegateDemo
{
    // Deletegate's return type is int
    public delegate int SampleDelegate();
    public class Program
    {
        static void Main()
        {
            SampleDelegate del = new SampleDelegate(MethodOne);
            del += MethodTwo;
            
            // The Value Returned By Delegate will be 2, returned by the MethodTwo(),
            // as it is the last method in the invocation list.
            int ValueReturnedByDelegate = del();
            Console.WriteLine($"Returned Value = {ValueReturnedByDelegate}");

            Console.ReadKey();
        }
        // This method returns one
        public static int MethodOne()
        {
            Console.WriteLine("MethodOne is Executed");
            return 1;
        }

        // This method returns two
        public static int MethodTwo()
        {
            Console.WriteLine("MethodTwo is Executed");
            return 2;
        }
    }
}
```

##### **مثالی برای درک Multicast Delegates با استفاده از پارامتر out در سی شارپ:**

اکنون مثالی از یک نماینده چندپخشی (Multicast delegate) در سی شارپ با استفاده از پارامتر out را خواهیم دید. لطفاً به مثال زیر نگاهی بیندازید. در اینجا، ما یک نماینده ایجاد کرده‌ایم که یک پارامتر out می‌گیرد و چیزی برنمی‌گرداند، یعنی void. سپس دو متد استاتیک ایجاد کرده‌ایم و هر دو متد استاتیک یک پارامتر out می‌گیرند. متد استاتیک اول مقدار ۱ را به پارامتر out اختصاص می‌دهد و متد استاتیک دوم مقدار ۲ را به پارامتر out اختصاص می‌دهد. سپس نمونه نماینده را ایجاد کرده‌ایم و ابتدا MethodOne و سپس MethodTwo را bind می‌کنیم. و وقتی نماینده را فراخوانی می‌کنیم، ابتدا MethodOne اجرا می‌شود و سپس MethodTwo اجرا می‌شود و ۲ را برمی‌گرداند زیرا آخرین متد فراخوانی شده MethodTwo است که مقدار ۲ را به پارامتر out اختصاص می‌دهد.

```csharp
using System;

namespace MulticastDelegateDemo
{
    // Deletegate has an int output parameter
    public delegate void SampleDelegate(out int Integer);

    public class Program
    {
        static void Main()
        {
            SampleDelegate del = new SampleDelegate(MethodOne);
            del += MethodTwo;

            // The ValueFromOutPutParameter will be 2, initialized by MethodTwo(),
            // as it is the last method in the invocation list.
            int ValueFromOutPutParameter = -1;
            del(out ValueFromOutPutParameter);

            Console.WriteLine($"Returned Value = {ValueFromOutPutParameter}");
            Console.ReadKey();
        }

        // This method sets ouput parameter Number to 1
        public static void MethodOne(out int Number)
        {
            Console.WriteLine("MethodOne is Executed");
            Number = 1;
        }

        // This method sets ouput parameter Number to 2
        public static void MethodTwo(out int Number)
        {
            Console.WriteLine("MethodTwo is Executed");
            Number = 2;
        }
    }
}
```
