##### **چگونه چندین بلوک catch را در سی شارپ پیاده سازی کنیم؟**

در سی شارپ می‌توان چندین بلوک catch برای یک بلوک try مشخص نوشت. وقتی چندین بلوک catch را در سی شارپ برای یک بلوک try مشخص پیاده‌سازی می‌کنیم، در هر نقطه از زمان فقط یک بلوک catch اجرا می‌شود و سایر بلوک‌های catch نادیده گرفته می‌شوند. با در نظر گرفتن این نکته، بیایید مثالی از نحوه پیاده‌سازی چندین بلوک catch در سی شارپ را بررسی کنیم.

##### **مثالی برای درک بلوک‌های catch چندگانه در سی شارپ**

بیایید یک مثال را بررسی کنیم و سعی کنیم نحوه پیاده‌سازی چندین بلوک catch برای یک بلوک try مشخص در C# را درک کنیم و همچنین سعی کنیم جریان اجرا را درک کنیم. لطفاً به مثال زیر نگاهی بیندازید. همانطور که می‌بینید، در اینجا، ما دو بلوک catch برای بلوک try مشخص ایجاد کرده‌ایم. بلوک catch اول، کلاس DivideByZeroException را به عنوان پارامتر ورودی و بلوک catch دوم، کلاس FormatException را به عنوان پارامتر ورودی می‌گیرد.

```csharp
using System;

namespace ExceptionHandlingDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int Number1, Number2, Result;
            try
            {
                Console.WriteLine("Enter First Number");
                Number1 = int.Parse(Console.ReadLine());
                Console.WriteLine("Enter Second Number");
                Number2 = int.Parse(Console.ReadLine());
                Result = Number1 / Number2;
                Console.WriteLine($"Result: {Result}");
            }
            catch (DivideByZeroException DBZE)
            {
                Console.WriteLine("Second Number Should Not Be Zero");
            }
            catch (FormatException FE)
            {
                Console.WriteLine("Enter Only Integer Numbers");
            }
            Console.ReadKey();
        }
    }
}
```

**آیا می‌توان با استفاده از یک بلوک catch واحد در C# همه استثنائات را دریافت کرد؟**

بله، این امکان وجود دارد. ما می‌توانیم همه استثنائات را با یک بلوک catch واحد با پارامتر Exception دریافت کنیم. کلاس Exception کلاس اصلی همه کلاس‌های Child Exception است و از این رو می‌تواند انواع استثنائاتی را که در بلوک try رخ می‌دهند، مدیریت کند. ما باید از این بلوک catch فقط برای متوقف کردن خاتمه غیرطبیعی، صرف نظر از استثنائاتی که از بلوک try مربوطه رخ می‌دهند، استفاده کنیم. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید.

```csharp
using System;

namespace ExceptionHandlingDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int Number1, Number2, Result;
            try
            {
                Console.WriteLine("Enter First Number");
                Number1 = int.Parse(Console.ReadLine());
                Console.WriteLine("Enter Second Number");
                Number2 = int.Parse(Console.ReadLine());
                Result = Number1 / Number2;
                Console.WriteLine($"Result: {Result}");
            }
            catch (Exception ex)
            {
                Console.WriteLine("Generic Catch Block...");
            }
            Console.ReadKey();
        }
    }
}
```

حال، در مثال بالا، برای هر نوع استثنا، بلوک catch عمومی اجرا خواهد شد. همیشه توصیه می‌شود که یک بلوک catch با پارامتر Exception به عنوان آخرین بلوک catch بنویسید، حتی اگر چندین بلوک catch می‌نویسیم. این بلوک catch به عنوان یک بلوک catch پشتیبان عمل می‌کند. در ادامه، سینتکس انجام همین کار آمده است.

در مثال زیر، ما بلوک catch عمومی را به عنوان آخرین بلوک catch می‌نویسیم و اگر هر خطایی که در بلوک try رخ می‌دهد توسط هیچ بلوک catch خاصی مدیریت نشود، بلوک generic اجرا خواهد شد.

```csharp
using System;

namespace ExceptionHandlingDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int Number1, Number2, Result;
            try
            {
                Console.WriteLine("Enter First Number");
                Number1 = int.Parse(Console.ReadLine());
                Console.WriteLine("Enter Second Number");
                Number2 = int.Parse(Console.ReadLine());
                Result = Number1 / Number2;
                Console.WriteLine($"Result: {Result}");
            }
            
            catch (DivideByZeroException DBZE)
            {
                Console.WriteLine("Second Number Should Not Be Zero");
            }
            catch (FormatException FE)
            {
                Console.WriteLine("Enter Only Integer Numbers");
            }
            catch (Exception ex)
            {
                Console.WriteLine("Generic Catch Block...");
            }
            Console.ReadKey();
        }
    }
}
```
