
## **مدیریت استثنا در سوءاستفاده در سی شارپ با یک مثال**

در این مقاله، قصد دارم در مورد **سوءاستفاده از مدیریت استثنا در سی شارپ** با یک مثال پرداختیم، مطالعه کنید.

##### **مدیریت استثنائات و سوءاستفاده در سی شارپ:**

استثناها چیزی جز کلاس‌هایی نیستند که اجرای برنامه را به طور غیرعادی متوقف می‌کنند، زمانی که خطاهای زمان اجرا در طول اجرای برنامه رخ می‌دهند. به عنوان مثال، هنگامی که یک برنامه در حال اجرای یک پرس و جو از پایگاه داده است، به دلایلی اتصال پایگاه داده از بین می‌رود، در این صورت با خطای زمان اجرای SQLException مواجه خواهیم شد. فرض کنید، ما سعی می‌کنیم داده‌ها را از یک فایل متنی دریافت کنیم و آن فایل متنی وجود ندارد، در این صورت با خطای File Not Found مواجه خواهیم شد. بنابراین، مدیریت استثنا معمولاً برای مدیریت این سناریوها استفاده می‌شود.

اما گاهی اوقات، به عنوان یک برنامه‌نویس، ما از مکانیزم‌های مدیریت استثنا برای پیاده‌سازی منطق برنامه‌نویسی استفاده می‌کنیم که کار بدی است و به این کار سوءاستفاده از مدیریت استثنا در سی‌شارپ می‌گویند.

##### **مثال: در اینجا ما از مدیریت استثنا برای پیاده‌سازی جریان منطقی استفاده می‌کنیم:**

در مثال زیر، ما از مکانیزم مدیریت استثنا برای پیاده‌سازی منطق کسب‌وکار استفاده می‌کنیم. یعنی ما از مدیریت استثنا برای مدیریت قالب ورودی، عملیات تقسیم و مدیریت حداکثر و حداقل طول استفاده می‌کنیم. این چیزی جز مدیریت استثنا نیست. این موارد را می‌توانیم بدون استفاده از مدیریت استثنا مدیریت کنیم.

```csharp
using System;

namespace ExceptionHandlingDemo
{
    class Program
    {
        public static void Main()
        {
            try
            {
                //Convert.ToInt32() می‌تواند در صورت وارد کردن مقدار، استثنای FormatException ایجاد کند.
                // نمی‌توان آن را به عدد صحیح تبدیل کرد. بنابراین به جای آن از int.TryParse() استفاده کنید.
                Console.WriteLine("لطفاً عدد اول را وارد کنید");
                int FirstNumber = Convert.ToInt32(Console.ReadLine());

                Console.WriteLine("لطفاً عدد دوم را وارد کنید");
                int SecondNumber = Convert.ToInt32(Console.ReadLine());

                int Result = FirstNumber / SecondNumber;
                Console.WriteLine($"نتیجه = {Result}");
            }
            catch (FormatException)
            {
                Console.WriteLine("فقط اعداد مجاز هستند!");
            }
            catch (OverflowException)
            {
                Console.WriteLine($"فقط اعداد بین {Int32.MinValue} و {Int32.MaxValue} مجاز هستند");
            }
            catch (DivideByZeroException)
            {
                Console.WriteLine("عدد دوم نمی‌تواند صفر باشد");
            }
            catch (Exception ex)
            {
                Console.WriteLine(ex.Message);
            }
            Console.ReadLine();
        }
    }
}
```

##### **جلوگیری از سوءاستفاده از مدیریت استثنا:**

بیایید همان مثال را که از مدیریت خطا برای کنترل جریان منطقی برنامه استفاده نمی‌کند، بازنویسی کنیم. در مثال زیر، ما از tryParse برای مدیریت قالب ورودی، شرط If Else برای مدیریت عملیات تقسیم و Max و Min Length استفاده می‌کنیم.

```csharp
using System;

namespace ExceptionHandlingDemo
{
    class Program
    {
        public static void Main()
        {
            try
            {
                Console.WriteLine("لطفاً عدد اول را وارد کنید");
                int FirstNumber;
                //int.TryParse() یک استثنا ایجاد نمی‌کند، در عوض مقدار false را برمی‌گرداند.
                //اگر مقدار وارد شده قابل تبدیل به عدد صحیح نباشد
                bool IsValidFirstNumber = int.TryParse(Console.ReadLine(), out FirstNumber);

                if (IsValidFirstNumber)
                {
                    Console.WriteLine("لطفاً عدد دوم را وارد کنید");
                    int SecondNumber;
                    bool isValidSNO = int.TryParse(Console.ReadLine(), out SecondNumber);

                    if (isValidSNO && SecondNumber != 0)
                    {
                        int Result = FirstNumber / SecondNumber;
                        Console.WriteLine($"نتیجه = {Result}");
                    }
                    else
                    {
                        // بررسی می‌کند که آیا عدد دوم صفر است یا خیر، یک خطای دوستانه چاپ می‌کند
                        // پیام به جای اجازه دادن به استثنای DivideByZeroException
                        // پرتاب شدن.
                        if (isValidSNO && SecondNumber == 0)
                        {
                            Console.WriteLine("عدد دوم نمی‌تواند صفر باشد");
                        }
                        else
                        {
                            Console.WriteLine($"فقط اعداد بین {Int32.MinValue} و {Int32.MaxValue} مجاز هستند");
                        }
                    }
                }
                else
                {
                    Console.WriteLine($"فقط اعداد بین {Int32.MinValue} و {Int32.MaxValue} مجاز هستند");
                }
            }
            catch (Exception ex)
            {
                Console.WriteLine(ex.Message);
            }
            Console.ReadLine();
        }
    }
}
```
