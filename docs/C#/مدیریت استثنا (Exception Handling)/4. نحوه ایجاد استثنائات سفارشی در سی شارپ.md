در این مقاله، قصد دارم در مورد **نحوه ایجاد استثنائات سفارشی در سی شارپ** با مثال صحبت کنم. . به عنوان بخشی از این مقاله، قصد داریم نکات زیر را به تفصیل مورد بحث قرار دهیم.

##### **انواع استثناها در سی شارپ:**

قبل از اینکه بفهمیم چگونه می‌توان در سی شارپ استثناهای سفارشی ایجاد کرد یا از استثناهای تعریف‌شده استفاده کرد، ابتدا اجازه دهید انواع مختلف استثناهای موجود را بررسی کنیم. در سی شارپ، استثناها به دو نوع تقسیم می‌شوند. آنها به شرح زیر هستند:

1. **استثنائات سیستمی:** این استثنائات توسط CLR ایجاد می‌شوند.
2. **استثنای برنامه:** این استثناها توسط برنامه‌نویس ایجاد می‌شوند.

برای درک بهتر، لطفاً به تصویر زیر نگاهی بیندازید. کلاس Exception کلاس والد همه کلاس‌های Exception است. کلاس‌های SystemException و ApplicationException از کلاس Exception تعریف شده‌اند.

![انواع استثناها در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/09/word-image-29911-1-5.png "انواع استثناها در سی شارپ")

همانطور که می‌بینید، برای هر دو کلاس SystemException و ApplicationException، والد فقط کلاس Exception است. به طور پیش‌فرض، تمام کلاس‌های System Exception از کلاس SystemException که خود از کلاس Exception ارث‌بری می‌کند، به ارث می‌رسند و اگر ما هر Application Exception مانند Custom Exception یا User-Defined Exception ایجاد کنیم، آن کلاس باید و باید از کلاس Exception ارث‌بری شود، حتی می‌توانیم کلاس‌های Custom Exception را نیز از کلاس ApplicationException ارث‌بری کنیم. قبلاً ما کلاس‌های CustomException را با ارث‌بری از کلاس ApplicationException ایجاد می‌کردیم، اما در حال حاضر، مایکروسافت توصیه می‌کند از کلاس Exception استفاده شود.

##### **استثنائات سیستمی در سی شارپ چیست؟**

استثنایی که به طور ضمنی تحت یک برنامه در زمان اجرا توسط مدیر استثنا (جزئی از CLR) به دلیل برخی اشتباهات منطقی (برخی شرایط از پیش تعریف شده) ایجاد می‌شود، به عنوان استثنای سیستمی شناخته می‌شود. به عنوان مثال، اگر شما یک عدد صحیح را با صفر تقسیم کنید، یک استثنای سیستمی به نام DivideByZero ایجاد می‌شود. به طور مشابه، اگر یک مقدار رشته‌ای الفبایی-عددی را از کاربر دریافت کنید و سعی کنید آن مقدار را در یک متغیر صحیح ذخیره کنید، یک استثنای سیستمی به نام FormatException ایجاد می‌شود. بنابراین، در C#، کلاس‌های استثنای سیستمی زیادی در دسترس هستند. برخی از آنها به شرح زیر هستند:

1. **استثنای تقسیم بر صفر**
2. **خطای IndexOutOfRange**
3. **استثنای قالب**
4. **SQLException**
5. **استثنای ارجاع صفر و غیره**

##### **استثنائات برنامه در سی شارپ چیست؟**

استثنایی که به طور صریح تحت یک برنامه بر اساس شرط خودمان (یعنی شرط تعریف شده توسط کاربر) ایجاد می‌شود، به عنوان یک استثنای کاربردی شناخته می‌شود. به عنوان یک برنامه‌نویس، می‌توانیم استثناهای کاربردی را در هر نقطه زمانی مشخص ایجاد کنیم. به عنوان مثال، نیاز ما این است که هنگام انجام عملیات تقسیم، باید بررسی کنیم که آیا عدد دوم یک عدد فرد است یا خیر، در این صورت باید یک استثنا ایجاد کنیم. این کار را نمی‌توان به طور خودکار توسط CLR مدیریت کرد. سپس به عنوان یک کاربر، باید استثنای سفارشی خود را ایجاد کنیم و باید یک نمونه از کلاس استثنای سفارشی خود ایجاد کنیم و باید آن نمونه استثنای سفارشی را با استفاده از کلمه کلیدی throw به طور صریح بر اساس نیاز تجاری خود ایجاد کنیم.

برای ایجاد یک استثنای برنامه در سی شارپ، باید فرآیند زیر را اتخاذ کنیم. ابتدا باید یک کلاس استثنای سفارشی با ارث بری از کلاس استثنای والد ایجاد کنیم و سپس باید یک نمونه از کلاس استثنای سفارشی ایجاد کنیم و سپس باید آن نمونه را پرتاب کنیم.

![استثنائات برنامه در سی شارپ چیست؟](https://dotnettutorials.net/wp-content/uploads/2022/09/application-exception-in-c.jpeg "استثنائات برنامه در سی شارپ چیست؟")

هنگام ایجاد و پرتاب دستی یک شیء از کلاس Exception، نباید از کلاس‌های خطای سیستمی مانند DivideByZeroException، FormatException، SQLException و غیره استفاده کنیم. در عوض باید کلاس Exception خودمان را ایجاد کرده و نمونه‌ای از کلاس Exception خودمان را ایجاد و پرتاب کنیم.

##### **چرا باید در سی شارپ استثناهای سفارشی ایجاد کنیم؟**

اگر هیچ یک از کلاس‌های خطای موجود در دات‌نت به هدف ما نرسید، باید به سراغ خطاهای سفارشی در سی‌شارپ برویم.

##### **چگونه در سی شارپ کلاس استثنای سفارشی خود را ایجاد کنیم؟**

قبل از ایجاد کلاس Custom Exception، باید تعریف کلاس Exception را ببینیم، زیرا کلاس Custom Exception ما قرار است از کلاس والد Exception به ارث برسد. اگر به تعریف کلاس Exception بروید، موارد زیر را مشاهده خواهید کرد.

![چرا باید در سی شارپ استثناهای سفارشی ایجاد کنیم؟](https://dotnettutorials.net/wp-content/uploads/2022/09/word-image-29911-3-6.png "چرا باید در سی شارپ استثناهای سفارشی ایجاد کنیم؟")

همانطور که می‌بینید، کلاس Exception دارای تعدادی سازنده، تعدادی ویژگی مجازی و غیر مجازی و تعدادی متد مجازی و غیر مجازی است. اعضای مجازی را می‌توانید در فرزند این کلاس Exception بازنویسی کنید و می‌توانید مستقیماً از اعضای غیر مجازی با استفاده از نمونه کلاس فرزند استفاده کنید.

حال، برای ایجاد یک کلاس Custom Exception در سی شارپ، باید مراحل زیر را دنبال کنیم.

1. **مرحله ۱:** یک کلاس جدید تعریف کنید که از کلاس Exception از پیش تعریف شده ارث بری کند، به طوری که کلاس جدید نیز به عنوان یک کلاس Exception عمل کند.
2. **مرحله ۲:** سپس طبق نیاز خود، اعضای مجازی که درون کلاس Exception مانند Message، Source، StackTrace و غیره تعریف شده‌اند را با پیام خطای مورد نیاز، بازنویسی کنید.

بیایید با یک مثال نحوه ایجاد یک استثنای سفارشی در سی شارپ را درک کنیم. یک فایل کلاس با نام OddNumberException.cs ایجاد کنید و سپس کد زیر را در آن کپی و جایگذاری کنید. در اینجا، می‌توانید ببینید که کلاس OddNumberException از کلاس Exception داخلی به ارث رسیده است و در اینجا ما دو ویژگی مجازی یعنی Message و HelpLink را دوباره پیاده‌سازی می‌کنیم. اکنون می‌توانیم یک نمونه از کلاس OddNumberException ایجاد کنیم و اگر ویژگی‌های Message و HelpLink را فراخوانی کنیم، این دو ویژگی فقط از این کلاس اجرا می‌شوند. اما اگر ویژگی‌های Source و StackTrace را فراخوانی کنید، آن ویژگی‌ها فقط از کلاس Exception اجرا می‌شوند زیرا ما این ویژگی‌ها را دوباره پیاده‌سازی نکرده‌ایم. این مفهوم **Method Overriding در سی شارپ** است .

```csharp
using System;

namespace ExceptionHandlingDemo
{
    //Creating our own Exception Class by inheriting Exception class
    public class OddNumberException : Exception
    {
        //Overriding the Message property
        public override string Message
        {
            get
            {
                return "Divisor Cannot be Odd Number";
            }
        }

        //Overriding the HelpLink Property
        public override string HelpLink
        {
            get
            {
                return "Get More Information from here: https://dotnettutorials.net/lesson/create-custom-exception-csharp/";
            }
        }
    }
}
```

حال، طبق منطق کاری‌مان، می‌توانیم به طور صریح یک نمونه از کلاس OddNumberException ایجاد کنیم و می‌توانیم به طور صریح آن نمونه را از کد برنامه‌مان حذف کنیم. برای درک بهتر، لطفاً به کد زیر نگاهی بیندازید. در اینجا، درون متد Main، دو عدد را از کاربر می‌گیریم و سپس بررسی می‌کنیم که آیا عدد دوم فرد است یا خیر. اگر عدد دوم، یعنی مقسوم علیه، فرد باشد، یک نمونه از کلاس OddNumberException ایجاد می‌کنیم و آن نمونه را حذف می‌کنیم. و در بلوک Catch، آن استثنا را مدیریت می‌کنیم و به سادگی ویژگی‌های Message، StackTrace، Source و HellpLink را چاپ می‌کنیم. در اینجا، اگر OddNumberException رخ داده باشد، ویژگی‌های Message و HelpLink از کلاس OddNumberException اجرا می‌شوند و ویژگی‌های Source و StackTrace از کلاس Exception والد از پیش تعریف شده اجرا می‌شوند.

```csharp
using System;

namespace ExceptionHandlingDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int Number1, Number2, Result;
            try
            {
                Console.WriteLine("Enter First Number:");
                Number1 = int.Parse(Console.ReadLine());

                Console.WriteLine("Enter Second Number:");
                Number2 = int.Parse(Console.ReadLine());

                if (Number2 % 2 > 0)
                {
                    //OddNumberException ONE = new OddNumberException();
                    //throw ONE;
                    throw new OddNumberException();
                }
                Result = Number1 / Number2;
                Console.WriteLine(Result);
            }
            catch (OddNumberException one)
            {
                Console.WriteLine($"Message: {one.Message}");
                Console.WriteLine($"HelpLink: {one.HelpLink}");
                Console.WriteLine($"Source: {one.Source}");
                Console.WriteLine($"StackTrace: {one.StackTrace}");
            }

            Console.WriteLine("End of the Program");
            Console.ReadKey();
        }
    }
}
```
