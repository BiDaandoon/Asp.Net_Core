##### **متغیرهای ثابت در سی شارپ:**

در سی شارپ، اگر یک متغیر را با استفاده از کلمه کلیدی const تعریف کنیم، آن متغیر یک متغیر ثابت است و مقدار متغیر ثابت را نمی‌توان پس از تعریف آن تغییر داد. بنابراین، مقداردهی اولیه متغیر ثابت فقط در زمان تعریف آن الزامی است. فرض کنید می‌خواهید یک PI ثابت را در برنامه خود تعریف کنید، می‌توانید متغیر ثابت را به صورت زیر تعریف کنید:

**const float PI = 3.14f;**

اگر در زمان تعریف متغیر const، آن را مقداردهی اولیه نکنید، همانطور که در تصویر زیر نشان داده شده است، با خطای کامپایلر مواجه خواهید شد.

![متغیرهای ثابت در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/07/constant-variables-in-c.png "متغیرهای ثابت در سی شارپ")

همانطور که می‌بینید، گفته شده است که **یک فیلد const نیاز به ارائه یک مقدار دارد** ، به این معنی که هنگام تعریف یک ثابت، مقداردهی اولیه متغیر ثابت الزامی است.

**توجه:** متغیرهای ثابت فقط و فقط یک بار ایجاد می‌شوند. دلیل این امر این است که ما نمی‌توانیم مقادیر ثابت را پس از تعریف آن، یک بار تغییر دهیم. اگر امکان ایجاد چندین کپی از متغیر ثابت وجود داشته باشد، همه آن کپی‌ها مقدار یکسانی را ذخیره می‌کنند که به معنای اتلاف حافظه است. بنابراین، وقتی نمی‌توانیم یک مقدار را تغییر دهیم، و اگر چندین بار یک کپی مشابه ایجاد کنیم، در واقع منابع را هدر می‌دهیم.

رفتار متغیر ثابت مشابه رفتار متغیرهای استاتیک است، یعنی فقط و فقط یک بار در چرخه حیات یک کلاس مقداردهی اولیه می‌شود و برای مقداردهی اولیه یا اجرا به نمونه‌ای از کلاس نیاز ندارد. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید. کد زیر خود توضیح است، بنابراین لطفاً خطوط کامنت را مطالعه کنید.

##### **مثال برای درک متغیر Const در سی شارپ**

بیایید متغیر Const را در سی شارپ با یک مثال درک کنیم. لطفاً به مثال زیر نگاهی بیندازید. همانطور که در کد زیر می‌بینید، ما یک متغیر const تعریف می‌کنیم، یعنی **const float PI = 3.14f؛** و در متد Main، با استفاده از نام کلاس به متغیر const دسترسی پیدا می‌کنیم و می‌توانیم مستقیماً به آن دسترسی داشته باشیم. دلیل این امر این است که const به طور پیش‌فرض static است و به عنوان static نیازی به شیء ندارد، بلکه می‌توان با استفاده از نام کلاس یا مستقیماً در همان کلاس به آن دسترسی پیدا کرد. همچنین می‌توان متغیرهای محلی را به عنوان const تعریف کرد، یعنی می‌توانیم متغیر const را در یک متد نیز تعریف کنیم. در مثال ما، در متد Main، یک متغیر const تعریف می‌کنیم، یعنی **const int Number = 10؛** اما وقتی متغیر const را تعریف می‌کنیم، نمی‌توانیم مقدار آن را تغییر دهیم و اگر سعی کنیم مقدار متغیر const را در سی شارپ تغییر دهیم، با خطای زمان کامپایل مواجه خواهیم شد.

```csharp
using System;

namespace ConstDemo
{
    class Program
    {
        //we need to assign a value to the const variable
        //at the time of const variable declaration else it will
        //give compile time error
        const float PI = 3.14f; //Constant Variable
        static void Main(string[] args)
        {
            //Const variables are static in nature
            //so we can access them by using class name 
            Console.WriteLine(Program.PI);
            //We can also access them directly within the same class
            Console.WriteLine(PI);

            //We can also declare a constant variable within a function
            const int Number = 10;
            Console.WriteLine(Number);

            //Once after declaration we cannot change the value 
            //of a constant variable. So, the below line gives an error
            //Number = 20;

            Console.ReadLine();
        }
    }
}
```

##### **نکاتی که باید هنگام کار با متغیر Const در سی شارپ به خاطر داشته باشید:**

1. کلمه کلیدی **const** » استفاده می‌شود **برای ایجاد یک متغیر « ثابت** . به این معنی که متغیری ایجاد می‌کند که مقدار آن هرگز تغییر نخواهد کرد. به عبارت ساده، می‌توان گفت متغیری که مقدار آن پس از تعریف، دیگر قابل تغییر یا اصلاح نیست، به عنوان یک متغیر ثابت شناخته می‌شود.
2. ثابت‌ها به طور پیش‌فرض ایستا هستند.
3. مقداردهی اولیه یک متغیر ثابت در زمان تعریف آن الزامی است.

##### **متغیرهای فقط خواندنی در سی شارپ**

وقتی یک متغیر را با استفاده از کلمه کلیدی readonly تعریف می‌کنیم، آن متغیر به عنوان یک متغیر فقط خواندنی شناخته می‌شود و این متغیرها را نمی‌توان مانند ثابت‌ها تغییر داد، اما پس از مقداردهی اولیه. این بدان معناست که مقداردهی اولیه یک متغیر فقط خواندنی در زمان تعریف آن اجباری نیست، بلکه می‌توان آنها را تحت سازنده نیز مقداردهی اولیه کرد. این بدان معناست که می‌توانیم مقدار متغیر فقط خواندنی را فقط درون یک سازنده تغییر دهیم.

رفتار متغیرهای فقط خواندنی مشابه رفتار متغیرهای غیر استاتیک در سی شارپ خواهد بود، یعنی فقط پس از ایجاد نمونه از کلاس و یک بار برای هر نمونه از کلاس که ایجاد می‌شود، مقداردهی اولیه می‌شوند. بنابراین، می‌توانیم آن را به عنوان یک متغیر غیر استاتیک در نظر بگیریم و برای دسترسی به متغیرهای فقط خواندنی به یک نمونه نیاز داریم.

##### **مثال برای درک متغیرهای فقط خواندنی در سی شارپ:**

در مثال زیر، متغیر فقط خواندنی x با هیچ مقداری مقداردهی اولیه نشده است، اما وقتی مقدار متغیر را با استفاده از نمونه چاپ می‌کنیم، مقدار پیش‌فرض int یعنی ۰ نمایش داده می‌شود.

```csharp
using System;

namespace TypesOfVariables
{
    internal class Program
    {
        readonly int x; //Readonly Variable
        static void Main(string[] args)
        {
            Program obj1 = new Program();
            //Accessing Readonly variable using instance
            Console.WriteLine($"{obj1.x}");
            Console.Read();
        }
    }
}
```

##### **مقداردهی اولیه متغیر فقط خواندنی در زمان تعریف آن در سی شارپ**

لطفاً به مثال زیر نگاهی بیندازید. همانطور که در کد زیر مشاهده می‌کنید، ما یک متغیر فقط خواندنی تعریف کرده‌ایم، یعنی readonly int number = 5; و در متد Main با استفاده از شیء ReadOnlyExample به متغیر فقط خواندنی دسترسی پیدا می‌کنیم. دلیل این امر این است که متغیرهای فقط خواندنی به طور پیش‌فرض غیر استاتیک هستند و به عنوان غیر استاتیک به یک شیء نیاز دارند. در اینجا، ما متغیر فقط خواندنی را با مقدار ۵ در زمان تعریف آن مقداردهی اولیه کرده‌ایم و هنگام چاپ متغیر number، مقدار ۵ نمایش داده می‌شود.

```csharp
using System;

namespace ReadOnlyDemo
{
    class Program
    {
        readonly int number = 5;
        static void Main(string[] args)
        {
            Program obj = new Program();
            Console.WriteLine(obj.number);
            Console.ReadLine();
        }
    }
}
```

##### **مقداردهی اولیه متغیر فقط خواندنی از طریق سازنده در سی شارپ**

در مثال زیر، ما متغیر فقط خواندنی را از طریق سازنده‌های کلاس مقداردهی اولیه می‌کنیم. می‌توانید متغیرهای فقط خواندنی را مستقیماً در زمان تعریف مقداردهی اولیه کنید یا می‌توانید از طریق سازنده‌های کلاس مقداردهی اولیه کنید. پس از مقداردهی اولیه، دیگر نمی‌توانید مقدار متغیرهای فقط خواندنی را در سی شارپ تغییر دهید.

```csharp
using System;

namespace ReadOnlyDemo
{
    class Program
    {
        readonly int number = 5;

        //You can also initialize through constructor
        public Program()
        {
            number = 20;
        }
        
        static void Main(string[] args)
        {
            Program obj = new Program();
            Console.WriteLine(obj.number);

            //You cannot change the value of a readonly variable once it is initialized
            //The following statement will give us compile time error 
            //obj.number = 20;

            Console.ReadLine();
        }
    }
}
```

##### **متغیرهای فقط خواندنی (Readonly variables) به ازای هر نمونه (Instance) یک بار ایجاد می‌شوند:**

رفتار متغیرهای فقط خواندنی بسیار شبیه به رفتار متغیرهای غیر استاتیک در C# است، یعنی فقط پس از ایجاد نمونه از کلاس و یک بار برای هر نمونه از کلاس که ایجاد می‌شود، مقداردهی اولیه می‌شوند. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید. در اینجا، سازنده یک پارامتر می‌گیرد و مقدار آن پارامتر را که برای مقداردهی اولیه متغیر فقط خواندنی استفاده کرده‌ایم، دریافت می‌کند. و در داخل متد Main، ما دو نمونه از کلاس ایجاد می‌کنیم و هر نمونه از متغیر فقط خواندنی را با مقدار متفاوتی مقداردهی اولیه می‌کنیم.

```csharp
using System;

namespace ReadOnlyDemo
{
    class Program
    {
        readonly int number;

        //Initialize Readonly Variable through constructor
        public Program(int num)
        {
            number = num;
        }
        
        static void Main(string[] args)
        {
            Program obj1 = new Program(100);
            Console.WriteLine(obj1.number);

            //You cannot change the value of a readonly variable once it is initialized
            //The following statement will give us compile time error 
            //obj1.number = 20;

            Program obj2 = new Program(200);
            Console.WriteLine(obj2.number);

            Console.ReadLine();
        }
    }
}
```

##### **نکاتی که باید هنگام کار با متغیرهای فقط خواندنی در سی شارپ به خاطر داشته باشید:**

1. متغیری که با استفاده از **کلمه کلیدی readonly** ایجاد می‌شود، در سی شارپ به عنوان متغیر فقط خواندنی شناخته می‌شود. مقدار متغیر فقط خواندنی پس از مقداردهی اولیه، دیگر قابل تغییر نیست.
2. مقداردهی اولیه متغیر فقط خواندنی مانند یک ثابت، در زمان تعریف آن اجباری یا الزامی نیست. می‌توانید متغیرهای فقط خواندنی را تحت یک سازنده مقداردهی اولیه کنید، اما نکته مهم این است که پس از مقداردهی اولیه، نمی‌توانید مقدار متغیر فقط خواندنی را خارج از سازنده تغییر دهید.
3. رفتار یک متغیر فقط خواندنی مشابه رفتار یک متغیر غیر استاتیک است. یعنی برای هر شیء یک کپی جداگانه نگهداری می‌کند. تنها تفاوت بین این دو این است که مقدار متغیر غیر استاتیک را می‌توان از خارج از سازنده تغییر داد در حالی که مقدار متغیر فقط خواندنی را نمی‌توان از خارج از بدنه سازنده تغییر داد.

##### **تفاوت بین متغیر ثابت و فقط خواندنی در C# چیست؟**

تفاوت بین یک متغیر ثابت و فقط خواندنی در سی شارپ این است که یک متغیر ثابت، یک مقدار ثابت برای کل کلاس است در حالی که فقط خواندنی، یک مقدار ثابت مختص به یک نمونه از کلاس و برای هر نمونه است.

##### **مثال متغیرهای ثابت، فقط خواندنی، استاتیک و غیر استاتیک در سی شارپ:**

بیایید متغیرهای Const، Readonly، Static و Non-Static را در سی شارپ با یک مثال درک کنیم. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید.

```csharp
using System;

namespace ConstReadOnlyStaticNonStaticDemo
{
    internal class Program
    {
        const float PI = 3.14f; //Constant Variable
        static int x = 100; //Static Variable
        //We are going to initialize variable y and z through constructor
        int y; //Non-Static or Instance Variable
        readonly int z; //Readonly Variable

        //Constructor
        public Program(int a, int b)
        {
            //Initializing non-static variable
            y = a;
            //Initializing Readonly variable
            z = b;
        }

        static void Main(string[] args)
        {
            //Accessing the static variable without instance
            Console.WriteLine($"x value: {x}");
            //Accessing the constant variable without instance
            Console.WriteLine($"PI value: {PI}");

            //Creating two instances
            Program obj1 = new Program(300, 45);
            Program obj2 = new Program(400, 55);
            //Accessing Non-Static and Readonly variables using instance
            Console.WriteLine($"obj1 y value: {obj1.y} and Readonly z value: {obj1.z}");
            Console.WriteLine($"obj2 y value: {obj2.y} and Readonly z value: {obj2.z}");
            Console.Read();
        }
    }
}
```
