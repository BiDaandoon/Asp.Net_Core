## **توابع تعریف شده توسط کاربر در سی شارپ به همراه مثال**

در این مقاله، قصد دارم **توابع تعریف شده توسط کاربر در زبان سی شارپ را** با مثال بررسی کنم. چهار نوع تابع تعریف شده توسط کاربر در سی شارپ وجود دارد. آنها به شرح زیر هستند:

1. **توابع بدون آرگومان و بدون نوع بازگشتی**
2. **توابع با آرگومان و نوع بدون بازگشت.**
3. **توابع بدون آرگومان و با نوع بازگشتی.**
4. **توابع با آرگومان و با نوع بازگشتی**

بیایید هر یک از این انواع توابع را با مثال‌هایی درک کنیم.

##### **تابعی در سی شارپ بدون ارسال آرگومان و بدون بازگشت مقدار:**

وقتی تابعی هیچ آرگومانی ندارد، هیچ داده‌ای از تابع فراخوانی‌کننده دریافت نمی‌کند. به طور مشابه، وقتی مقداری را برنمی‌گرداند، تابع فراخوانی‌کننده هیچ داده‌ای از تابع فراخوانی‌شده دریافت نمی‌کند. بنابراین، هیچ انتقال داده‌ای بین توابع فراخوانی‌کننده و فراخوانی‌شده وجود ندارد. تابعی که هیچ مقداری را برنمی‌گرداند، نمی‌تواند در یک عبارت استفاده شود. فقط می‌تواند به عنوان یک دستور مستقل استفاده شود.

##### **مثالی برای درک تابع بدون آرگومان ارسالی و بدون مقدار بازگشتی در زبان سی شارپ:**

در مثال زیر، تابع Sum() هیچ پارامتری نمی‌گیرد، یا حتی مقداری را برنمی‌گرداند. نوع مقدار برگشتی تابع void است. از این رو، هیچ مقداری از تابع برگردانده نمی‌شود.

```csharp
using System;

namespace FunctionDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Sum();
            Console.ReadKey();
        }
        static void Sum()
        {
            int x = 10;
            int y = 20;
            int sum = x + y;
            Console.WriteLine($"Sum of {x} and {y} is {sum}");
        }
    }
}
```

##### **هیچ آرگومانی ارسال نمی‌شود اما تابعی در سی شارپ یک مقدار برمی‌گرداند:**

وقتی تابعی هیچ آرگومانی ندارد، هیچ داده‌ای از تابع فراخواننده دریافت نمی‌کند، اما یک مقدار برمی‌گرداند. تابع فراخواننده، داده‌ها را از تابع فراخوانده شده دریافت می‌کند. بنابراین، هیچ انتقال داده‌ای بین تابع فراخواننده و تابع فراخوانده شده وجود ندارد، بلکه انتقال داده از تابع فراخوانده شده به تابع فراخواننده انجام می‌شود. تابع فراخوانده شده به صورت عادی خط به خط اجرا می‌شود تا زمانی که به دستور return برسد.

##### **مثالی برای درک تابعی در زبان سی شارپ که هیچ آرگومانی ارسال نمی‌کند اما مقداری را برمی‌گرداند:**

در مثال زیر، پرانتزهای خالی در **عبارت int Result = Sum();** نشان می‌دهد که هیچ آرگومانی به تابع ارسال نشده است. و مقدار برگردانده شده از تابع به متغیر Result اختصاص داده شده است. در اینجا، تابع Sum() دو عدد را جمع کرده و نتیجه را برمی‌گرداند.

```csharp
using System;

namespace FunctionDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int Result=Sum();
            Console.WriteLine($"Sum is {Result}");
            Console.ReadKey();
        }
        static int Sum()
        {
            int x = 10;
            int y = 20;
            int sum = x + y;
            return sum;
        }
    }
}
```

##### **تابعی در زبان سی شارپ: آرگومان ارسال می‌شود اما مقدار برگشتی ندارد**

وقتی یک تابع آرگومان دارد، داده‌ها را از تابع فراخواننده دریافت می‌کند اما هیچ مقداری را برنمی‌گرداند. بنابراین، انتقال داده بین تابع فراخواننده و تابع فراخوانده شده وجود دارد، اما هیچ انتقال داده‌ای از تابع فراخوانده شده به تابع فراخواننده وجود ندارد. ماهیت ارتباط داده بین تابع فراخواننده و تابع فراخوانده شده با آرگومان‌ها اما بدون مقدار بازگشتی.

##### **مثالی برای درک تابعی در زبان سی شارپ که آرگومان ارسالی دارد اما مقداری را برنمی گرداند:**

در مثال زیر، ما دو مقدار را به تابع Sum ارسال می‌کنیم، اما تابع Sum هیچ مقداری را به تابع اصلی برنمی‌گرداند.

```csharp
using System;

namespace FunctionDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int x = 10, y = 20;
            Sum(x, y);
            Console.ReadKey();
        }
        static void Sum(int x, int y)
        {
            int sum = x + y;
            Console.WriteLine($"Sum is {sum}");
        }
    }
}
```

##### **تابع ارسال آرگومان و بازگرداندن مقدار در زبان سی شارپ:**

یک تابع مستقل و خودکفا باید مانند یک «جعبه سیاه» رفتار کند که ورودی را دریافت کرده و مقداری را به عنوان خروجی ارائه می‌دهد. چنین توابعی ارتباط داده دو طرفه خواهند داشت.

##### **مثالی برای درک تابع ارسال آرگومان و مقدار بازگشتی در زبان سی شارپ:**

```csharp
using System;

namespace FunctionDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int x = 10, y = 20;
            int Result = Sum(x, y);
            Console.WriteLine($"Sum is {Result}");
            Console.ReadKey();
        }
        static int Sum(int x, int y)
        {
            int sum = x + y;
            return sum;
        }
    }
}
```

##### **سربارگذاری تابع (Function Overloading) در سی شارپ چیست؟**

در سی شارپ، می‌توانیم بیش از یک تابع با نام یکسان بنویسیم، اما با آرگومان یا لیست پارامترهای متفاوت، و وقتی این کار را انجام می‌دهیم، به آن سربارگذاری تابع (function overloading) می‌گویند. اجازه دهید این موضوع را با یک مثال درک کنیم.  
**استاتیک از درجه اعتبار ساقط اصلی(){**  
**عدد صحیح a = 10، b = 2، c؛**  
**c = جمع (الف، ب)؛**  
**}**  
این تابع اصلی ما است. درون این تابع، ۳ متغیر تعریف کرده‌ایم. سپس، نتیجه تابع 'add()' را در متغیر 'c' ذخیره می‌کنیم. در ادامه تابع add آمده است.  
**جمع استاتیک عدد صحیح (عدد x، عدد صحیح y){**  
**x + y را برگردان؛**  
**}**  
در اینجا ما هیچ متغیری را تعریف نکرده‌ایم؛ مقدار 'x + y' را برمی‌گرداند. وقتی تابع 'add' را درون تابع اصلی فراخوانی می‌کنیم، a در 'x' و 'b' در 'y' کپی می‌شوند و این دو عدد را جمع می‌کنند و نتیجه در 'c' ذخیره می‌شود. اکنون می‌خواهیم یک تابع دیگر در اینجا بنویسیم،  
**جمع استاتیک عدد صحیح (عدد صحیح x، عدد صحیح y، عدد صحیح z){**  
**x + y + z را برگردان؛**  
**}**  
ما تابع اصلی را به صورت زیر تغییر داده‌ایم.  
استاتیک از درجه اعتبار ساقط اصلی(){  
**عدد صحیح a = 10، b = 2، c، d؛**  
**ج = جمع (الف، ب)؛**  
**d = جمع (الف، ب، ج)؛**  
**}**  
در اینجا ما یک تابع دیگر با همان نام، 'add'، ایجاد کرده‌ایم، اما ۳ پارامتر می‌گیرد. درون تابع اصلی، ' **add(x,y,z)** ' را فراخوانی کرده‌ایم و نتیجه را در متغیر 'd' ذخیره کرده‌ایم. بنابراین، می‌توانیم دو تابع با نام یکسان اما با پارامترهای متفاوت داشته باشیم.

» را فراخوانی می‌کنیم، **بنابراین وقتی « add(a, b)** فراخوانی می‌کند **در واقع «add(int x, int y)» را** و وقتی «add(a, b, c)» را فراخوانی می‌کنیم، در واقع « **add(int x, int y, int z)» را** فراخوانی می‌کند . کامپایلر سی‌شارپ می‌تواند بین این دو تابع تفاوت قائل شود و این مفهوم سربارگذاری تابع در سی‌شارپ است.

##### **مثال برای درک سربارگذاری تابع در سی شارپ:**

```csharp
using System;

namespace FunctionDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int a = 10, b = 2, c, d;
            c = add(a, b);
            Console.WriteLine($"Sum of {a} and {b} is {c}");
            d = add(a, b, c);
            Console.WriteLine($"Sum of {a} and {b} and {c} is {d}");
            Console.ReadKey();
        }
        static int add(int x, int y)
        {
            return x + y;
        }
        static int add(int x, int y, int z)
        {
            return x + y + z;
        }
    }
}
```

##### **مزایای سربارگذاری تابع در سی شارپ**

فایده‌ی این کار این است که لازم نیست هر بار به نام‌های جدید فکر کنیم. از آنجایی که هر دو تابع برای جمع اعداد صحیح هستند، لازم نیست نام‌های مختلفی بدهیم. نوشتن برنامه‌ها آسان است و لازم نیست نام‌های تابع زیادی را به خاطر بسپارید. این فایده‌ای است که اکنون به دست می‌آوریم. بیایید یک تابع جمع دیگر تعریف کنیم که مجموع دو عدد اعشاری را برمی‌گرداند.

**تابع اعشاری استاتیک add(float x, float y){**  
**x + y را برگردان؛**  
**}**

این تابع مقدار نوع اعشاری (float) را برمی‌گرداند. حالا ما دو تابع جمع داریم که تعداد پارامترهای یکسانی می‌گیرند. آیا در سی شارپ چنین چیزی امکان‌پذیر است؟ بله، دو تابع می‌توانند نام و تعداد پارامترهای یکسانی داشته باشند، اما نوع داده پارامترها باید متفاوت باشد. آن‌ها نمی‌توانند یکسان باشند.

بنابراین **تابع int add (int x, int y)** و **تابع float add (float x, float y)** دو تابع متفاوت هستند. در سی شارپ، دو تابع متفاوت نامیده می‌شوند اگر نام یکسانی داشته باشند اما لیست پارامترهای آنها متفاوت باشد.

##### **لیست پارامترها چگونه می‌تواند متفاوت باشد؟**

یا نوع داده پارامتر یا تعداد پارامترها. برای درک بهتر، لطفاً به تصویر زیر نگاهی بیندازید.

![چطور لیست پارامترها می‌تواند متفاوت باشد؟](https://dotnettutorials.net/wp-content/uploads/2022/05/how-the-parameters-list-can-be-different.png "چطور لیست پارامترها می‌تواند متفاوت باشد؟")

در بالا امضاهای توابع مختلف 'جمع' آمده است. حال بیایید بررسی کنیم که کدام معتبر یا کدام نامعتبر است.

1. **تابع int add(int, int)** معتبر است؛ دو پارامتر از نوع 'int' می‌گیرد و مقدار 'int' را برمی‌گرداند.
2. **تابع float add(float, float)** معتبر است زیرا دو پارامتر 'float' می‌گیرد و مقدار 'float' را برمی‌گرداند. این تابع در مقایسه با اولی، همان تعداد پارامتر را می‌گیرد اما نوع داده متفاوتی دارد.
3. **تابع int add(int, int, int)** معتبر است زیرا ۳ پارامتر 'int' می‌گیرد و مقدار 'int' را برمی‌گرداند. این تابع تعداد پارامترهای متفاوتی می‌گیرد اما در مقایسه با اولی، نوع داده‌ی یکسانی دارد.
4. **تابع float add(int, int)** است **نامعتبر** ؛ این تابع مشابه تابع اول است که تعداد پارامترهای یکسانی را دریافت می‌کند و نوع پارامترها نیز یکسان است. بنابراین، این تابع نامعتبر است. فرقی نمی‌کند که تابع چه نوع داده‌ای را برمی‌گرداند. اگر دو تابع تعداد پارامترهای یکسانی داشته باشند و از یک نوع باشند، این تابع نامعتبر است.

##### **مثالی برای درک سربارگذاری تابع در سی شارپ**

```csharp
using System;

namespace FunctionDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int a = 10, b = 2, c, d;
            c = add(a, b);
            Console.WriteLine($"Sum of {a} and {b} is {c}");
            d = add(a, b, c);
            Console.WriteLine($"Sum of {a} and {b} and {c} is {d}");
            Console.WriteLine($"Sum of 10.5 and 25.6 is {add(10.5f, 25.6f)}");
            Console.ReadKey();
        }
        static int add(int x, int y)
        {
            return x + y;
        }
        static int add(int x, int y, int z)
        {
            return x + y + z;
        }
        static float add(float x, float y)
        {
            return x + y;
        }
    }
}
```
