##### **اعضای استاتیک و غیر استاتیک در سی شارپ چیستند؟**

اعضای یک کلاس به دو دسته تقسیم می‌شوند

1. **اعضای استاتیک**
2. **اعضای غیر استاتیک**

اعضایی (متغیرها، سازنده‌ها و متدها) که با استفاده از اصلاح‌کننده استاتیک ایجاد می‌شوند، اعضای استاتیک نامیده می‌شوند و بقیه اعضا، اعضای غیر استاتیک نامیده می‌شوند.

به عبارت دیگر، می‌توانیم اینگونه تعریف کنیم که اعضای یک کلاس که برای مقداردهی اولیه یا اجرا به نمونه نیاز ندارند، به عنوان عضو ایستا شناخته می‌شوند. از سوی دیگر، اعضایی که برای مقداردهی اولیه و اجرا به نمونه‌ای از یک کلاس نیاز دارند، به عنوان اعضای غیر ایستا شناخته می‌شوند.

##### **آشنایی با متغیرهای استاتیک و غیر استاتیک در سی شارپ**

هر زمان که یک متغیر را با استفاده از اصلاحگر static یا وقتی یک متغیر را درون هر بلوک static تعریف می‌کنیم، آن متغیرها، متغیرهای static در نظر گرفته می‌شوند در حالی که بقیه متغیرها، متغیرهای non-static در نظر گرفته می‌شوند.

برای درک بهتر، لطفاً به کد زیر نگاهی بیندازید. در مثال زیر، متغیر y ایستا و متغیر x غیر ایستا است. باز هم، متغیر a درون یک بلوک ایستا تعریف شده است، و از این رو متغیر a به طور پیش‌فرض ایستا است اما یک متغیر محلی است. به طور مشابه، متغیر b درون یک بلوک غیر ایستا تعریف شده است و از این رو یک متغیر غیر ایستا است اما یک متغیر محلی است.

![آشنایی با متغیرهای استاتیک و غیر استاتیک در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2018/07/Understanding-Static-and-Non-Static-Variables-in-C.png "آشنایی با متغیرهای استاتیک و غیر استاتیک در سی شارپ")

اگر می‌خواهید یک متغیر در تمام نمونه‌های یک کلاس مقدار یکسانی داشته باشد، باید آن متغیر را به عنوان یک متغیر استاتیک تعریف کنید. بنابراین، متغیرهای استاتیک قرار است داده‌های سطح برنامه را که برای همه اشیاء یکسان خواهد بود، نگه دارند.

متغیر استاتیک بلافاصله پس از شروع اجرای کلاس مقداردهی اولیه می‌شود در حالی که متغیرهای غیر استاتیک فقط پس از ایجاد شیء کلاس و آن هم برای هر بار ایجاد شیء کلاس مقداردهی اولیه می‌شوند.

یک متغیر استاتیک فقط یک بار در طول چرخه حیات یک کلاس مقداردهی اولیه می‌شود در حالی که یک متغیر غیر استاتیک بسته به تعداد اشیاء ایجاد شده برای آن کلاس، 0 یا n بار مقداردهی اولیه می‌شود.

اگر می‌خواهید به اعضای استاتیک یک کلاس دسترسی داشته باشید، باید مستقیماً در همان کلاس به آنها دسترسی پیدا کنید و از نام کلاس خارج از کلاس استفاده کنید، در حالی که برای دسترسی به اعضای غیر استاتیک به یک نمونه از کلاس نیاز دارید.

##### **مثال برای درک متغیرهای استاتیک و غیر استاتیک در سی شارپ**

برای درک بهتر متغیرهای استاتیک و غیر استاتیک در سی شارپ، مثالی را بررسی می‌کنیم. لطفاً به مثال زیر نگاهی بیندازید. در اینجا، ما دو متغیر ایجاد کرده‌ایم که یکی استاتیک است (یعنی استاتیک int y = 200;) و دیگری غیر استاتیک (یعنی int x;). سپس با استفاده از سازنده کلاس، متغیر غیر استاتیک را مقداردهی اولیه می‌کنیم. علاوه بر این، اگر به متد Main نگاه کنید، ما هم با استفاده از نام کلاس و هم مستقیماً (مانند دسترسی به متغیر استاتیک در همان کلاس) به متغیر استاتیک دسترسی پیدا می‌کنیم و برای دسترسی به متغیر غیر استاتیک، دو نمونه از کلاس ایجاد کرده‌ایم.

```csharp
using System;

namespace StaticNonStaticDemo
{
    class Example
    {
        int x; // Non-Static Variable
        static int y = 200; //Static Variable
        public Example(int x)
        {
            //Initializing Non-Static Variable
            this.x = x;
        }
        static void Main(string[] args)
        {
            //Accessing Static Variable using class name
            //Before Object Creation
            Console.WriteLine($"Static Variable Y = {Example.y}");

            //Accessing Static Variable using without class name
            //It’s possible because we are accessing the variable
            //from the same class
            Console.WriteLine($"Static Variable Y = {y}");

            //Creating Object1
            Example obj1 = new Example(50);

            //Creating Object2
            Example obj2 = new Example(100);

            //Accessing Non-Static Variable using Object of the class
            Console.WriteLine($"Object1 x = {obj1.x} and Object2 x = {obj2.x}");
            Console.ReadLine();
        }
    }
}
```

##### **دامنه متغیرهای استاتیک و غیر استاتیک در سی شارپ چیست؟**

متغیرهای غیر استاتیک هنگام ایجاد شیء ایجاد می‌شوند و با تخریب شیء از بین می‌روند. شیء زمانی از بین می‌رود که متغیر مرجع آن از بین برود یا با null مقداردهی اولیه شود. بنابراین، می‌توانیم بگوییم که دامنه شیء، دامنه متغیرهای مرجع آن است. از سوی دیگر، دامنه متغیر استاتیک، دامنه برنامه است. به این معنی که به محض شروع برنامه و شروع اجرای کلاس، متغیرهای استاتیک ایجاد می‌شوند و تا زمانی که برنامه شما در حال اجرا است، آنجا خواهند بود. به محض اینکه برنامه متوقف شود، متغیرهای استاتیک حذف می‌شوند. بنابراین، دامنه، دامنه برنامه است.

##### **متدهای استاتیک و غیر استاتیک در سی شارپ**

وقتی با استفاده از اصلاحگر static یک متد ایجاد می‌کنیم، آن متد، متد static نامیده می‌شود و بقیه متدها، متدهای non-static هستند. شما نمی‌توانید اعضای non-static را مستقیماً درون یک متد static استفاده کنید. اگر می‌خواهید اعضای non-static را با یک متد static استفاده کنید، باید یک شیء ایجاد کنید و سپس از طریق آن شیء، به اعضای non-static دسترسی پیدا کنید. از طرف دیگر، می‌توانید اعضای static را مستقیماً درون یک متد non-static و بدون هیچ محدودیتی استفاده کنید.

##### **قوانینی که باید هنگام کار با اعضای استاتیک و غیر استاتیک در سی شارپ رعایت کنید:**

1. **تبدیل غیر استاتیک به استاتیک:** اعضای غیر استاتیک فقط با استفاده از شیء آن کلاس در داخل یک بلوک استاتیک قابل استفاده هستند.
2. **استاتیک به استاتیک:** اعضای استاتیک را می‌توان مستقیماً (درون همان کلاس) یا با استفاده از نام کلاس (از خارج از کلاس) در داخل یک بلوک استاتیک دیگر استفاده کرد.
3. **تبدیل استاتیک به غیر استاتیک:** اعضای استاتیک را می‌توان مستقیماً یا با استفاده از نام کلاس درون یک بلوک غیر استاتیک استفاده کرد.
4. **غیر استاتیک به غیر استاتیک:** اعضای غیر استاتیک را می‌توان مستقیماً یا با استفاده از کلمه کلیدی "this" درون یک بلوک غیر استاتیک دیگر استفاده کرد.

##### **مثال برای درک متدهای استاتیک و غیر استاتیک در سی شارپ**

بیایید برای درک بهتر متدهای استاتیک و غیر استاتیک در سی شارپ، مثالی بزنیم. لطفاً به مثال زیر نگاهی بیندازید. در اینجا، ما دو متغیر ایجاد کرده‌ایم. یک متغیر استاتیک است، یعنی متغیر y و متغیر دیگر غیر استاتیک است، یعنی متغیر x. سپس دو متد ایجاد کرده‌ایم، یکی متد Add که یک متد استاتیک است و دیگری متد Mul که یک متد غیر استاتیک است. از متد استاتیک، یک نمونه از کلاس Example ایجاد می‌کنیم و متغیر غیر استاتیک را فراخوانی می‌کنیم و می‌توانیم متغیر استاتیک را مستقیماً یا با استفاده از نام کلاس فراخوانی کنیم. از متد غیر استاتیک، می‌توانیم اعضای غیر استاتیک را مستقیماً یا با استفاده از عملگر "this" و اعضای استاتیک را با استفاده از نام کلاس یا مستقیماً فراخوانی کنیم. مثال زیر خود توضیح است، بنابراین لطفاً خطوط کامنت را مرور کنید.

```csharp
using System;

namespace StaticNonStaticDemo
{
    class Example
    {
        int x = 100;
        static int y = 200;
        static void Add()
        {
            //This is a static block
            //we can access non static members X with the help of Example object
            //We can access the static member directly or through class name
            Example obj = new Example();
            //Console.WriteLine(obj.x + Example.y);
            Console.WriteLine("Sum of 100 and 200 is :" + (obj.x + y));
            Console.WriteLine("Sum of 100 and 200 is :" + (obj.x + Example.y));
        }

        void Mul()
        {
            //This is a Non-Static method
            //we can access static members directly or through class name
            //we can access the non-static members directly or through this keyword
            Console.WriteLine("Multiplication of 100 and 200 is :" + (this.x * Example.y));
            Console.WriteLine("Multiplication of 100 and 200 is :" + (x * y));
        }
        static void Main(string[] args)
        {
            // Main method is a static method
            // ADD() method is a static method
            // Statid to Static 
            // we can call the add method directly or through class name
            Example.Add(); //Calling Add Method using Class Name
            Add(); //Calling Add Method Directly

            // Mul() method is a Non-Static method
            // We can call the non-static method using object only from a static method
            // Static to Non-Static
            Example obj = new Example();
            obj.Mul(); //Calling Mul Method using Instance
            Console.ReadLine();
        }
    }
}
```

##### **آشنایی با سازنده‌های استاتیک و غیر استاتیک در سی شارپ:**

اگر سازنده را به طور صریح توسط اصلاح‌کننده استاتیک ایجاد کنیم، آن را سازنده استاتیک می‌نامیم و بقیه سازنده‌ها غیراستاتیک هستند.

مهمترین نکته‌ای که باید به خاطر داشته باشید این است که سازنده استاتیک اولین بلوک کدی است که تحت یک کلاس اجرا می‌شود. مهم نیست چند شیء برای کلاس ایجاد کرده‌اید، سازنده استاتیک فقط یک بار اجرا می‌شود. از طرف دیگر، یک سازنده غیر استاتیک فقط زمانی اجرا می‌شود که شیء کلاس را ایجاد کنیم و این برای هر شیء از کلاس نیز همینطور است.

ایجاد یک سازنده استاتیک با پارامترها امکان‌پذیر نیست. دلیل این امر آن است که سازنده استاتیک اولین بلوک کدی است که قرار است تحت یک کلاس اجرا شود. و این سازنده استاتیک به صورت ضمنی فراخوانی می‌شود، حتی اگر پارامتردهی شود، هیچ شانسی برای ارسال مقادیر پارامتر وجود ندارد.

##### **مثالی برای درک سازنده‌های استاتیک و غیر استاتیک در سی شارپ**

بیایید برای درک بهتر سازنده‌های ایستا و غیرایستا در سی‌شارپ، مثالی بزنیم. لطفاً به مثال زیر نگاهی بیندازید. در مثال زیر، ما دو سازنده ایجاد کرده‌ایم. از بین دو سازنده، یکی از آنها ایستا است و آن سازنده ایستا ابتدا اجرا می‌شود و آن سازنده فقط یک بار در طول عمر خود اجرا می‌شود. پس از اجرای سازنده ایستا، متد main اجرای خود را آغاز می‌کند. سپس ما دو نمونه از کلاس Example ایجاد کرده‌ایم و این بدان معناست که سازنده غیرایستا دو بار اجرا خواهد شد. کد مثال زیر خود توضیح است، بنابراین لطفاً خطوط کامنت را بررسی کنید.

```csharp
using System;

namespace StaticNonStaticDemo
{
    class Example
    {
        //Static Constructor
        //Executed only once
        //First block of code to be executed inside a class
        //Before Main Method body start executing, this constructor will execute
        static Example()
        {
            Console.WriteLine("Static Constructor is Called");
        }

        //Non-Static Constructor
        //Executed once per object
        //When we create an instance, this constructor will execute
        public Example()
        {
            Console.WriteLine("Non-Static Constructor is Called");
        }

        //Program Execution will start from the Main method
        //But before executing the Main method body, it will
        //execute the static constructor
        static void Main(string[] args)
        {
            Console.WriteLine("Main Method Execution Start");
            Example obj1 = new Example();
            Example obj2 = new Example();
            Console.WriteLine("Main Method Execution End");
            Console.ReadLine();
        }
    }
}
```

##### **کلاس استاتیک در سی شارپ:**

کلاسی که با استفاده از اصلاحگر static ایجاد می‌شود، در سی شارپ کلاس static نامیده می‌شود. یک کلاس static می‌تواند فقط شامل اعضای static باشد. ایجاد نمونه‌ای از یک کلاس static امکان‌پذیر نیست. دلیل این امر این است که فقط شامل اعضای static است. و می‌دانیم که می‌توانیم با استفاده از نام کلاس به اعضای static آن دسترسی پیدا کنیم.

##### **مثال برای درک کلاس استاتیک در سی شارپ**

بیایید برای درک بهتر کلاس استاتیک در سی شارپ، مثالی بزنیم. لطفاً به مثال زیر نگاهی بیندازید. همانطور که در کد زیر مشاهده می‌کنید، ما دو کلاس داریم. کلاس اول TemperatureConverter یک کلاس استاتیک است و این کلاس شامل دو متد استاتیک است. از آنجایی که این یک کلاس استاتیک است، فقط می‌تواند شامل اعضای استاتیک باشد. TestTemperatureConverter یک کلاس معمولی است و از آن کلاس، متدهای استاتیک را با استفاده از نام کلاس استاتیک فراخوانی می‌کنیم.

```csharp
namespace StaticNonStaticDemo
{
    public static class TemperatureConverter
    {
        public static double CelsiusToFahrenheit(string temperatureCelsius)
        {
            // Convert argument to double for calculations.
            double celsius = Double.Parse(temperatureCelsius);

            // Convert Celsius to Fahrenheit.
            double fahrenheit = (celsius * 9 / 5) + 32;

            return fahrenheit;
        }

        public static double FahrenheitToCelsius(string temperatureFahrenheit)
        {
            // Convert argument to double for calculations.
            double fahrenheit = Double.Parse(temperatureFahrenheit);

            // Convert Fahrenheit to Celsius.
            double celsius = (fahrenheit - 32) * 5 / 9;

            return celsius;
        }
    }
    class TestTemperatureConverter
    {
        static void Main()
        {
            Console.WriteLine("Please select the convertor direction");
            Console.WriteLine("1. From Celsius to Fahrenheit.");
            Console.WriteLine("2. From Fahrenheit to Celsius.");
            Console.Write(":");

            string selection = Console.ReadLine();
            double F, C = 0;

            switch (selection)
            {
                case "1":
                    Console.Write("Please enter the Celsius temperature: ");
                    F = TemperatureConverter.CelsiusToFahrenheit(Console.ReadLine());
                    Console.WriteLine("Temperature in Fahrenheit: {0:F2}", F);
                    break;

                case "2":
                    Console.Write("Please enter the Fahrenheit temperature: ");
                    C = TemperatureConverter.FahrenheitToCelsius(Console.ReadLine());
                    Console.WriteLine("Temperature in Celsius: {0:F2}", C);
                    break;

                default:
                    Console.WriteLine("Please select a convertor.");
                    break;
            }

            // Keep the console window open in debug mode.
            Console.WriteLine("Press any key to exit.");
            Console.ReadKey();
        }
    }
}
```
