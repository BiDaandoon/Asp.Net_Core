## **فراخوانی با مقدار و فراخوانی با ارجاع در سی شارپ به همراه مثال**

در این مقاله، قصد دارم در مورد **فراخوانی با مقدار (Call By Value) و فراخوانی با ارجاع (Call By Reference)** به همراه مثال صحبت کنم.

##### **فراخوانی با مقدار و فراخوانی با ارجاع در سی شارپ:**

این یک سوال رایج است که توسط مصاحبه‌کنندگان در مصاحبه‌ها پرسیده می‌شود: «فراخوانی/ارسال با مقدار چیست و فراخوان/ارسال با ارجاع چیست». اکثر توسعه‌دهندگان مبتدی یا متوسط ​​در مورد این سوال تصورات غلطی دارند و در طول مصاحبه‌ها به آن پاسخ نادرست می‌دهند. آنها معمولاً می‌گویند وقتی انواع اولیه را ارسال می‌کنیم، آنها با مقدار ارسال می‌شوند و وقتی انواع مرجع را ارسال می‌کنیم، آنها با ارجاع ارسال می‌شوند. این پاسخ اشتباه است. بنابراین، امروز تفاوت واقعی بین آنها را برای شما توضیح خواهم داد.

##### **فراخوانی با مقدار در سی شارپ**

در چارچوب .NET، به طور پیش‌فرض، تمام اشیاء توسط مقدار فراخوانی می‌شوند، نه توسط ارجاع. بنابراین، چه از نوع داده‌ی Value (انواع داده‌ی اولیه مانند int، char، double و غیره) باشد و چه از نوع داده‌ی Reference (کلاس، رابط، نماینده، رشته و غیره)، به طور پیش‌فرض توسط مقدار فراخوانی می‌شوند. بیایید با چند مثال، فراخوانی توسط مقدار در C# را درک کنیم. مثال زیر را در نظر بگیرید. در مثال زیر، من از نوع داده‌ی value استفاده می‌کنم.

```csharp
using System;

namespace FunctionsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int a = 15;
            int b = a;
            b = 30;
            Console.WriteLine(a);
            Console.ReadKey();
        }
    }
}
```

###### **بنابراین، چه چیزی روی کنسول چاپ خواهد شد؟**

اگر پاسخ شما ۱۵ است، کاملاً درست حدس زده‌اید، زیرا **int** یک نوع داده مقداری است و به طور پیش‌فرض، توسط مقدار ارسال می‌شود، به این معنی که برای کد بالا، متغیر «a» مقدار ۱۵ را در خود ذخیره کرده است. وقتی متغیر b را ایجاد می‌کنیم و a را به آن اختصاص می‌دهیم، مقدار a در b کپی می‌شود و پس از آن، اگر b را تغییر دهیم، روی a تأثیری نخواهد گذاشت. دلیل این امر این است که ما مقدار a را در b کپی کرده‌ایم.

##### **نمایش حافظه فراخوانی با مقدار در انواع مقداری در سی شارپ:**

برای درک بهتر، لطفاً به تصویر زیر نگاهی بیندازید. فرض کنید که مرجع حافظه متغیر a برابر با 1010 است و مقدار 15 را در خود نگه می‌دارد. و هنگامی که دستور **int b = a** را اجرا می‌کنیم، در این حالت، مقدار i,e, 15 به متغیر b ارسال می‌شود. این عمل توسط مکانیزم مقدار در C# با نوع داده value یا primitive فراخوانی می‌شود.

![نمایش حافظه فراخوانی با مقدار در انواع مقداری در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/07/word-image-27425-1-4.png "نمایش حافظه فراخوانی با مقدار در انواع مقداری در سی شارپ")

همین اتفاق وقتی که انواع مقادیر را به متدها ارسال می‌کنیم نیز می‌افتد. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید. در اینجا، ما متدی به نام UpdateValue ایجاد کرده‌ایم که مقدار یک متغیر int از نوع b را به 30 به‌روزرسانی می‌کند.

```csharp
using System;

namespace FunctionsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int a = 15;
            UpdateValue(a);
            Console.WriteLine(a);
            Console.ReadKey();
        }

        private static void UpdateValue(int b)
        {
            b = 30;
        }
    }
}
```

##### **خب، حالا چه چیزی روی کنسول چاپ خواهد شد؟**

بله، عدد ۱۵ را در کنسول چاپ می‌کند. همانطور که قبلاً گفتم، آن مقدار کپی می‌شود، بنابراین وقتی متد UpdateValue فراخوانی می‌شود، مقدار متغیر a در متغیر b کپی می‌شود، بنابراین تغییر متغیر b، متغیر a را تغییر نمی‌دهد.

##### **فراخوانی با مقدار با انواع ارجاعی در سی شارپ**

من کلاس Employee زیر را برای درک فراخوانی با مقدار با انواع مرجع در C# ایجاد کرده‌ام. ما از قبل می‌دانیم که این کلاس یک نوع داده مرجع است.

```csharp
public class Employee
{
    public int EmployeeID;
    public string Name;
}
```

حالا، لطفاً به کد زیر توجه کنید. در اینجا، ما یک نمونه از کلاس Employee، یعنی Emp1، ایجاد می‌کنیم و دو عضو داده‌ای آن را تنظیم می‌کنیم. سپس، آن (نمونه Emp1) را به نمونه دیگری از کلاس Employee، یعنی Emp2، اختصاص می‌دهیم و ویژگی Name را تغییر می‌دهیم و سپس EmployeeID و Name را در کنسول چاپ می‌کنیم تا بررسی کنیم چه چیزی چاپ شده است:

```csharp
using System;

namespace FunctionsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Employee Emp1 = new Employee();
            Emp1.EmployeeID = 1001;
            Emp1.Name = "James";

            Employee Emp2 = Emp1;
            Emp1.Name = "Smith";
            Console.WriteLine($"Emp1 Name = {Emp1.Name}");
            Console.WriteLine($"Emp2 Name = {Emp2.Name}");
            Console.ReadKey();
        }
    }

    public class Employee
    {
        public int EmployeeID;
        public string Name;
    }
}
```

وقتی یک نمونه از کلاس Employee ایجاد می‌کنیم، یک شیء در حافظه heap ایجاد می‌شود و حافظه به آن اختصاص داده می‌شود. ما مرجع آن مکان حافظه را در متغیر مرجع Emp1، عمدتاً در حافظه stack، ذخیره می‌کنیم.

وقتی Emp1 را به Emp2 اختصاص می‌دهیم، مرجع مکان حافظه شیء که Emp1 در اختیار دارد، در Emp2 کپی می‌شود. اکنون ما دو کپی از مرجع داریم، اما هر دو به یک مکان حافظه اشاره می‌کنند. بنابراین، تغییر مقدار ویژگی Name، مقدار شیء در حافظه را تغییر می‌دهد، که ما در Emp1 و Emp2 به آن ارجاع داریم. بنابراین، "Smith" برای هر دو ارجاع در کنسول چاپ می‌شود. بنابراین، وقتی کد بالا را اجرا می‌کنید، خروجی زیر را دریافت خواهید کرد.

##### **فراخوانی با مقدار در نمایش حافظه با انواع ارجاعی در سی شارپ**

وقتی اولین نمونه از شیء Employee، یعنی Emp1، را ایجاد می‌کنیم، باید بفهمیم که درون حافظه چه اتفاقی می‌افتد. در حافظه heap، یک شیء ایجاد می‌شود که حافظه را برای اعضای داده EmployeeId و Name اختصاص می‌دهد. سپس، فرض می‌کنیم که آدرس حافظه شیء 101011 است و سپس آن مرجع حافظه درون متغیر مرجع Emp1 ذخیره می‌شود. دوباره، فرض کنید که آدرس حافظه متغیر مرجع Emp1، 111000 است. بنابراین، نکته‌ای که باید به خاطر داشته باشید این است که Emp1 یک متغیر مرجع است که آدرس آن 111000 است و آدرس مرجع 101011 را در خود نگه می‌دارد.

![فراخوانی با مقدار در نمایش حافظه با انواع ارجاعی در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/07/word-image-27425-3-5.png "فراخوانی با مقدار در نمایش حافظه با انواع ارجاعی در سی شارپ")

حال، بیایید بفهمیم چه زمانی متغیر مرجع Emp1 را به متغیر مرجع Emp2، یعنی **Employee Emp2 = Emp1**، اختصاص می‌دهیم و چه اتفاقی می‌افتد وقتی نام را در متغیر مرجع Emp1، یعنی **Emp1.Name = “Smith”**، به‌روزرسانی می‌کنیم. برای درک بهتر، لطفاً به تصویر زیر نگاهی بیندازید. وقتی **دستور Employee Emp2 = Emp1;** اجرا می‌شود، Emp1 چه مقداری را در خود نگه می‌دارد که در متغیر مرجع Emp2 کپی می‌شود، Emp1 مقدار 101011 را در خود نگه می‌دارد، بنابراین آن نیز در Emp2 کپی خواهد شد. این توسط مقداری با نوع مرجع فراخوانی می‌شود. زیرا در اینجا، ما آدرس حافظه Emp1، که 111000 است را کپی نمی‌کنیم؛ در عوض، مقداری را که توسط Emp1 نگهداری می‌شود، یعنی 101011، کپی می‌کنیم که چیزی جز آدرس حافظه شیء Employee نیست که در داخل حافظه Heap ایجاد شده است. این بدان معناست که هر دو متغیر مرجع اکنون به یک مکان حافظه اشاره می‌کنند. اکنون، هنگامی که **عبارت Emp1.Name = “Smith”;** اجرا می‌شود، نام شیء Employee در حافظه heap به Smith به‌روزرسانی می‌شود و از این رو تغییرات در متغیرهای مرجع منعکس می‌شوند.

![فراخوانی با مقدار و فراخوانی با ارجاع در سی شارپ به همراه مثال](https://dotnettutorials.net/wp-content/uploads/2022/07/word-image-27425-4-7.png "فراخوانی با مقدار و فراخوانی با ارجاع در سی شارپ به همراه مثال")

ما همچنین می‌توانیم همین رفتار را با استفاده از یک متد به دست آوریم. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید.

```csharp
using System;

namespace FunctionsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Employee Emp1 = new Employee();
            Emp1.EmployeeID = 1001;
            Emp1.Name = "James";

            UpdateName(Emp1);
            Console.WriteLine($"Emp1 Name = {Emp1.Name}");
            Console.ReadKey();
        }

        public static void UpdateName(Employee Emp2)
        {
            Emp2.Name = "Smith";
            Console.WriteLine($"Emp2 Name = {Emp2.Name}");
        }
    }

    public class Employee
    {
        public int EmployeeID;
        public string Name;
    }
}
```

وقتی مرجع Emp1 از شیء Employee را به متد UpdateName ارسال می‌کنیم، مرجع مکان حافظه به شیء محلی Emp2 از متد UpdateName کپی می‌شود. و هر دو (Emp1 و Emp2) به یک مکان حافظه اشاره می‌کنند. بنابراین، تغییر مقدار متغیر Name، مقدار شیء را در حافظه تغییر می‌دهد و این تغییرات در هر دو مرجع، یعنی Emp1 و Emp2، و مقدار به‌روزرسانی‌شده‌ای که Smith در کنسول چاپ خواهد کرد، منعکس می‌شود.

**اگر به صورت زیر، مقدار Emp2 را درون متد UpdateName برابر با null قرار دهیم، چه اتفاقی می‌افتد؟**

```csharp
using System;

namespace FunctionsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Employee Emp1 = new Employee();
            Emp1.EmployeeID = 1001;
            Emp1.Name = "James";

            UpdateName(Emp1);
            Console.WriteLine($"Emp1 Name = {Emp1.Name}");
            Console.ReadKey();
        }

        public static void UpdateName(Employee Emp2)
        {
            Emp2 = null;
        }
    }

    public class Employee
    {
        public int EmployeeID;
        public string Name;
    }
}
```

حال، اگر فکر می‌کنید که در متد Main در دستور WriteLine که نام را چاپ می‌کند، با خطای Null Reference Exception مواجه خواهید شد، در اشتباه هستید. اگر فکر می‌کنید که خروجی James خواهد بود، حق با شماست و متوجه شده‌اید که به طور پیش‌فرض، ارجاع در C# با مقدار ارسال می‌شود، نه با ارجاع.

##### **نمایش حافظه‌ای مثال بالا:**

![فراخوانی با مقدار و فراخوانی با ارجاع در سی شارپ به همراه مثال](https://dotnettutorials.net/wp-content/uploads/2022/07/word-image-27425-5-5.png "فراخوانی با مقدار و فراخوانی با ارجاع در سی شارپ به همراه مثال")

همانطور که در تصویر بالا مشاهده می‌کنید، وقتی **Emp2 را برابر با null قرار می‌دهیم،** ارجاعات Emp2 حذف می‌شوند. اما همچنان ارجاع Emp1 به شیء Employee وجود دارد. این مکانیسم فراخوانی با مقدار در C# است. حال، امیدوارم مکانیسم فراخوانی با مقدار در C# را با استفاده از انواع مقدار و ارجاع با مثال‌ها درک کرده باشید. اگر بخواهیم Emp1 را نیز null کنیم، وقتی Emp2 را null قرار می‌دهید، باید آن را از طریق ارجاع به متد ارسال کنید، که این کار در C# با استفاده از کلمه کلیدی ref انجام می‌شود، یعنی چیزی جز ارسال با ارجاع در C# نیست. بیایید Call by Reference را با جزئیات بیشتر درک کنیم.

##### **فراخوانی با ارجاع در سی شارپ**

بیایید با مثال‌هایی، فراخوانی با ارجاع یا ارسال با ارجاع را با هر دو نوع مقداری و همچنین با نوع مرجع در سی‌شارپ درک کنیم. برای فراخوانی با ارجاع، باید پارامتر را با کلمه کلیدی ref پیشوند دهیم. من قصد دارم فراخوانی با ارجاع را با استفاده از همان مثال‌هایی که در فراخوانی با مقدار بحث کردیم، توضیح دهم تا درک مفهوم و تفاوت‌های بین آنها برای شما آسان‌تر شود.

##### **فراخوانی با ارجاع با انواع مقداری در سی شارپ**

ما از همان مثال استفاده خواهیم کرد، اما این بار، فراخوانی با ارجاع انجام خواهیم داد. بنابراین، کاری که باید انجام دهیم این است که باید متغیر را با کلمه کلیدی ref پیشوند کنیم. لطفاً به مثال زیر نگاهی بیندازید. در اینجا، می‌توانید ببینید که در داخل متد UpdateValue، به جای **int b** استفاده می‌کنیم **، اکنون از ref int b** . در اینجا، کلمه کلیدی ref را به پارامتر ورودی اضافه کرده‌ایم، به این معنی که آرگومان هنگام فراخوانی این متد باید با ارجاع ارسال شود.

```csharp
using System;

namespace FunctionsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            int a = 15;
            UpdateValue(ref a);
            Console.WriteLine(a);
            Console.ReadKey();
        }

        static void UpdateValue(ref int b)
        {
            b = 30;
        }
    }
}
```

سپس متد UpdateValue را فراخوانی می‌کنیم، اما باید قبل از نام آرگومان از کلمه کلیدی ref استفاده کنیم. این همان فراخوانی با ارجاع (Call by Reference) در سی شارپ با نوع مقدار (Value Type) است. اگر از کلمه کلیدی ref استفاده نکنید، با خطای زمان کامپایل مواجه خواهید شد.

وقتی کد بالا را اجرا می‌کنید، عدد 30 در کنسول چاپ می‌شود. این به دلیل کلمه کلیدی ref است. در مثال بالا، وقتی متد UpdateValue فراخوانی می‌شود، پارامتر ورودی b همان آدرس حافظه a را دارد که به عنوان آرگومان ارسال می‌شود. به همین دلیل است که تغییر مقدار b، تغییر در a را نیز منعکس می‌کند. در این حالت، یک مکان حافظه جدید برای پارامتر متد b ایجاد نمی‌شود. در اینجا، می‌توانید b را فقط به عنوان یک نام مستعار a یا نام جایگزین a بگویید. بنابراین، اگر با استفاده از متغیر b تغییری ایجاد کنیم، در متغیر a نیز منعکس خواهد شد.

برای درک بهتر مثال بالا، لطفاً به نمودار زیر که نمایش حافظه مثال بالا را نشان می‌دهد، توجه کنید.

##### **فراخوانی با ارجاع با انواع ارجاع در سی شارپ**

حال، همین مورد را با نوع مرجع خواهیم دید. رفتار برای انواع مرجع نیز یکسان خواهد بود. ما همچنین با همان مثالی که با فراخوانی با مقدار با نوع مرجع کار کردیم، کار خواهیم کرد. بنابراین، لطفاً به مثال زیر نگاهی بیندازید. اکنون، در متد UpdateName، کلمه کلیدی ref را اضافه کرده‌ایم. در متد main، هنگام فراخوانی متد UpdateName، متغیر Emp1 را نیز با کلمه کلیدی ref به صورت زیر پیشوند کرده‌ایم:

```csharp
using System;

namespace FunctionsDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Employee Emp1 = new Employee();
            Emp1.EmployeeID = 1001;
            Emp1.Name = "James";

            UpdateName(ref Emp1);
            Console.WriteLine($"Emp1 Name = {Emp1.Name}");
            Console.ReadKey();
        }

        public static void UpdateName(ref Employee Emp2)
        {
            Emp2 = null;
        }
    }

    public class Employee
    {
        public int EmployeeID;
        public string Name;
    }
}
```

حال، وقتی متد UpdateName مقدار Emp2 را null قرار می‌دهد، مقدار Emp1 را نیز null می‌کند. این به دلیل فراخوانی با ارجاع (Call By Reference) است. بنابراین، در این حالت، ارجاع Emp1 به متد ارسال می‌شود و از این رو ارجاع جدیدی برای Emp2 ایجاد نمی‌شود. بنابراین، هنگامی که کد بالا را اجرا می‌کنید، همانطور که در تصویر زیر نشان داده شده است، با خطای زمان اجرا مواجه خواهید شد. بنابراین، در اینجا، می‌توانید Emp2 را به عنوان نام مستعار Emp1 یا نام دیگری برای Emp1 در نظر بگیرید.

![فراخوانی با ارجاع در سی شارپ به همراه مثال](https://dotnettutorials.net/wp-content/uploads/2022/07/word-image-27425-7-6.png "فراخوانی با ارجاع در سی شارپ به همراه مثال")

برای درک اینکه چرا با خطای Null Reference Exception مواجه می‌شویم، باید نحوه نمایش حافظه در مثال بالا را درک کنیم. فرض کنید آدرس حافظه Emp1 برابر با 111000 است و به شیء Employee که آدرس آن 101011 است اشاره می‌کند. این بدان معناست که آدرس Emp1 برابر با 111000 است و آدرس مرجع شیء Employee برابر با 101011 است. در مورد فراخوانی با ارجاع، وقتی ref Emp1 را ارسال می‌کنیم، در این صورت، آدرس حافظه Emp2، یعنی 111000 را ارسال می‌کنیم. و در رایانه، نمی‌توانیم دو بار آدرس حافظه یکسانی داشته باشیم. بنابراین، در اینجا Emp2 نیز به همان آدرس حافظه Emp1 اشاره می‌کند. می‌توانیم بگوییم که Emp2 نام مستعار Emp1 یا نام جایگزین Emp1 است. بنابراین، وقتی Emp2 را null می‌کنیم، به این معنی است که آدرس حافظه 111000، که به شیء Employee اشاره می‌کند، یعنی 101011، null می‌شود و سپس سعی می‌کنیم به ویژگی Name دسترسی پیدا کنیم، با خطای زمان اجرا مواجه می‌شویم.

**نکته:** اگر می‌خواهید در سی‌شارپ از قابلیت فراخوانی با ارجاع (Call by Reference) استفاده کنید، باید از کلمات کلیدی out یا ref استفاده کنید، صرف نظر از اینکه نوع آرگومان از نوع مقداری باشد یا از نوع ارجاعی. در مقاله بعدی، به تفصیل در مورد نیاز و کاربرد کلمات کلیدی ref و out صحبت خواهیم کرد.

##### **تفاوت بین فراخوانی با مقدار و فراخوانی با ارجاع در سی شارپ:**

مهمترین نکته‌ای که باید به خاطر داشته باشید این است که در سی شارپ، تمام اشیاء/مقادیر به طور پیش‌فرض توسط مقدار فراخوانی می‌شوند، نه توسط ارجاع، صرف نظر از اینکه از نوع مقداری باشند یا از نوع ارجاعی. با در نظر گرفتن این نکته، بیایید ادامه دهیم و تفاوت‌ها را بررسی کنیم.

![تفاوت بین فراخوانی با مقدار و فراخوانی با ارجاع در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/07/word-image-27425-9-4.png "تفاوت بین فراخوانی با مقدار و فراخوانی با ارجاع در سی شارپ")

##### **فراخوانی با مقدار در سی شارپ:**

1. در فراخوانی با مقدار، یک کپی از متغیر اصلی به تابع فراخوانی شده ارسال می‌شود.
2. در فراخوانی با مقدار، مقدار پارامتر اصلی در پارامتر تابع کپی می‌شود. در نتیجه، اگر هرگونه تغییری در پارامترهای رسمی ایجاد کنیم، هیچ تاثیری بر پارامتر واقعی نخواهند داشت.
3. این به شما اجازه نمی‌دهد متغیرهای واقعی را با استفاده از فراخوانی‌های تابع تغییر دهید.
4. مکان حافظه‌ای که پارامترهای رسمی و آرگومان‌های واقعی به آن اشاره می‌کنند، متفاوت است.
5. در سی شارپ نیازی به کلمه کلیدی ref یا out نیست.

##### **فراخوانی با ارجاع در سی شارپ:**

1. در فراخوانی با ارجاع، مرجع متغیر (یا آدرس واقعی) به تابع فراخوانی شده ارسال می‌شود.
2. در فراخوانی با ارجاع، پارامترهای رسمی و واقعی به یک آدرس حافظه اشاره می‌کنند. در نتیجه، هرگونه تغییری که در پارامترهای رسمی ایجاد شود، در پارامترهای واقعی نیز منعکس می‌شود.
3. این به شما امکان می‌دهد مقدار واقعی متغیر را با استفاده از فراخوانی‌های تابع تغییر دهید.
4. مکان حافظه‌ای که پارامترهای رسمی و آرگومان‌های واقعی به آن اشاره می‌کنند، یکسان است.
5. برای دستیابی به قابلیت ارجاع از طریق فراخوانی (Call By Reference) در سی شارپ، به یک کلمه کلیدی ref یا out نیاز است.
