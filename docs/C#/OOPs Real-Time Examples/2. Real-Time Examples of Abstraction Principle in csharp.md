##### **انتزاع (Abstraction) در سی شارپ چیست؟**

انتزاع یک مفهوم بنیادی در برنامه‌نویسی شی‌گرا (OOP) است و فقط به C# محدود نمی‌شود. در C#، مانند سایر زبان‌های OOP، انتزاع به پنهان کردن جزئیات پیاده‌سازی پیچیده و نمایش تنها ویژگی‌های اساسی یک شیء اشاره دارد. در اینجا توضیح مفصل‌تری ارائه شده است:

- **پنهان کردن پیچیدگی:** درست مانند داشبورد ماشین که تمام جزئیات پیچیده نحوه کار موتور را نشان نمی‌دهد اما چند کنترل ضروری (مانند فرمان، گاز، ترمز و غیره) را در اختیار شما قرار می‌دهد، انتزاع در برنامه‌نویسی به معنای پنهان کردن جزئیات پیچیده نحوه کار چیزی و ارائه تنها عملیات ضروری برای کار با آن است.
- **تسهیل توسعه‌پذیری:** از آنجا که انتزاع به شما امکان می‌دهد جزئیات پیاده‌سازی را پشت یک رابط سازگار پنهان کنید، می‌توانید بعداً پیاده‌سازی زیربنایی را بدون تأثیر بر کدی که از این انتزاع استفاده می‌کند، تغییر دهید.
- **استفاده از کلاس‌ها و رابط‌ها:** در سی‌شارپ، انتزاع می‌تواند با استفاده از کلاس‌ها و رابط‌ها حاصل شود. در حالی که نمی‌توانید نمونه‌ای از یک کلاس انتزاعی ایجاد کنید، می‌توانید از شکل آن (خصوصیات و متدها) در کلاس‌های فرزند استفاده کنید. رابط‌ها با تعریف شکل بدون هیچ پیاده‌سازی، یک قدم فراتر می‌روند و کلاس‌های عینی را برای ارائه جزئیات اعمال می‌کنند.

##### **مثال بلادرنگ از اصل انتزاع در سی شارپ: سیستم خودرو**

بیایید یک مثال از سیستم خودرو در زمان واقعی را در نظر بگیریم. فرض کنید می‌خواهید یک سیستم خودرو پایه را مدل‌سازی کنید. همه خودروها می‌توانند روشن و خاموش شوند، اما جزئیات اساسی نحوه انجام این کار می‌تواند برای هر نوع خودرو متفاوت باشد. بیایید ببینیم چگونه می‌توانیم این مثال را با استفاده از اصل انتزاع در C# پیاده‌سازی کنیم:

```csharp
using System;

namespace AbstractionPrincipleCSharp
{
    //Abstract Base Class (Abstraction)
    public abstract class Vehicle
    {
        // These are abstract methods; the derived classes will provide the implementation.
        public abstract void Start();
        public abstract void Stop();
    }

    //Concrete Implementations
    public class Car : Vehicle
    {
        public override void Start()
        {
            Console.WriteLine("Car is starting with a key turn.");
        }

        public override void Stop()
        {
            Console.WriteLine("Car is stopping using its brakes.");
        }
    }

    public class ElectricTrain : Vehicle
    {
        public override void Start()
        {
            Console.WriteLine("Electric train is starting by powering up.");
        }

        public override void Stop()
        {
            Console.WriteLine("Electric train is stopping by cutting off the power.");
        }
    }
    
    //Testing Abstraction Principle
    public class Program
    {
        static void Main(string[] args)
        {
            //Using the Abstraction
            Vehicle myCar = new Car();
            Vehicle myTrain = new ElectricTrain();

            StartVehicle(myCar);  // Output: Car is starting with a key turn.
            StartVehicle(myTrain); // Output: Electric train is starting by powering up.

            Console.Read();
        }

        static void StartVehicle(Vehicle vehicle)
        {
            vehicle.Start();
        }
    }
}
```

**در مثال:**

- کلاس Vehicle یک انتزاع برای هر وسیله نقلیه ارائه می‌دهد. این کلاس جزئیات نحوه شروع یا توقف یک وسیله نقلیه را شرح نمی‌دهد. فقط اعلام می‌کند که هر وسیله نقلیه باید بتواند شروع و متوقف شود.
- کلاس‌های Car و ElectricTrain پیاده‌سازی‌های عینی از انتزاع Vehicle هستند. آن‌ها روش‌های خاصی برای شروع و توقف ارائه می‌دهند.
- در متد Main، ما از انتزاع Vehicle برای شروع انواع مختلف وسایل نقلیه استفاده می‌کنیم. متد واقعی که فراخوانی می‌شود بستگی به نوع شیء ارسالی دارد که نشان‌دهنده‌ی چندریختی، یکی دیگر از اصول OOP، است.

در برنامه‌های دنیای واقعی، انتزاع به شما امکان می‌دهد بدون تعهد به جزئیات خاص، یک قرارداد (در این مورد، «وسایل نقلیه می‌توانند شروع و متوقف شوند») تعریف کنید. این امر افزودن انواع جدید وسایل نقلیه را در آینده بدون تغییر کد موجود آسان‌تر می‌کند.

##### **مثال بلادرنگ از اصل انتزاع در سی شارپ: سیستم پیام رسانی**

سیستمی را تصور کنید که باید با چندین پلتفرم پیام‌رسانی ارتباط برقرار کند: ایمیل، پیامک و اعلان‌های فوری. بدون انتزاع، ممکن است در نهایت با متدهایی مانند SendEmail()، SendSms()، SendPushNotification() و غیره که در سراسر برنامه شما پراکنده شده‌اند، مواجه شوید. اگر یک متد پیام‌رسانی جدید اضافه شود، باید متد دیگری اضافه کنید و آن را به طور متفاوتی فراخوانی کنید. بیایید ببینیم چگونه می‌توانیم این مثال را با استفاده از اصل انتزاع در C# پیاده‌سازی کنیم:

```csharp
using System;

namespace AbstractionPrincipleCSharp
{
    //Abstraction Layer
    //Define a common interface for all messaging platforms
    public interface IMessagingService
    {
        void SendMessage(string recipient, string message);
    }

    //Concrete Implementations
    //Now, we'll create concrete classes that implement this interface.
    public class EmailService : IMessagingService
    {
        public void SendMessage(string recipient, string message)
        {
            Console.WriteLine($"Sending Email to {recipient}: {message}");
            // Here, you'd have the actual logic to send an email.
        }
    }

    public class SmsService : IMessagingService
    {
        public void SendMessage(string recipient, string message)
        {
            Console.WriteLine($"Sending SMS to {recipient}: {message}");
            // Here, you'd have the actual logic to send an SMS.
        }
    }

    public class PushNotificationService : IMessagingService
    {
        public void SendMessage(string recipient, string message)
        {
            Console.WriteLine($"Sending Push Notification to {recipient}: {message}");
            // Here, you'd have the actual logic to send a push notification.
        }
    }
    
    //Testing Abstraction Principle
    public class Program
    {
        static void Main(string[] args)
        {
            // Using the Abstraction
            // With the Abstraction Principle applied, if you need to send a message, 
            // you don’t need to know if it’s an email, SMS, or push notification.
            // You just call SendMessage() on any service implementing IMessagingService.
            IMessagingService emailService = new EmailService();
            IMessagingService smsService = new SmsService();
            IMessagingService pushService = new PushNotificationService();

            SendAlert(emailService, "example@example.com", "Hello via Email!");
            SendAlert(smsService, "1234567890", "Hello via SMS!");
            SendAlert(pushService, "User123", "Hello via Push Notification!");

            Console.Read();
        }

        static void SendAlert(IMessagingService service, string recipient, string message)
        {
            service.SendMessage(recipient, message);
        }
    }
}
```

با پیروی از اصل انتزاع، ما موارد زیر را تضمین کرده‌ایم:

- روشی سازگار برای ارسال پیام در پلتفرم‌های مختلف.
- سهولت توسعه. اگر یک پلتفرم پیام‌رسانی جدید اضافه کنیم، IMessagingService را بدون تغییر نحوه ارسال پیام‌ها در برنامه اصلی خود پیاده‌سازی می‌کنیم.
- جداسازی. منطق سطح بالا (ارسال هشدار) از جزئیات پیام‌رسانی سطح پایین (ایمیل، پیامک، پوش) جدا می‌شود.

##### **مثال بلادرنگ از اصل انتزاع در سی شارپ: برنامه چندرسانه‌ای**

سناریویی را در نظر بگیرید که در آن در حال توسعه یک برنامه چندرسانه‌ای هستید که می‌تواند فایل‌های رسانه‌ای مختلف را پخش کند: صدا و تصویر. بدون انتزاع، ممکن است متدهای جداگانه‌ای برای پخش هر نوع رسانه داشته باشید، مانند PlayAudio()، PlayVideo() و غیره. اگر قرار باشد پشتیبانی از فرمت‌ها یا انواع رسانه بیشتری را اضافه کنید، نیاز به تغییرات قابل توجهی در برنامه شما خواهد بود. بیایید ببینیم چگونه می‌توانیم این مثال را با استفاده از اصل انتزاع در C# پیاده‌سازی کنیم:

```csharp
using System;

namespace AbstractionPrincipleCSharp
{
    //Abstraction Layer
    //First, define an interface representing any media player.
    public interface IMediaPlayer
    {
        void Play(string filePath);
    }

    //Concrete Implementations
    //Implement the interface for different media types.
    public class AudioPlayer : IMediaPlayer
    {
        public void Play(string filePath)
        {
            Console.WriteLine($"Playing audio from {filePath}");
            // Logic to play audio files
        }
    }

    public class VideoPlayer : IMediaPlayer
    {
        public void Play(string filePath)
        {
            Console.WriteLine($"Playing video from {filePath}");
            // Logic to play video files
        }
    }
    
    //Testing Abstraction Principle
    public class Program
    {
        static void Main(string[] args)
        {
            // Using the Abstraction
            // Now, you can use the abstraction to play any media without worrying about its type.
            IMediaPlayer audioPlayer = new AudioPlayer();
            IMediaPlayer videoPlayer = new VideoPlayer();

            PlayMedia(audioPlayer, "song.mp3");
            PlayMedia(videoPlayer, "movie.mp4");

            Console.Read();
        }

        static void PlayMedia(IMediaPlayer player, string filePath)
        {
            player.Play(filePath);
        }
    }
}
```

با پیروی از اصل انتزاع:

- ما یک روش یکپارچه برای مدیریت انواع مختلف رسانه ایجاد کرده‌ایم.
- ما می‌توانیم به راحتی با اضافه کردن انواع جدید رسانه با پیاده‌سازی رابط IMediaPlayer، برنامه را گسترش دهیم.
- برنامه اصلی از پیاده‌سازی‌های خاص برای هر نوع رسانه جدا باقی می‌ماند.

این طراحی، سیستم را انعطاف‌پذیرتر، ماژولارتر و قابل نگهداری‌تر می‌کند. هر زمان که عملیات یا رفتارهای مشابهی را در کلاس‌ها یا کامپوننت‌ها مشاهده کردید، برای دستیابی به یک طراحی تمیزتر و مقیاس‌پذیرتر، آن‌ها را انتزاعی کنید.

##### **مثال بلادرنگ از اصل انتزاع در سی شارپ: پردازش پرداخت**

بیایید سناریویی را تصور کنیم که در آن شما در حال توسعه سیستمی برای یک فروشگاه آنلاین هستید که از چندین روش پرداخت مانند کارت‌های اعتباری، پی‌پال و بیت‌کوین پشتیبانی می‌کند. بدون انتزاع، شما به طور بالقوه روش‌های متفاوتی مانند ProcessCreditCardPayment()، ProcessPayPalPayment() و غیره خواهید داشت. این امر باعث می‌شود سیستم سفت و سخت شود و در صورت معرفی روش‌های پرداخت جدید، گسترش آن دشوار باشد. بیایید ببینیم چگونه می‌توانیم این مثال را با استفاده از اصل انتزاع در C# پیاده‌سازی کنیم:

```csharp
using System;

namespace AbstractionPrincipleCSharp
{
    //Abstraction Layer
    //Define an interface representing any payment method
    public interface IPaymentMethod
    {
        bool ProcessPayment(decimal amount);
    }

    //Concrete Implementations
    //Implement the interface for each payment method
    public class CreditCardPayment : IPaymentMethod
    {
        public bool ProcessPayment(decimal amount)
        {
            Console.WriteLine($"Processing credit card payment of {amount:C}");
            // Logic to process credit card payment
            return true; // Let's assume the payment is successful for this example.
        }
    }

    public class PayPalPayment : IPaymentMethod
    {
        public bool ProcessPayment(decimal amount)
        {
            Console.WriteLine($"Processing PayPal payment of {amount:C}");
            // Logic to process PayPal payment
            return true;
        }
    }

    public class BitcoinPayment : IPaymentMethod
    {
        public bool ProcessPayment(decimal amount)
        {
            Console.WriteLine($"Processing Bitcoin payment of {amount:C}");
            // Logic to process Bitcoin payment
            return true;
        }
    }

    //Using the Abstraction
    //With the abstraction in place, the checkout process becomes simplified
    public class CheckoutSystem
    {
        public void Checkout(IPaymentMethod paymentMethod, decimal amount)
        {
            if (paymentMethod.ProcessPayment(amount))
            {
                Console.WriteLine("Payment successful!");
            }
            else
            {
                Console.WriteLine("Payment failed.");
            }
        }
    }

    //Testing Abstraction Principle
    public class Program
    {
        static void Main(string[] args)
        {
            CheckoutSystem checkoutSystem = new CheckoutSystem();

            IPaymentMethod creditCard = new CreditCardPayment();
            IPaymentMethod payPal = new PayPalPayment();
            IPaymentMethod bitcoin = new BitcoinPayment();

            checkoutSystem.Checkout(creditCard, 99.99M);
            checkoutSystem.Checkout(payPal, 49.99M);
            checkoutSystem.Checkout(bitcoin, 29.99M);

            Console.Read();
        }
    }
}
```

در این مثال، با انتزاع:

- ما فرآیند مدیریت روش‌های مختلف پرداخت را ساده‌سازی کرده‌ایم.
- معرفی یک روش پرداخت جدید در آینده ساده می‌شود: رابط IPaymentMethod را پیاده‌سازی کنید.
- کلاس CheckoutSystem از پیاده‌سازی‌های پرداخت خاص جدا شده است و این امر باعث می‌شود که نگهداری و گسترش کدبیس آسان‌تر شود.

اینگونه است که انتزاع، انعطاف‌پذیری و مقیاس‌پذیری را در طراحی نرم‌افزار فراهم می‌کند. این روش، جزئیات پیچیده هر روش پرداخت خاص را پشت یک رابط کاربری یکپارچه پنهان می‌کند و تعامل سازگار و ساده‌شده را تضمین می‌کند. وقتی کد بالا را اجرا می‌کنید، خروجی زیر را دریافت خواهید کرد:

##### **مثال بلادرنگ از اصل انتزاع در سی شارپ: واکشی داده‌ها از منابع مختلف**

بیایید سیستمی را در نظر بگیریم که در آن باید داده‌ها را از منابع مختلف دریافت کنید: یک پایگاه داده، یک API وب و یک سیستم فایل. بدون انتزاع، متدهایی مانند FetchFromDatabase()، FetchFromAPI() و FetchFromFile() خواهید داشت. وقتی منابع داده جدیدی اضافه می‌کنید، باید متدهای بیشتری اضافه کنید و آنها را جداگانه مدیریت کنید. بیایید ببینیم چگونه می‌توانیم این مثال را با استفاده از اصل انتزاع در C# پیاده‌سازی کنیم:

```csharp
using System;

namespace AbstractionPrincipleCSharp
{
    //Abstraction Layer
    //Define an interface to abstract data fetching:
    public interface IDataFetcher
    {
        string FetchData();
    }

    //Concrete Implementations
    //Provide specific implementations for each data source
    public class DatabaseFetcher : IDataFetcher
    {
        public string FetchData()
        {
            // Assume logic to connect and fetch from a database
            return "Data from Database";
        }
    }

    public class ApiFetcher : IDataFetcher
    {
        public string FetchData()
        {
            // Assume logic to fetch from a web API
            return "Data from API";
        }
    }

    public class FileFetcher : IDataFetcher
    {
        public string FetchData()
        {
            // Assume logic to read from a file system
            return "Data from File";
        }
    }

    //Using the Abstraction
    //With the abstraction in place, data fetching becomes a unified process
    class DataProcessing
    {
        public void ProcessData(IDataFetcher dataFetcher)
        {
            string data = dataFetcher.FetchData();
            Console.WriteLine(data);
            // Further processing of data if needed
        }
    }

    //Testing Abstraction Principle
    public class Program
    {
        static void Main(string[] args)
        {
            DataProcessing processor = new DataProcessing();

            IDataFetcher dbFetcher = new DatabaseFetcher();
            IDataFetcher apiFetcher = new ApiFetcher();
            IDataFetcher fileFetcher = new FileFetcher();

            processor.ProcessData(dbFetcher);  // Outputs: Data from Database
            processor.ProcessData(apiFetcher); // Outputs: Data from API
            processor.ProcessData(fileFetcher); // Outputs: Data from File

            Console.Read();
        }
    }
}
```

در این مثال، با انتزاع:

- ما واکشی داده‌ها از منابع مختلف را یکپارچه کرده‌ایم.
- برای معرفی یک منبع داده جدید بدون تغییر کد موجود، فقط به یک پیاده‌سازی جدید از IDataFetcher نیاز است.
- کلاس DataProcessing از جزئیات خاص واکشی داده‌ها جدا می‌ماند و سیستم را ماژولارتر و مقیاس‌پذیرتر می‌کند.

این مثال نشان می‌دهد که چگونه انتزاع، انعطاف‌پذیری و توسعه‌پذیری را در طراحی نرم‌افزار ارتقا می‌دهد. این سیستم می‌تواند به راحتی تغییرات، اضافه کردن یا حذف منابع داده را بدون نیاز به تغییرات اساسی قابل توجه، بپذیرد.

##### **مثال بلادرنگ از اصل انتزاع در سی شارپ: سیستم مدیریت باغ وحش**

یک سیستم مدیریت باغ وحش را در نظر بگیرید که در آن انواع مختلفی از حیوانات وجود دارد و هر حیوان می‌تواند صدایی تولید کند. بدون انتزاع، برای صدای هر حیوان متدهای جداگانه‌ای خواهید داشت. اما با اصل انتزاع، می‌توانید این کار را ساده کنید. بیایید ببینیم چگونه می‌توانیم این مثال را با استفاده از اصل انتزاع در C# پیاده‌سازی کنیم:

```csharp
using System;

namespace AbstractionPrincipleCSharp
{
    //Abstraction Layer
    //First, define an interface that represents the common behavior of making a sound
    public interface IAnimal
    {
        void MakeSound();
    }

    //Concrete Implementations
    //Now, let's implement this interface for various animals
    public class Lion : IAnimal
    {
        public void MakeSound()
        {
            Console.WriteLine("Lion roars!");
        }
    }

    public class Snake : IAnimal
    {
        public void MakeSound()
        {
            Console.WriteLine("Snake hisses!");
        }
    }

    public class Bird : IAnimal
    {
        public void MakeSound()
        {
            Console.WriteLine("Bird chirps!");
        }
    }

    //Using the Abstraction
    //With the abstraction in place, you can easily handle any animal without knowing its specific type
    public class ZooKeeper
    {
        public void CheckAnimalSound(IAnimal animal)
        {
            animal.MakeSound();
        }
    }

    //Testing Abstraction Principle
    public class Program
    {
        static void Main(string[] args)
        {
            ZooKeeper zooKeeper = new ZooKeeper();

            IAnimal lion = new Lion();
            IAnimal snake = new Snake();
            IAnimal bird = new Bird();

            zooKeeper.CheckAnimalSound(lion);  // Outputs: Lion roars!
            zooKeeper.CheckAnimalSound(snake); // Outputs: Snake hisses!
            zooKeeper.CheckAnimalSound(bird);  // Outputs: Bird chirps!

            Console.Read();
        }
    }
}
```

###### **مزایا:**

- **ثبات** : ZooKeeper می‌تواند با هر حیوانی با استفاده از یک متد ثابت، MakeSound()، صرف نظر از نوع خاص حیوان، تعامل داشته باشد.
- **توسعه‌پذیری** : برای معرفی یک حیوان جدید به سیستم، شما یک کلاس جدید ایجاد می‌کنید که رابط IAnimal را پیاده‌سازی می‌کند. بقیه سیستم نیازی به تغییر ندارد.
- **جداسازی** : کلاس ZooKeeper اکنون از پیاده‌سازی‌های خاص حیوانات جدا شده است. این امر باعث می‌شود که کدبیس قابل نگهداری‌تر و درک آن آسان‌تر شود.

این طراحی، قدرت اصل انتزاع را در ارتقای قابلیت استفاده مجدد از کد، انعطاف‌پذیری و قابلیت نگهداری نشان می‌دهد.

##### **مثال بلادرنگ از اصل انتزاع در سی شارپ: سیستم رزرو هتل**

یک سیستم رزرو هتل را تصور کنید که در آن مهمانان می‌توانند اتاق‌های مختلفی مانند استاندارد، لوکس یا سوئیت رزرو کنند. هر نوع اتاق می‌تواند نرخ‌ها، امکانات و ظرفیت‌های متفاوتی داشته باشد. به جای داشتن متدهای جداگانه برای مدیریت هر نوع اتاق، از انتزاع برای ساده‌سازی مدیریت این اتاق‌ها استفاده خواهیم کرد. بیایید ببینیم چگونه می‌توانیم این مثال را با استفاده از اصل انتزاع در C# پیاده‌سازی کنیم:

```csharp
using System;

namespace AbstractionPrincipleCSharp
{
    //Abstraction Layer
    //Define an abstract class or interface that encapsulates the common behavior of all room types
    public interface IRoom
    {
        decimal Rate { get; }
        int Capacity { get; }
        string Description { get; }

        void DisplayRoomDetails();
    }

    //Concrete Implementations
    //Implement the interface for each room type
    public class StandardRoom : IRoom
    {
        public decimal Rate => 100.00M;
        public int Capacity => 2;
        public string Description => "Standard Room with basic amenities.";

        public void DisplayRoomDetails()
        {
            Console.WriteLine($"Type: Standard Room | Rate: {Rate:C} | Capacity: {Capacity} | Description: {Description}");
        }
    }

    public class DeluxeRoom : IRoom
    {
        public decimal Rate => 200.00M;
        public int Capacity => 4;
        public string Description => "Deluxe Room with premium amenities.";

        public void DisplayRoomDetails()
        {
            Console.WriteLine($"Type: Deluxe Room | Rate: {Rate:C} | Capacity: {Capacity} | Description: {Description}");
        }
    }

    public class Suite : IRoom
    {
        public decimal Rate => 500.00M;
        public int Capacity => 6;
        public string Description => "Luxurious suite with top-notch amenities.";

        public void DisplayRoomDetails()
        {
            Console.WriteLine($"Type: Suite | Rate: {Rate:C} | Capacity: {Capacity} | Description: {Description}");
        }
    }

    //Using the Abstraction
    //You can now handle room bookings in a unified manner
    public class HotelBooking
    {
        public void BookRoom(IRoom room)
        {
            room.DisplayRoomDetails();
            Console.WriteLine("Room booked successfully!\\n");
        }
    }
    
    //Testing Abstraction Principle
    public class Program
    {
        static void Main(string[] args)
        {
            HotelBooking booking = new HotelBooking();

            IRoom standard = new StandardRoom();
            IRoom deluxe = new DeluxeRoom();
            IRoom suite = new Suite();

            booking.BookRoom(standard);
            booking.BookRoom(deluxe);
            booking.BookRoom(suite);

            Console.Read();
        }
    }
}
```

###### **مزایا:**

- **انعطاف‌پذیری:** هتل می‌تواند در آینده انواع جدیدی از اتاق‌ها را تنها با ایجاد یک کلاس جدید که رابط IRoom را پیاده‌سازی می‌کند، معرفی کند.
- **ثبات:** کلاس HotelBooking صرف نظر از نوع اتاق‌ها، به طور مداوم با آنها تعامل دارد.
- **جداسازی:** منطق رزرو از جزئیات خاص اتاق جدا شده است و سیستم را قابل نگهداری‌تر و مقیاس‌پذیرتر می‌کند.

این مثال نشان می‌دهد که چگونه اصل انتزاع به ما امکان می‌دهد یک سیستم انعطاف‌پذیر و سازمان‌یافته طراحی کنیم و مدیریت ویژگی‌های مختلف را تحت یک رابط کاربری یکپارچه آسان کنیم.

##### **مزایا و معایب اصل کپسوله‌سازی در سی‌شارپ**

کپسوله‌سازی یکی از چهار اصل اولیه برنامه‌نویسی شی‌گرا (OOP) است. این به دسته‌بندی داده‌ها (ویژگی‌ها) و روش‌ها (توابع) که روی داده‌ها عمل می‌کنند در یک واحد واحد و محدود کردن دسترسی مستقیم به اجزای خاص شیء اشاره دارد. در C# (و سایر زبان‌های OOP)، این امر معمولاً با استفاده از اصلاح‌کننده‌های دسترسی مانند public، private، protected و internal حاصل می‌شود.

###### **مزایای اصل کپسوله‌سازی در سی‌شارپ:**

- **کنترل بر داده‌ها:** کپسوله‌سازی به شما امکان می‌دهد نحوه دسترسی یا تغییر داده‌ها را کنترل کنید. با افشای فیلدهای خصوصی از طریق ویژگی‌ها یا متدهای عمومی، می‌توانید اعتبارسنجی‌ها، تبدیل‌ها یا منطق اضافی را هنگام تنظیم یا بازیابی داده‌ها اضافه کنید.
- **ماژولاریتی:** با جداسازی دغدغه‌ها، طراحی ماژولار را ارتقا می‌دهد. هر کلاس مسئول رفتار خود است.
- **انعطاف‌پذیری و نگهداری:** از آنجایی که عملکرد داخلی یک کلاس پنهان است، تغییرات در منطق داخلی کلاس، بخش‌هایی از برنامه که از آن کلاس استفاده می‌کنند را تحت تأثیر قرار نمی‌دهد. این امر باعث می‌شود سیستم نسبت به تغییرات انعطاف‌پذیرتر و نگهداری آن آسان‌تر شود.
- **کاهش پیچیدگی:** کاربران کلاس فقط موارد ضروری را می‌بینند و این امر کار با کلاس را بدون غرق شدن در جزئیات آسان‌تر می‌کند.
- **افزایش امنیت:** کپسوله‌سازی با جلوگیری از عملیات غیرمجاز روی داده‌ها، از یکپارچگی آنها محافظت می‌کند. این امر به ویژه در محیط‌های چندرشته‌ای که ثبات داده‌ها از اهمیت بالایی برخوردار است، اهمیت دارد.
- **قابلیت استفاده مجدد:** کلاس‌های کپسوله‌شده، در صورت طراحی خوب، اغلب می‌توانند در بخش‌های مختلف یک برنامه یا حتی در پروژه‌های مختلف بدون تغییر، مجدداً مورد استفاده قرار گیرند.

###### **معایب اصل کپسوله‌سازی در سی‌شارپ:**

- **پیچیدگی اولیه:** معرفی کپسوله‌سازی می‌تواند مانند اضافه کردن پیچیدگی غیرضروری برای کارهای ساده به نظر برسد. استفاده از فیلدهای عمومی برای یک برنامه کوچک ممکن است سریع‌تر به نظر برسد، اما این می‌تواند با رشد برنامه منجر به مشکلاتی شود.
- **سربار عملکرد:** استفاده از getterها و setterها (متدها) برای دسترسی به ویژگی‌ها ممکن است در مقایسه با دسترسی مستقیم به فیلدها، سربار عملکرد کمی ایجاد کند. با این حال، کامپایلرهای مدرن و کامپایل just-in-time اغلب این نگرانی را کاهش می‌دهند.
- **سربار بالقوه در طراحی:** کپسوله‌سازی بیش از حد، یا ندانستن اینکه چه زمانی باید همه چیز را ساده نگه داشت، می‌تواند منجر به کد حجیم شود. گاهی اوقات، طراحی‌های بیش از حد تدافعی می‌توانند درک یا گسترش سیستم را دشوارتر کنند.
- **منحنی یادگیری برای توسعه‌دهندگان جدید:** برای کسانی که در OOP یا C# تازه‌کار هستند، درک کپسوله‌سازی و اهمیت آن ممکن است زمان‌بر باشد. اشتباهات در یادگیری اولیه، مانند افشای بیش از حد، می‌تواند منجر به شیوه‌های نادرست شود.
- **سوء استفاده احتمالی:** سوء تفاهم در مورد کپسوله‌سازی می‌تواند منجر به پنهان‌سازی نامناسب داده‌ها یا افشای بیش از حد آنها شود. برای مثال، اگر همه چیز عمومی شود، اصل کپسوله‌سازی نقض می‌شود.

اگرچه کپسوله‌سازی پیچیدگی ایجاد می‌کند، اما مزایای آن مانند قابلیت نگهداری، انعطاف‌پذیری و یکپارچگی داده‌ها اغلب بر معایب آن غلبه می‌کند. کپسوله‌سازی صحیح منجر به طراحی نرم‌افزار قوی‌تر و ماژولارتر می‌شود.
