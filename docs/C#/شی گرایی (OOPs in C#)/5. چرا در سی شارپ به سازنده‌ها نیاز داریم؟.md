## **چرا در سی شارپ به سازنده‌ها نیاز داریم؟**

در این مقاله، قصد دارم **اینکه چرا در سی‌شارپ به سازنده‌ها نیاز داریم،** با مثال‌های واقعی در مورد **بحث کنم. این بخش سوم از مبحث سازنده‌ها است. در بخش اول در مورد اینکه یک سازنده دقیقاً چیست** بحث کردیم **و در بخش دوم در مورد انواع مختلف سازنده‌ها در سی‌شارپ**. در این بخش، به شما نشان خواهم داد که چرا در کلاس خود به سازنده‌ها نیاز داریم.

##### **چرا در سی شارپ به سازنده‌ها نیاز داریم؟**

هر کلاسی برای ایجاد نمونه از آن کلاس، نیاز به یک سازنده (constructor) دارد. هر کلاسی اگر به طور صریح توسط برنامه‌نویس تعریف نشده باشد، حاوی یک سازنده ضمنی (implicit constructor) است و با کمک آن سازنده ضمنی، می‌توانیم نمونه‌ای از آن کلاس ایجاد کنیم.

اگر هیچ سازنده‌ای تعریف نکنیم، یک سازنده ضمنی وجود دارد که توسط کامپایلر در زمان کامپایل ارائه می‌شود و با استفاده از آن سازنده ضمنی می‌توانیم نمونه را ایجاد کنیم، حال سوال این است که چرا باید دوباره سازنده را به صراحت تعریف کنیم یا چه زمانی نیاز به تعریف یک سازنده صریح در سی شارپ داریم؟

##### **چه نیازی به تعریف مجدد صریح یک سازنده است؟**

این یکی از سوالات متداول مصاحبه است. بیایید این را درک کنیم. سازنده‌های ضمنی یک کلاس، متغیرهای یک کلاس را با مقدار یکسان مقداردهی اولیه می‌کنند، حتی اگر چندین نمونه از آن کلاس ایجاد کنیم.

بگذارید این را با یک مثال درک کنیم. لطفاً به کد زیر نگاهی بیندازید. در اینجا، ما یک کلاس به نام First با یک متغیر داریم و سپس از داخل متد Main، سه نمونه از کلاس First ایجاد می‌کنیم.

```csharp
using System;

namespace ConstructorDemo
{
    class First
    {
        public int x = 100;
    }
    class Test
    {
        static void Main(string[] args)
        {
            First f1 = new First();
            First f2 = new First();
            First f3 = new First();
            Console.WriteLine($"{f1.x}   {f2.x}   {f3.x}");
            Console.ReadKey();
        }
    }
}
```

حالا، به صورت داخلی، همانطور که در تصویر زیر نشان داده شده است، حافظه را به طور جداگانه برای هر نمونه اختصاص می‌دهد. نکته‌ای که باید به خاطر داشته باشید، ما سه نمونه ایجاد کردیم و این سه نمونه یک کپی از x دارند. و مقدار برای همه یکسان خواهد بود.

![چه نیازی به تعریف مجدد صریح یک سازنده است؟](https://dotnettutorials.net/wp-content/uploads/2022/06/word-image-27828-1.png "چه نیازی به تعریف مجدد صریح یک سازنده است؟")

مشکل همین جاست. در حال حاضر، کلاس شامل یک سازنده به نام سازنده ضمنی (Implicit constructor) است. و آن سازنده قرار است متغیر x را با مقدار ۱۰۰ مقداردهی اولیه کند. بنابراین، هر چند نمونه که قرار است ایجاد شود، همه نمونه‌ها با مقدار یکسانی ایجاد خواهند شد.

اگر سازنده‌ها را به طور صریح با پارامترها تعریف کنیم، می‌توانیم هر بار که می‌خواهیم نمونه‌ای از آن کلاس ایجاد کنیم، فیلدها یا متغیرهای کلاس را با مقدار جدید مقداردهی اولیه کنیم.

حال به مثال زیر توجه کنید، در اینجا ما کلاس دیگری به نام Second ایجاد کرده‌ایم و این کلاس متغیری به نام x دارد. حال، سازنده ضمنی، متغیر x را با مقدار پیش‌فرض ۰ مقداردهی اولیه می‌کند، زمانی که یک نمونه از کلاس Second ایجاد می‌کنیم و برای هر نمونه نیز به همین ترتیب.

```csharp
using System;

namespace ConstructorDemo
{
    class First
    {
        public int x = 100;
    }
    class Second
    {
        //Initialize with default value
        public int x;
    }
    class Test
    {
        static void Main(string[] args)
        {
            First f1 = new First();
            First f2 = new First();
            First f3 = new First();
            Console.WriteLine($"{f1.x}   {f2.x}   {f3.x}");

            Second s1 = new Second();
            Second s2 = new Second();
            Second s3 = new Second();
            Console.WriteLine($"{s1.x}   {s2.x}   {s3.x}");
            Console.ReadKey();
        }
    }
}
```

حالا می‌خواهم مقدار x در سه نمونه متفاوت باشد. نمی‌دانم چه مقداری را، وقتی می‌خواهم نمونه‌ها را ایجاد کنم، متوجه آن خواهم شد. یک نکته را به خاطر داشته باشید، هر زمان که یک کلاس ایجاد می‌کنیم، به این معنی است که می‌توانیم از کلاس دوباره استفاده کنیم و می‌توانیم نمونه را هر زمان و هر کجا که می‌خواهیم ایجاد کنیم. این قابلیت استفاده مجدد از کد است.

بنابراین، در آینده هر زمان که بخواهم نمونه‌ای از کلاس Second ایجاد کنم، فقط باید مقدار x را ارسال کنم. این سناریویی است که در آن باید یک سازنده با پارامتر صریح تعریف کنیم. بیایید کلاس Second را به صورت زیر تغییر دهیم تا یک سازنده با پارامتر صریح برای مقداردهی اولیه متغیر x اضافه کنیم.

و در اینجا x به متغیر محلی x اشاره دارد. ببینید، وقتی x را انتخاب می‌کنم، به طور خودکار متغیر محلی x همانطور که در تصویر زیر نشان داده شده است، هایلایت می‌شود.

این یعنی متغیر محلی x به متغیر کلاس x انتساب داده می‌شود. حال، هنگام ایجاد نمونه از کلاس Second، باید یک مقدار ارسال کنیم و آن مقدار در متغیر محلی x ذخیره می‌شود. و به این ترتیب، هنگام ایجاد چندین نمونه از کلاس Second، می‌توانیم مقادیر بسیار متفاوتی را همانطور که در تصویر زیر نشان داده شده است، ارسال کنیم.

کد کامل مثال در زیر آمده است.

```csharp
using System;

namespace ConstructorDemo
{
    class First
    {
        public int x = 100;
    }
    
    class Second
    {
        public int x;

        //Parameterized Explicit Constructor
        public Second(int x)
        {
            this.x = x;
        }
    }
    
    class Test
    {
        static void Main(string[] args)
        {
            First f1 = new First();
            First f2 = new First();
            First f3 = new First();
            Console.WriteLine($"{f1.x}   {f2.x}   {f3.x}");

            Second s1 = new Second(100); //100 wll send to local variable x
            Second s2 = new Second(200); //200 wll send to local variable x
            Second s3 = new Second(300); //300 wll send to local variable x

            Console.WriteLine($"{s1.x}   {s2.x}   {s3.x}");
            Console.ReadKey();
        }
    }
}
```

وقتی یک کلاس تعریف می‌کنیم، ابتدا مشخص می‌کنیم که آیا متغیرهای کلاس برای اجرا به مقادیری نیاز دارند یا خیر، و اگر چنین مقادیری لازم هستند، آنگاه یک سازنده (constructor) را به صراحت تعریف می‌کنیم و مقادیر را از طریق سازنده ارسال می‌کنیم، به طوری که هر بار که نمونه‌ای از کلاس ایجاد می‌شود، شانس ارسال مقادیر جدید را داشته باشیم.

**نکته:** به طور کلی، هر کلاس برای اجرا به برخی مقادیر نیاز دارد و مقادیری که برای اجرای یک کلاس مورد نیاز هستند، همیشه فقط با استفاده از سازنده به آن کلاس ارسال می‌شوند.

##### **مثال بلادرنگ سازنده پارامتری در سی شارپ**

با کمک یک سازنده پارامتری، می‌توانیم هر نمونه از کلاس را با مجموعه‌ای متفاوت از مقادیر مقداردهی اولیه کنیم. این بدان معناست که با استفاده از سازنده پارامتری می‌توانیم مجموعه‌ای متفاوت از مقادیر را در اشیاء مختلف ایجاد شده در کلاس ذخیره کنیم. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید.

```csharp
using System;

namespace ConstructorDemo
{
    class Employee
    {
        public int Id, Age;
        public string Address, Name;
        public bool IsPermanent;

        //User Defined Parameterized Constructor
        public Employee(int id, int age, string name, string address, bool isPermanent)
        {
            Id = id;
            Age = age;
            Address = address;
            Name = name;
            IsPermanent = isPermanent;
        }

        public void Display()
        {
            Console.WriteLine("Employee Id is:  " + Id);
            Console.WriteLine("Employee Name is:  " + Age);
            Console.WriteLine("Employee Address is:  " + Address);
            Console.WriteLine("Employee Name is:  " + Name);
            Console.WriteLine("Is Employee Permanent:  " + IsPermanent);
        }
    }

    class Program
    {
        static void Main(string[] args)
        {
            Employee e1 = new Employee(101, 30, "Pranaya", "Mumbai", true);
            e1.Display();
            Console.WriteLine();

            Employee e2 = new Employee(101, 28, "Rout", "BBSR", false);
            e2.Display();
            Console.ReadKey();
        }
    }
}
```

##### **مثال بلادرنگ از سازنده کپی در سی شارپ**

سازنده‌ای که پارامتری از نوع کلاس می‌گیرد، سازنده کپی نامیده می‌شود. این سازنده برای کپی کردن داده‌های یک شیء در شیء دیگر استفاده می‌شود. هدف اصلی سازنده کپی، مقداردهی اولیه یک شیء جدید (نمونه) با مقادیر یک شیء موجود (نمونه) است. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید.

```csharp
using System;

namespace ConstructorDemo
{
    class Employee
    {
        public int Id, Age;
        public string Address, Name;
        public bool IsPermanent;

        //Parameterized Constructor
        public Employee(int id, int age, string name, string address, bool isPermanent)
        {
            Id = id;
            Age = age;
            Address = address;
            Name = name;
            IsPermanent = isPermanent;
        }

        //Copy Constructor
        public Employee(Employee emp)
        {
            Id = emp.Id;
            Age = emp.Age;
            Address = emp.Address;
            Name = emp.Name;
            IsPermanent = emp.IsPermanent;
        }

        public void Display()
        {
            Console.WriteLine("Employee Id is:  " + Id);
            Console.WriteLine("Employee Name is:  " + Age);
            Console.WriteLine("Employee Address is:  " + Address);
            Console.WriteLine("Employee Name is:  " + Name);
            Console.WriteLine("Is Employee Permanent:  " + IsPermanent);
        }
    }

    class Program
    {
        static void Main(string[] args)
        {
            Employee e1 = new Employee(101, 30, "Pranaya", "Mumbai", true);
            e1.Display();
            Console.WriteLine();
            
            Employee e2 = new Employee(e1);
            e2.Display();
            Console.ReadKey();
        }
    }
}
```

##### **سازنده استاتیک** **مثال بلادرنگ** **در سی شارپ**

سازنده‌ی استاتیک در سی‌شارپ فقط یک بار فراخوانی می‌شود. فرقی نمی‌کند چند نمونه (شیء) از کلاس ایجاد شود، فقط یک بار فراخوانی می‌شود و آن هم زمانی است که کلاس برای اولین بار بارگذاری می‌شود.

سازنده استاتیک برای مقداردهی اولیه فیلدهای استاتیک کلاس استفاده می‌شود. همچنین می‌توانید کدی را درون سازنده استاتیک بنویسید که فقط یک بار اجرا شود. اعضای داده استاتیک در C# فقط یک بار ایجاد می‌شوند، حتی اگر هر تعداد شیء ایجاد کنیم.

```csharp
using System;

namespace StaticConstructorDemo
{
    class Example
    {
        int i;
        static int j;

        //Default Constructor
        public Example()
        {
            Console.WriteLine("Default Constructor Executed");
            i = 100;
        }

        //static Constructor
        static Example()
        {
            Console.WriteLine("Static Constructor Executed");
            j = 100;
        }
        public void Increment()
        {
            i++;
            j++;
        }
        public void Display()
        {
            Console.WriteLine("Value of i : " + i);
            Console.WriteLine("Value of j : " + j);
        }
    }
    class Test
    {
        static void Main(string[] args)
        {
            Example e1 = new Example();
            e1.Increment();
            e1.Display();
            e1.Increment();
            e1.Display();
            Example e2 = new Example();
            e2.Increment();
            e2.Display();
            e2.Increment();
            e2.Display();
            Console.ReadKey();
        }
    }
}
```
