## **تفاوت‌های بین Finalize و Dispose در سی شارپ**

در این مقاله، تفاوت‌های بین Finalize و Dispose در سی‌شارپ را با مثال توضیح خواهم دا در سی‌شارپ، Finalize و Dispose هر دو روشی هستند که برای آزادسازی منابع استفاده می‌شوند، اما اهداف متفاوتی را دنبال می‌کنند و در سناریوهای مختلف مورد استفاده قرار می‌گیرند:

##### **متد Finalize در سی شارپ:**

- **هدف:** متد Finalize برای عملیات پاکسازی قبل از جمع‌آوری زباله (garbage collection) یک شیء استفاده می‌شود. این متد معمولاً برای آزادسازی منابع مدیریت نشده‌ای که شیء در اختیار دارد، بازنویسی (override) می‌شود. جمع‌آوری‌کننده زباله (garbage collector) به طور خودکار متد Finalize را فراخوانی می‌کند.
- **کنترل:** شما تابع Finalize را مستقیماً فراخوانی نمی‌کنید. این تابع توسط garbage collector فراخوانی می‌شود.
- **غیر قطعی:** زمان دقیق فراخوانی Finalize غیر قطعی است و به برنامه‌ی جمع‌آوری زباله بستگی دارد.
- **وراثت:** متد Finalize از کلاس Object به ارث رسیده است. در صورت لغو شدن، باید همیشه متد Finalize کلاس پایه خود را فراخوانی کند تا از آزادسازی صحیح همه منابع اطمینان حاصل شود.

##### **روش دفع:**

- **هدف:** متد Dispose بخشی از رابط IDisposable است و برای آزادسازی قطعی منابع مدیریت‌شده و مدیریت‌نشده پیاده‌سازی شده است.
- **کنترل:** برخلاف Finalize، Dispose به طور صریح در کد شما فراخوانی می‌شود، معمولاً زمانی که استفاده از یک شیء تمام می‌شود. این امر امکان آزادسازی فوری منابع را فراهم می‌کند.
- **قطعی:** Dispose یک روش قطعی برای آزادسازی منابع ارائه می‌دهد، به این معنی که شما دقیقاً می‌دانید منابع چه زمانی آزاد می‌شوند.
- **الگو:** هنگام پیاده‌سازی Dispose، معمولاً از الگوی dispose پیروی می‌شود که شامل فراخوانی finalizer (GC.SuppressFinalize(this)) است تا از فراخوانی Finalize توسط garbage collector در صورتی که Dispose قبلاً فراخوانی شده باشد، جلوگیری شود.

##### **مثالی برای درک finalize و dispose در سی شارپ**

بیایید مثالی بزنیم تا کاربرد متدهای Finalize و Dispose را در مدیریت منابع درک کنیم. در این مثال، یک کلاس ساده به نام ResourceHolder ایجاد خواهیم کرد که مدیریت یک منبع مدیریت نشده را شبیه‌سازی می‌کند. مثال زیر نیازی به توضیح ندارد، بنابراین لطفاً برای درک بهتر، خط توضیحات را مطالعه کنید.

```csharp
using System;

namespace GarbageCollectionDemo
{
    public class ResourceHolder : IDisposable
    {
        // To track whether Dispose has been called.
        private bool _disposed = false;

        // Constructor
        public ResourceHolder()
        {
            // Allocate or initialize an unmanaged resource.
            Console.WriteLine("Unmanaged resource allocated.");
        }

        // Implementing Dispose method from IDisposable interface
        public void Dispose()
        {
            Dispose(true);
            GC.SuppressFinalize(this); // Prevent finalizer from being called.
        }

        protected virtual void Dispose(bool disposing)
        {
            if (!_disposed)
            {
                if (disposing)
                {
                    // Free any other managed objects here.
                    Console.WriteLine("Free other managed objects");
                }

                // Free unmanaged resources here.
                Console.WriteLine("Unmanaged resource released.");
                _disposed = true;
            }
        }

        // Finalizer is nothing but the destructor
        ~ResourceHolder()
        {
            Dispose(false);
            Console.WriteLine("Finalizer called.");
        }
    }

    //How to Use the above class
    class Program
    {
        static void Main(string[] args)
        {
            // Using the ResourceHolder with using statement
            using (var resourceHolder = new ResourceHolder())
            {
                // Use the resource...
            } // Dispose is called automatically when exiting the using block.

            // If not using 'using', dispose should be called manually.
            var anotherResourceHolder = new ResourceHolder();
            // Use the resource...
            anotherResourceHolder.Dispose();

            // Without calling Dispose, finalizer will be called by GC at some point.
            var finalResourceHolder = new ResourceHolder();
            // Use the resource...

            Console.ReadKey();
        }
    }
}
```

##### **توضیح:**

###### **کلاس دارنده منابع:**

- کلاس ResourceHolder مدیریت یک منبع مدیریت نشده را شبیه‌سازی می‌کند.
- متد Dispose از رابط IDisposable پیاده‌سازی شده است. این متد برای آزادسازی قطعی منابع فراخوانی می‌شود.
- متد protected Dispose(bool disposing) پاکسازی واقعی منابع را انجام می‌دهد. پارامتر disposing نشان می‌دهد که آیا متد از متد Dispose فراخوانی می‌شود یا از finalizer.
- اگر شیء به درستی از بین نرود، تابع finalizer (~ResourceHolder) توسط garbage collector فراخوانی می‌شود. این تابع Dispose(false) را فراخوانی می‌کند.

###### **استفاده از کلاس:**

- دستور using تضمین می‌کند که Dispose به طور خودکار برای resourceHolder فراخوانی شود.
- اگر از using استفاده نمی‌شود، Dispose باید به صورت دستی فراخوانی شود، همانطور که با anotherResourceHolder نشان داده شده است.
- finalResourceHolder سناریویی را نشان می‌دهد که در آن Dispose فراخوانی نمی‌شود، از این رو finalizer در نهایت توسط garbage collector فراخوانی خواهد شد.

مثال بالا پیاده‌سازی و استفاده صحیح از Dispose و Finalize را برای مدیریت منابع در سی‌شارپ نشان می‌دهد.

##### **تفاوت‌های کلیدی بین Finalize و Dispose در سی شارپ:**

- **زمان‌بندی:** تابع Finalize توسط garbage collector به صورت غیرقطعی فراخوانی می‌شود، در حالی که تابع Dispose به طور صریح در نقطه‌ای مشخص از برنامه فراخوانی می‌شود.
- **منابع:** Finalize معمولاً برای منابع مدیریت نشده استفاده می‌شود، در حالی که Dispose می‌تواند هم برای منابع مدیریت شده و هم برای منابع مدیریت نشده استفاده شود.
- **کنترل:** Dispose در مقایسه با Finalize کنترل بیشتری بر مدیریت منابع به شما می‌دهد.

##### **بهترین شیوه‌ها:**

- پیاده‌سازی Dispose برای پاکسازی قطعی منابع.
- از یک finalizer (متد Finalize) فقط برای پاکسازی منابع مدیریت نشده‌ای که در یک دسته امن قرار نگرفته‌اند و زمانی که هیچ تضمینی برای فراخوانی Dispose وجود ندارد، استفاده کنید.
- در متد Dispose، اگر قبلاً از شر Finalize خلاص شده‌اید، برای جلوگیری از فراخوانی Finalize توسط garbage collector، GC.SuppressFinalize را فراخوانی کنید.
- از الگوی dispose پیروی کنید، به خصوص اگر کلاس شما دارای منابع مدیریت نشده است.
