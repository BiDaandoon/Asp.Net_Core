## **سازنده‌ها در سی شارپ به همراه مثال**

در این مقاله، قصد دارم در مورد **سازنده‌ها در سی‌شارپ** با مثال‌ها صحبت کنم. برنامه‌نویسی شی‌گرا تماماً در مورد نوشتن کد درون یک کلاس است. یک کلاس مجموعه‌ای از اعضا مانند فیلدها، متدها، سازنده‌ها و غیره است. در داخل یک کلاس، یکی از مهمترین اعضای موجود، سازنده نامیده می‌شود.

##### **سازنده (Constructor) در سی شارپ چیست؟**

این یک متد ویژه است که درون یک کلاس وجود دارد و مسئول مقداردهی اولیه متغیرهای آن کلاس است. در ادامه این مقاله به این نکته خواهیم پرداخت.

نام متد سازنده دقیقاً همان نام کلاسی است که در آن وجود داشته است. شما نمی‌توانید نام را تغییر دهید. اگر نام کلاس شما Employee است، نام متد سازنده نیز Employee خواهد بود و اگر نام کلاس شما Student است، نام محدودکننده نیز Student خواهد بود.

متد سازنده هیچ مقداری را برنمی‌گرداند. این بدان معناست که این متد، متدی بدون مقدار است. به طور کلی، متدها دو نوع هستند: متدهای با مقدار و متدهای با مقدار. متدهای با مقدار کاملاً بدون مقدار هستند. یعنی هرگز هیچ مقداری را برنمی‌گردانند.

##### **مثالی برای درک سازنده در سی شارپ**

اگر بخواهیم نمونه‌ای از کلاس ایجاد کنیم، تک تک کلاس‌ها به این سازنده نیاز دارند. اگر سازنده‌ای نداشته باشیم، نمی‌توانیم نمونه‌ای از کلاس ایجاد کنیم. در این مرحله، شما یک شک دارید، قبلاً کلاس‌های زیادی تعریف کرده‌ایم اما هرگز از سازنده استفاده نکرده‌ایم، اما با این حال، می‌توانیم نمونه‌ای از کلاس ایجاد کنیم، چگونه؟ بیایید این شک را روشن کنیم. فرض کنید کلاسی به شکل زیر داریم:

```csharp
class Test
{
    int i;
}
```

سپس، یک نمونه از کلاس Test فوق را در جایی از برنامه خود به صورت زیر ایجاد می‌کنیم:

**Test obj = new Test();**

آیا عبارت بالا معتبر است؟ بله، معتبر است. دلیل آن این است که تعریف یک سازنده در کلاس خود بر عهده‌ی برنامه‌نویس است و اگر او این کار را انجام ندهد، از طرف برنامه‌نویس، یک سازنده‌ی ضمنی توسط کامپایلر در آن کلاس تعریف می‌شود. برای درک بهتر، لطفاً به نمودار زیر که کد را قبل و بعد از کامپایل نشان می‌دهد، نگاهی بیندازید.

![مثالی برای درک سازنده در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2018/07/word-image-1122-1.png "مثالی برای درک سازنده در سی شارپ")

در اینجا می‌توانید ببینید که پس از کامپایل، کامپایلر سازنده عمومی را به کلاس اضافه می‌کند و متغیر را مقداردهی اولیه می‌کند و این مسئولیت یک سازنده است، یعنی مقداردهی اولیه متغیرهای آن کلاس. در اینجا، مقداردهی اولیه متغیر با 0 است. اگر یک متغیر کلاس به طور ضمنی مقداردهی اولیه شود، به این معنی است که توسط یک سازنده انجام می‌شود.

هر متغیری که درون یک کلاس تعریف می‌کنیم و هر فیلدی که درون یک کلاس تعریف می‌کنیم، یک مقدار پیش‌فرض دارد. همه انواع عددی با ۰، انواع بولی با false و انواع رشته‌ای و شیء با null مقداردهی اولیه می‌شوند. برای درک بهتر، لطفاً به تصویر زیر نگاهی بیندازید.

![سازنده (Constructor) در سی شارپ چیست؟](https://dotnettutorials.net/wp-content/uploads/2018/07/word-image-1122-2.png "سازنده (Constructor) در سی شارپ چیست؟")

به این ترتیب، مقداردهی اولیه برای تک تک متغیرهای موجود در کلاس انجام می‌شود و این مسئولیت سازنده است. به همین دلیل است که وجود سازنده در داخل کلاس برای ما بسیار مهم است.

ما مقداری را اختصاص نمی‌دهیم، اما وقتی مقداری به متغیرها اختصاص داده می‌شود، به این معنی است که کسی آن مقدار را به این متغیرها اختصاص داده است. خب، چه کسی قرار است این کار را انجام دهد؟ سازنده قرار است این کار را انجام دهد. و این سازنده توسط ما تعریف نشده است. پس چه کسی این سازنده را تعریف کرده است؟ کامپایلر این سازنده را برای ما تعریف کرده است. و ما به آن سازنده ضمنی می‌گوییم. و اگر ما هم همین را تعریف کنیم، به آن سازنده صریح می‌گویند.

##### **نکاتی که باید هنگام کار با سازنده‌ها در سی شارپ به خاطر داشته باشید:**

1. سازنده‌های تعریف‌شده ضمنی، بدون پارامتر هستند و به عنوان سازنده‌های پیش‌فرض نیز شناخته می‌شوند. دلیل این امر این است که از آنها برای مقداردهی اولیه متغیرها با مقادیر پیش‌فرض استفاده می‌شود.
2. سازنده‌های تعریف‌شده ضمنی عمومی هستند. اگر در مثال ما ببینید، کلاس Test را با یک مشخص‌کننده دسترسی پیش‌فرض تعریف می‌کنیم، اما سازنده آن عمومی است که توسط کامپایلر تولید می‌شود.
3. ما همچنین می‌توانیم یک سازنده (constructor) زیر کلاس تعریف کنیم و اگر آن را تعریف کنیم، می‌توانیم آن را سازنده صریح (Explicit Constructor) بنامیم و یک سازنده صریح می‌تواند بدون پارامتر باشد و همچنین پارامتردهی شود.

##### **مثال برای درک سازنده ضمنی در سی شارپ:**

در مثال زیر، ما یک کلاس با سه متغیر ایجاد می‌کنیم و در کلاس، هیچ سازنده‌ای را به طور صریح تعریف نکرده‌ایم. بنابراین، در اینجا کامپایلر سازنده ضمنی را ارائه می‌دهد و متغیرها را با مقدار پیش‌فرض مقداردهی اولیه می‌کند. سپس از متد Main، یک نمونه از کلاس ایجاد می‌کنیم و مقادیر متغیرها را چاپ می‌کنیم و باید مقادیر پیش‌فرض را بر اساس نوع متغیر چاپ کند. به عنوان مثال، برای int، مقدار پیش‌فرض 0، برای bool مقدار پیش‌فرض false و برای string یا object مقدار پیش‌فرض null است. و این مقداردهی اولیه پیش‌فرض توسط سازنده ضمنی که توسط کامپایلر ارائه می‌شود، انجام می‌شود.

```csharp
using System;

namespace ConstructorDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Test obj = new Test();
            Console.WriteLine($"i = {obj.i}");
            Console.WriteLine($"b = {obj.b}");

            //value null will be printed, so here we checking the null
            if (obj.s == null)
            {
                Console.WriteLine("s = null");
            }

            Console.ReadKey();
        }
    }
    class Test
    {
        public int i;
        public bool b;
        public string s;
    }
}
```

```csharp
using System;

namespace ConstructorDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            ExplicitConstructor obj = new ExplicitConstructor();
            
            Console.ReadKey();
        }
    }
    class ExplicitConstructor
    {
        public ExplicitConstructor()
        {
            Console.WriteLine("Explicit Constructor is Called!");
        }
    }
}
```

##### **تعریف و فراخوانی سازنده در سی شارپ:**

**تعریف کردن:** تعریف یک سازنده به معنای پیاده‌سازی یک سازنده در کلاس شماست. تعریف کردن می‌تواند دو نوع باشد: ضمنی (Implicit) و صریح (Explicit). ضمنی به این معنی است که کامپایلر سازنده را تعریف می‌کند. صریح به این معنی است که ما به عنوان یک برنامه‌نویس سازنده را تعریف می‌کنیم. کد زیر تعریف یک سازنده را به صورت صریح نشان می‌دهد.

#### **سوالات متداول مصاحبه:**

##### **سازنده (Constructor) در سی شارپ چیست؟**

به عبارت ساده، می‌توانیم سازنده‌ها را در سی‌شارپ تعریف کنیم که انواع خاصی از متدهای یک کلاس هستند که هر زمان که یک نمونه (شیء) از آن کلاس ایجاد می‌کنیم، اجرا می‌شوند. سازنده‌ها مسئول دو چیز هستند. یکی مقداردهی اولیه شیء و دیگری تخصیص حافظه. نقش کلمه کلیدی new ایجاد شیء و نقش سازنده مقداردهی اولیه متغیرها است.

##### **هنگام کار با سازنده در سی شارپ چه قوانینی را باید رعایت کرد؟**

1. نام سازنده باید با نام کلاس یکسان باشد.
2. نباید حاوی نوع داده‌ی برگشتی باشد، حتی void هم نباید باشد.
3. به عنوان بخشی از بدنه سازنده، دستور return با مقدار مجاز نیست.

##### **سازنده (Constructor) در سی شارپ چه کاربردی دارد؟**

1. می‌تواند هر پنج اصلاح‌کننده دسترسی یعنی عمومی، خصوصی، محافظت‌شده و غیره را داشته باشد.
2. سازنده می‌تواند بدون پارامتر یا با پارامتر باشد.
3. می‌تواند یک بند throws داشته باشد که به این معنی است که می‌توانیم یک استثنا را از سازنده پرتاب کنیم.
4. سازنده می‌تواند منطق داشته باشد، به عنوان بخشی از منطق، می‌تواند تمام دستورات مجاز C#.NET به جز دستورات return با مقدار را داشته باشد.
5. می‌توانیم یک return; در سازنده قرار دهیم.
