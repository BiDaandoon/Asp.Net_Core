## **عمومی‌سازی و تخصصی‌سازی در سی‌شارپ با مثال‌ها**

در این مقاله، قصد دارم در **مورد تعمیم و تخصص در سی شارپ** با مثال بحث کنم. بخوانید. این یک موضوع بسیار جالب است که تعمیم در مقابل تخصص است. این مربوط به وراثت است. بگذارید منظور شما از تعمیم و تخصص در سی شارپ با مثال را بفهمیم.

##### **تخصص در زبان سی شارپ**

فرض کنید کلاسی به نام Rectangle داریم. آیا می‌توانیم از کلاس Rectangle شیء بسازیم؟ بله، می‌توانیم اشیاء Rectangle بسازیم و توابع کلاس Rectangle را فراخوانی کنیم. حال، کلاسی به نام Cuboid داریم که از کلاس Rectangle ارث‌بری شده است.

![تخصص در زبان سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/06/generalization-and-specialization-in-c-with-exam.png "تخصص در زبان سی شارپ")

حالا، برای کلاس Cuboid هم می‌توانیم اشیاء ایجاد کنیم و توابع Cuboid و Rectangle را فراخوانی کنیم. این دو چیز، یعنی **Rectangle و Cuboid، واقعاً در جهان وجود دارند** . می‌توانید یک Rectangle به من نشان دهید؟ بله، می‌توانید چند شکل مستطیلی یا کاغذی که مستطیلی است را به من نشان دهید. بنابراین، مستطیل‌ها در دنیای واقعی وجود دارند. و Cuboid چطور؟ بله، آن هم در دنیای واقعی وجود دارد. هر چیز جعبه‌ای شکلی، یک مکعب مستطیل است.

بنابراین، این دو چیز در دنیای واقعی وجود دارند. حال، اگر بگویم کدام یک در مثال ما ابتدا وجود دارد؟ پاسخ Rectangle خواهد بود. و از کلاس Rectangle، کلاس Cuboid را مشتق کرده‌ایم. بنابراین، کلاس Rectangle از قبل وجود دارد و از کلاس Rectangle، یک کلاس جدید با برخی ویژگی‌های اضافی تعریف کرده‌ایم، یعنی یک کلاس تخصصی به نام Cuboid داریم. این یک تخصص در C# است.

##### **مثال برای درک تخصص در سی شارپ:**

در مثال زیر ابتدا کلاس Rectangle را ایجاد کرده‌ایم و از کلاس Rectangle، یک کلاس تخصصی با برخی ویژگی‌های اضافی به نام کلاس Cuboid ایجاد کرده‌ایم. بنابراین، Cuboid یک کلاس تخصصی از Rectangle است.

```csharp
using System;

namespace IsAHasADemo
{
    public class Program
    {
        static void Main()
        {
            Cuboid cuboid = new Cuboid(3, 5, 7);
            Console.WriteLine($"Volume is : {cuboid.Volume()}");
            Console.WriteLine($"Area is : {cuboid.Area()}");
            Console.WriteLine($"Perimeter is : {cuboid.Perimeter()}");
            
            Console.ReadKey();
        }
    }
    class Rectangle
    {
        public int Length;
        public int Breadth;
        public int Area()
        {
            return Length * Breadth;
        }
        public int Perimeter()
        {
            return 2 * (Length + Breadth);
        }
    }
    class Cuboid : Rectangle
    {
        public int Height;
        public Cuboid(int l, int b, int h)
        {
            Length = l;
            Breadth = b;
            Height = h;
        }
        public int Volume()
        {
            return Length * Breadth * Height;
        }
    }
}
```

![عمومی‌سازی و تخصصی‌سازی در سی‌شارپ با مثال‌ها](https://dotnettutorials.net/wp-content/uploads/2022/06/generalization-and-specialization-in-c-with-exam-1.png "عمومی‌سازی و تخصصی‌سازی در سی‌شارپ با مثال‌ها")

اگر از این موارد اطلاع دارید، باید بدانید که ابتدا اینووا توسط تویوتا عرضه شد و سپس خودروی جدیدی به نام فورچونر به بازار عرضه شد که امتداد اینووا است. به عنوان یک فرد عادی، اگر فورچونر را امتداد اینووا بدانید، در مقایسه با اینووا، ظرفیت صندلی یکسانی دارد اما ویژگی‌های اضافی زیادی دارد. بنابراین، به این معنی است که خودروی فورچونر امتداد خودروی اینووا است.

Innova یک کلاس پایه و Fortuner یک کلاس مشتق شده است. Innova از قبل وجود داشته و Fortuner شکل تخصصی Innova است. این تخصص است. **چیزی از قبل وجود دارد و سپس ما با اضافه کردن برخی ویژگی‌های اضافی، چیزی را از آن موجود مشتق می‌کنیم.** در دنیای واقعی، می‌توانیم مثال‌های زیادی از تخصص بزنیم.

##### **تعمیم در سی شارپ**

فرض کنید سه کلاس داریم که عبارتند از Square، Rectangle و Circle. سپس برای هر سه این کلاس‌ها، یک کلاس به نام Shape تعریف می‌کنیم. Square، Rectangle و Circle از کلاس Shape به ارث برده‌اند. برای درک بهتر، لطفاً به نمودار زیر نگاهی بیندازید.

![تعمیم در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/06/generalization-and-specialization-in-c.png "تعمیم در سی شارپ")

خب، از بین این کلاس‌ها، کدام یک زودتر وجود دارد؟ می‌دانیم که همه شکل‌ها، یعنی مربع، مستطیل و دایره، واقعاً در دنیای واقعی وجود دارند. بنابراین، ما آنها را شکل می‌نامیم.

آیا فکر می‌کنید شکل واقعاً در دنیای واقعی وجود دارد؟ خیر. این فقط یک اصطلاح مجازی است. اگر از شما بخواهم چند شکل به من نشان دهید، شما یک مستطیل، یک دایره، یک مربع، یک لوزی و غیره رسم خواهید کرد، اما نمی‌توانید فقط یک شکل را به من نشان دهید. بنابراین، شکل یک اصطلاح یا اصطلاح عمومی یا اصطلاح تعمیم‌یافته است که در دنیای واقعی وجود ندارد.

این کلاس‌های فرزند که از قبل در دنیای واقعی وجود دارند و برای اینکه آنها را روی یک پلتفرم مشترک قرار دهیم، کلمه Shape را به آنها داده‌ایم که یک اصطلاح منطقی یا مجازی است، یک اصطلاح واقعی نیست، واقعاً در دنیای واقعی وجود ندارد.

آیا می‌توانید مساحت یا محیط یک مستطیل را پیدا کنید؟ بله. آیا می‌توانید مساحت یا محیط مربع را پیدا کنید؟ بله. آیا می‌توانید مساحت یا محیط دایره را پیدا کنید؟ بله. آیا هر شکلی مساحت و محیط دارد؟ بله.

این یک ویژگی رایج است، بنابراین ما آن را به شکل درآورده‌ایم. حالا آیا می‌توانید مساحت و محیط شکل را محاسبه کنید؟ نه، نمی‌توانید. هیچ بعدی برای شکل وجود ندارد. این یک اصطلاح تعمیم‌یافته است. بنابراین، این یک مثال از تعمیم‌پذیری در سی‌شارپ است.

##### **مثال دیگری برای درک تعمیم:**

فرض کنید شما ماشین‌های اینووا، بی‌ام‌و و آئودی دارید. بنابراین، می‌توانید در هر یک از این‌ها روشن، خاموش، گاز دهید و دنده عوض کنید. خب، همه اینها چه هستند؟ همه اینها ماشین هستند. اینووا، بی‌ام‌و و آئودی از نظر فیزیکی در دنیای واقعی موجود هستند. همه اینها چیزی جز ماشین نیستند.

![عمومی‌سازی و تخصصی‌سازی در سی‌شارپ با مثال‌ها](https://dotnettutorials.net/wp-content/uploads/2022/06/generalization-and-specialization-in-c-1.png "عمومی‌سازی و تخصصی‌سازی در سی‌شارپ با مثال‌ها")

بنابراین، همه آنها از ماشین‌ها ارث برده‌اند. در یک ماشین چه کاری می‌توانید انجام دهید؟ می‌توانید رانندگی کنید، روشن کنید، توقف کنید، دنده عوض کنید و غیره. همه این ویژگی‌ها در این ماشین‌ها موجود است. پس در مورد ماشین‌ها چطور؟ آیا واقعاً وجود دارد؟ نه، یک اصطلاح کلی است. فقط یک اصطلاح کلی که تعریف منطقی گروهی از چیزها است.

بنابراین، ما به گروهی از کلاس‌ها برای داوری چیزی، یک کلاس برتر (superclass) داده‌ایم. درست مثل این است که من به Innova نگاه کنم تا بتوانم بگویم که یک ماشین است. بنابراین، برای هر وسیله نقلیه چهارچرخ از اصطلاح «ماشین» استفاده می‌کنیم.

ما از این اصطلاح تعمیم برای قضاوت در مورد آنها استفاده می‌کنیم. لازم نیست نام تجاری یا نام محصول را کاملاً بدانیم. وقتی چیزی شبیه ماشین است، می‌توانیم بگوییم که آن یک ماشین است. بنابراین، در دنیای واقعی یا در زندگی روزمره، چنین اصطلاحات کلی را تعریف می‌کنیم.

##### **مثالی برای درک تعمیم در سی شارپ**

```csharp
using System;

namespace IsAHasADemo
{
    public class Program
    {
        static void Main()
        {
            Innova innova = new Innova();
            innova.Start();
            innova.Stop();

            BMW bmw = new BMW();
            bmw.Start();
            bmw.Stop();

            Console.ReadKey();
        }
    }
    interface ICar
    {
        void Start();
        void Stop();
    }

    public class Innova : ICar
    {
        public void Start()
        {
            Console.WriteLine($"Innova Start");
        }
        public void Stop()
        {
            Console.WriteLine($"Innova Stop");
        }
    }
    public class BMW : ICar
    {
        public void Start()
        {
            Console.WriteLine($"BMW Start");
        }
        public void Stop()
        {
            Console.WriteLine($"BMW Stop");
        }
    }
}
```

##### **خلاصه‌ای از عمومی‌سازی و تخصصی‌سازی در سی‌شارپ**

بنابراین، نکته‌ای که باید به خاطر داشته باشید این است که هم تعمیم و هم تخصص تنها با استفاده از وراثت حاصل می‌شوند. ما چهار مثال را دیده‌ایم که در آنها دو مورد تخصص و دو مورد عمومیت هستند. در تخصص، کلاس والد وجود داشته و کلاس فرزند بعداً تعریف شده است. در تعمیم، کلاس فرزند وجود داشته و سپس کلاس پایه را تعریف می‌کنیم. بنابراین، تخصص یک رویکرد از بالا به پایین و تعمیم یک رویکرد از پایین به بالا است.

در حالت تخصصی، کلاس پایه چیزی برای ارائه به کلاس فرزند دارد، در حالی که در حالت عمومی، کلاس پایه چیزی برای ارائه به کلاس‌های فرزند خود ندارد. فقط هدف آنها گروه‌بندی آنها با هم است تا بتوانیم به راحتی همه آن چیزها را مدیریت کنیم.

هدف از تعمیم، دستیابی به چندریختی (پلی‌مورفیسم) است و هدف از تخصص، به اشتراک گذاشتن ویژگی‌های آن با کلاس‌های فرزند است. چندریختی چیست و چگونه می‌توان چندریختی را پیاده‌سازی کرد، در مقالات بعدی ما مورد بحث قرار خواهد گرفت.

بنابراین، دو هدف از ارث‌بری وجود دارد. یکی به اشتراک گذاشتن ویژگی‌ها با کلاس‌های فرزند و دوم دستیابی به چندریختی است. بنابراین در اینجا من به شما ایده روشنی از تخصص‌گرایی و تعمیم‌گرایی داده‌ام.
