## **سازنده‌های خصوصی در سی شارپ به همراه مثال**

در این مقاله، قصد دارم **به همراه مثال‌هایی، در مورد سازنده‌های خصوصی در سی‌شارپ** بحث کنم. سردرگمی‌ها و شبهات زیادی در مورد سازنده‌های خصوصی در بین توسعه‌دهندگان وجود دارد و امیدوارم در پایان این مقاله، تمام شبهات و ابهامات شما برطرف شده باشد.

##### **سازنده خصوصی در سی شارپ چیست؟**

در سی شارپ، وقتی سازنده با استفاده از **مشخص کننده دسترسی خصوصی (Private Access** **Specifier** ) ​​ایجاد می‌شود، به آن **سازنده خصوصی (Private Constructor)** می‌گویند. وقتی یک کلاس شامل یک سازنده خصوصی باشد و اگر کلاس **هیچ سازنده عمومی (Public Constructor) دیگری نداشته باشد**، **نمی‌توانید برای کلاسی خارج از کلاس، شیء ایجاد کنید**. اما می‌توانیم **اشیاء را برای کلاس، درون همان کلاس ایجاد کنیم**.

بنابراین، مهمترین نکته‌ای که باید در نظر داشته باشیم این است که محدودیت‌هایی مانند محدودیت‌های ایجاد اشیاء، محدودیت‌های دسترسی به اعضا و غیره، زمانی مطرح می‌شوند که شما سعی در ایجاد اشیاء و دسترسی به اعضا از خارج از کلاس دارید. اگر اشیاء را ایجاد می‌کنید و به اعضا در همان کلاس دسترسی دارید، چنین محدودیت‌هایی مطرح نمی‌شوند.

##### **ایجاد شیء با استفاده از سازنده خصوصی در همان کلاس در C#:**

بسیاری از مقالات در وب می‌گویند که اگر کلاسی دارای سازنده‌ی خصوصی باشد، نمی‌توانید از آن نمونه بسازید. اما این تا حدودی درست است. شما نمی‌توانید از خارج از کلاس نمونه بسازید، اما می‌توانید نمونه را از داخل کلاس بسازید. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید. در اینجا، در کلاس Program، ما یک سازنده‌ی خصوصی تعریف کرده‌ایم و متد Main نیز در همان کلاس Program تعریف شده است. همانطور که می‌بینید، در داخل متد Main، ما در حال ایجاد نمونه‌ای از کلاس Program و فراخوانی Method1 هستیم.

```csharp
using System;

namespace PrivateConstructorDemo
{
    class Program
    {
        //Private Constructor
        private Program()
        {
            Console.WriteLine("This is Private Constructor");
        }
        public void Method1()
        {
            Console.WriteLine("Method1 is Called");
        }

        static void Main(string[] args)
        {
            //Creating instance of Program class using Private Constructor
            Program obj = new Program();
            obj.Method1();
            Console.ReadKey();
        }
    }
}
```

##### **ایجاد نمونه از خارج از کلاس در سی شارپ:**

نکته‌ای که باید به خاطر داشته باشید این است که هنگام ایجاد نمونه از خارج از کلاس، کلاس باید یک سازنده عمومی (public constructor) داشته باشد. فرقی نمی‌کند که یک کلاس سازنده خصوصی (private constructor) داشته باشد یا نه، اما اگر یک کلاس سازنده عمومی (public constructor) داشته باشد، با استفاده از آن سازنده عمومی، می‌توانیم نمونه کلاس را ایجاد کرده و اعضای غیر استاتیک عمومی (public) را فراخوانی کنیم.

برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید. در اینجا، ما کلاس Test را با سازنده‌های Private و Public و یک متد public داریم. اکنون، از متد Main کلاس Program (که خارج از کلاس Test است)، یک نمونه از کلاس Test ایجاد می‌کنیم و Methdo1 را فراخوانی می‌کنیم.

```csharp
using System;

namespace PrivateConstructorDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            //Creating instance of Test class using public Constructor
            Test obj = new Test(10);
            obj.Method1();
            Console.ReadKey();
        }
    }

    public class Test
    {
        //Private Constructor
        private Test()
        {
            Console.WriteLine("This is Private Constructor");
        }

        //Public Constructor
        public Test(int x)
        {
            Console.WriteLine("This is public Constructor");
        }

        public void Method1()
        {
            Console.WriteLine("Method1 is Called");
        }
    }
}
```

##### **محدود کردن وراثت توسط سازنده خصوصی در سی شارپ:**

در بسیاری از وب‌سایت‌ها، متوجه خواهید شد که می‌گویند سازنده خصوصی، وراثت را در سی‌شارپ محدود می‌کند. این بدان معناست که اگر در یک کلاس سازنده خصوصی داشته باشید، آن کلاس نمی‌تواند به ارث برده شود. این نیز تا حدی درست است. بگذارید این نکته را با چند مثال ثابت کنیم.

ببینید، اگر کلاس شما علاوه بر هر سازنده عمومی دیگری، یک سازنده خصوصی هم داشته باشد، می‌توانید کلاس خود را به ارث ببرید. چیزی که کلاس فرزند برای ایجاد رابطه والد-فرزندی نیاز دارد، یک سازنده با دسترسی عمومی است. این بدان معناست که فرقی نمی‌کند کلاس شما سازنده خصوصی داشته باشد یا نه، اگر سازنده عمومی داشته باشد، می‌توانید کلاس را به ارث ببرید.

برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید. در اینجا، درون کلاس والد، دو سازنده داریم، یکی سازنده خصوصی و دیگری سازنده عمومی. سپس کلاس فرزند از کلاس والد ارث‌بری می‌کند و ما هیچ خطایی دریافت نمی‌کنیم.

```csharp
using System;

namespace PrivateConstructorDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            //Creating instance of Child class
            Child obj = new Child();
            Console.ReadKey();
        }
    }

    public class Parent
    {
        //Private Constructor
        private Parent()
        {
            Console.WriteLine("Parent Class Private Constructor is Called");
        }

        //Public Constructor
        public Parent(string Message)
        {
            Console.WriteLine("Parent Class Public Constructor is Called");
        }
    }
    public class Child : Parent
    {
        public Child() : base ("Hello")
        {
            Console.WriteLine("Child Class Public Constructor is Called");
        }
    }
}
```

##### **وقتی کلاس شما فقط اعضای استاتیک در سی شارپ دارد، از سازنده خصوصی استفاده کنید:**

در اینترنت، متوجه خواهید شد که بسیاری از افراد یا مقالات می‌گویند که اگر کلاس شما فقط شامل اعضای استاتیک است، سازنده کلاس را private قرار دهید. اما این اشتباه است. من قبلاً به شما نشان داده‌ام که حتی اگر یک سازنده private داشته باشید، می‌توانیم با کلاس‌های داخلی نیز ارث‌بری کنیم و همچنین می‌توانیم نمونه‌هایی را در همان کلاس ایجاد کنیم.

ببینید، سازنده‌ها چه خصوصی و چه عمومی برای ایجاد نمونه‌ها و مقداردهی اولیه متغیرهای غیر استاتیک یک کلاس استفاده می‌شوند. اگر کلاس شما هیچ عضو غیر استاتیکی ندارد، آیا نیازی به سازنده وجود دارد؟ پاسخ خیر است. اگر کلاس شما فقط شامل اعضای استاتیک است، بهتر است کلاس را استاتیک کنید که نه تنها نمونه کلاس شما را برای ایجاد محدود می‌کند، بلکه ارث‌بری را نیز محدود می‌کند.

برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید. در اینجا، ما یک کلاس استاتیک به نام Test با دو عضو داده استاتیک داریم. همانطور که می‌بینید، کلاس Child هنگام تلاش برای ارث‌بری از کلاس Test، خطاهای زمان کامپایل دریافت می‌کند و همچنین هنگام ایجاد نمونه از کلاس Test استاتیک، خطاهای زمان کامپایل نیز دریافت می‌کند.

```csharp
using System;

namespace PrivateConstructorDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            //Cannot Create an instance of the Static class
            Test test = new Test();
            Console.ReadKey();
        }
    }

    public static class Test
    {
        public static double PI = 3.14;
        public static int GetSquare(int x)
        {
            return x * x;
        }
    }

    //A class cannot Derive from a Static Class
    public class Child: Test
    {
    }
}
```

```csharp
    {
        public static double PI = 3.14;
        public static int GetSquare(int x)
        {
            return x * x;
        }
    }
}
```

##### **آیا سازنده خصوصی در سی شارپ بدون پارامتر خواهد بود؟**

باز هم، در بسیاری از مقالات و ویدیوهای یوتیوب، خواهید دید که بسیاری از مردم می‌گویند سازنده‌های خصوصی باید بدون پارامتر باشند. اما این درست نیست. ببینید، در سی شارپ، سازنده‌ها overload می‌شوند. این بدان معناست که می‌توانیم چندین سازنده با پارامترهای مختلف تعریف کنیم. در overloading آنچه مهم است نام و پارامترها است و مشخص‌کننده‌های دسترسی مانند private، public، protected و غیره را به عنوان بخشی از overloading در نظر نمی‌گیرد. بنابراین، می‌توان چندین سازنده خصوصی را در سی شارپ تعریف کرد.

برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید. در اینجا، در کلاس Program، دو سازنده خصوصی تعریف می‌کنیم. یک سازنده خصوصی بدون پارامتر و یک سازنده خصوصی دیگر با پارامتر است. همانطور که می‌بینید، در اینجا ما دو نمونه را در داخل متد Main با استفاده از هر دو سازنده خصوصی ایجاد می‌کنیم.

```csharp
using System;

namespace PrivateConstructorDemo
{
    class Program
    {
        private Program()
        {
            Console.WriteLine("Private Parameterless Constructor is Called");
        }
        private Program(string message)
        {
            Console.WriteLine("Private Parameterized Constructor is Called");
        }
        static void Main(string[] args)
        {
            Program obj1 = new Program();
            Program obj2 = new Program("Hello");
            Console.ReadKey();
        }
    }
}
```

##### **چه زمانی از سازنده‌های خصوصی در سی شارپ استفاده کنیم؟**

در اینترنت، مقالات زیادی پیدا خواهید کرد و افراد زیادی می‌گویند که از سازنده خصوصی برای پیاده‌سازی الگوی طراحی Singleton استفاده می‌شود. بله، کاملاً درست است. یکی از موارد استفاده از سازنده‌های خصوصی، پیاده‌سازی الگوی طراحی Singleton است. بیایید ببینیم چگونه می‌توان یک الگوی طراحی Singleton را با استفاده از سازنده خصوصی در C# پیاده‌سازی کرد.

##### **الگوی سینگلتون در سی شارپ چیست؟**

الگوی طراحی Singleton تضمین می‌کند که فقط یک نمونه از یک کلاس خاص ایجاد می‌شود و سپس دسترسی سراسری ساده‌ای را برای کل برنامه به آن نمونه فراهم می‌کند.

##### **چگونه الگوی طراحی Singleton را در C# پیاده سازی کنیم؟**

مراحل پیاده‌سازی الگوی طراحی Singleton در زبان برنامه‌نویسی سی‌شارپ به شرح زیر است.

1. شما فقط باید یک سازنده (constructor) در کلاس خود تعریف کنید و آن سازنده باید خصوصی (private) و بدون پارامتر باشد. این امر ضروری است زیرا اجازه نمونه‌سازی از خارج از کلاس را ندارد. فقط از داخل کلاس نمونه‌سازی انجام می‌شود.
2. کلاس باید مهر و موم شده (sealed) اعلام شود که تضمین می‌کند نمی‌توان آن را به ارث برد.
3. شما باید یک متغیر استاتیک خصوصی ایجاد کنید که قرار است ارجاعی به تنها نمونه ایجاد شده از کلاس را در خود نگه دارد.
4. همچنین باید یک ویژگی/متد استاتیک عمومی ایجاد کنید که نمونه‌ی تک‌ساخته‌شده از کلاس سینگلتون را برمی‌گرداند. این روش یا ویژگی ابتدا بررسی می‌کند که آیا نمونه‌ای از کلاس سینگلتون در دسترس است یا خیر. اگر نمونه‌ی سینگلتون در دسترس باشد، آن نمونه‌ی سینگلتون را برمی‌گرداند، در غیر این صورت یک نمونه ایجاد می‌کند و سپس آن نمونه را برمی‌گرداند.

##### **مثال پیاده‌سازی الگوی طراحی Singleton در سی‌شارپ:**

```csharp
using System;

namespace PrivateConstructorDemo
{
    public sealed class Singleton
    {
        private static int counter = 0;
        private static Singleton instance = null;
        private static readonly object Instancelock = new object();
        public static Singleton GetSingletonInstance()
        {
            lock (Instancelock)
            {
                if (instance == null)
                {
                    instance = new Singleton();
                }
                return instance;
            }
        }

        private Singleton()
        {
            counter++;
            Console.WriteLine($"Singleton Constructor Executed {counter} Time");
        }
        public void SomeMethod(string Message)
        {
            Console.WriteLine($"Some Method Called : {Message}");
        }
    }
}
```

سپس، متد Main از کلاس Program را به صورت زیر تغییر دهید:

```csharp
using System;

namespace PrivateConstructorDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Singleton fromPlace1 = Singleton.GetSingletonInstance();
            fromPlace1.SomeMethod("From Place 1");
            Singleton fromPlace2 = Singleton.GetSingletonInstance();
            fromPlace2.SomeMethod("From Place 2");
            Console.ReadKey();
        }
    }
}
```
