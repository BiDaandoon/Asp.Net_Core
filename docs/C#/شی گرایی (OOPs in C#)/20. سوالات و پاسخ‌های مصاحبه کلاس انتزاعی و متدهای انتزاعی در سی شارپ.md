## **سوالات و پاسخ‌های مصاحبه کلاس انتزاعی و متدهای انتزاعی در سی شارپ**

در این مقاله، قصد دارم **به سوالات و پاسخ‌های مصاحبه کلاس انتزاعی و متدهای انتزاعی در سی شارپ** به همراه مثال‌ها بپردازم. در رابطه با کلاس انتزاعی و متدهای انتزاعی، در مصاحبه سوالات زیادی خواهید پرسید. بنابراین، در این جلسه، قصد دارم در مورد سوالات و پاسخ‌های متداول مصاحبه کلاس انتزاعی و متدهای انتزاعی در سی شارپ بحث کنم.

##### **کلاس انتزاعی در سی شارپ چیست؟**

کلاسی که با استفاده از کلمه کلیدی abstract تعریف می‌شود، کلاس abstract نامیده می‌شود. یک کلاس abstract، کلاسی است که تا حدی پیاده‌سازی شده و برای پیاده‌سازی برخی از متدهای یک شیء که برای همه زیرکلاس‌های سطح بعدی مشترک هستند، یعنی همه کلاس‌های فرزند و متدهای انتزاعی باقی‌مانده که توسط کلاس‌های فرزند پیاده‌سازی می‌شوند، استفاده می‌شود. بنابراین، این کلاس شامل متدهای انتزاعی و متدهای عینی (متدهای غیر انتزاعی) شامل متغیرها، ویژگی‌ها و شاخص‌ها است.

![کلاس انتزاعی در سی شارپ چیست؟](https://dotnettutorials.net/wp-content/uploads/2022/09/word-image-30084-1-1.png "کلاس انتزاعی در سی شارپ چیست؟")

یک کلاس انتزاعی را نمی‌توان مستقیماً نمونه‌سازی کرد. ایجاد یک زیرکلاس یا کلاس فرزند از کلاس انتزاعی به منظور فراهم کردن پیاده‌سازی متدهای انتزاعی آن اجباری است.

##### **متد انتزاعی (Abstract Method) در سی شارپ چیست؟**

متدی که بدنه ندارد، متد انتزاعی نامیده می‌شود. این متد با اصلاح‌کننده abstract تعریف می‌شود. این متد فقط شامل یک اعلان/امضا است و شامل پیاده‌سازی یا بدنه یا تعریف متد نیست. یک تابع انتزاعی باید با یک نقطه‌ویرگول خاتمه یابد. بازنویسی یک متد انتزاعی اجباری است.

![متد انتزاعی (Abstract Method) در سی شارپ چیست؟](https://dotnettutorials.net/wp-content/uploads/2022/09/word-image-30084-2.png "متد انتزاعی (Abstract Method) در سی شارپ چیست؟")

##### **چرا باید متد کلمه کلیدی انتزاعی داشته باشد اگر در سی شارپ بدنه ندارد؟**

در یک کلاس، ما فقط مجاز به تعریف یک متد با بدنه (body) هستیم. از آنجایی که ما رفتار پیش‌فرض آن را تغییر می‌دهیم (که به معنای حذف بدنه آن است)، باید کلمه کلیدی abstract را در نمونه اولیه خود داشته باشد.

##### **چه زمانی باید یک کلاس در سی شارپ به صورت انتزاعی تعریف شود؟**

یک کلاس در سی شارپ باید در ۳ حالت زیر به صورت انتزاعی (abstract) تعریف شود.

**مورد اول:** اگر کلاس متدهای انتزاعی داشته باشد، باید کلاس را به صورت انتزاعی تعریف کنیم. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید.

```csharp
public abstract class Example
{
    public abstract void TestMethod();
}
```

**مورد دوم:** اگر کلاس فرزند هیچ یک از متدهای انتزاعی والد را پیاده‌سازی نکند، در این صورت، کلاس فرزند باید به عنوان یک کلاس انتزاعی تعریف شود. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید.

```csharp
public abstract class Parent
{
    public abstract void Method1();
    public abstract void Method2();
}

// Since Child does not implement any abstract method, it must be abstract
public abstract class Child : Parent
{
    // Optionally can implement some, but if not, must be abstract
}
```

**مورد سوم:** اگر کلاس فرزند هیچ یک از متدهای یک رابط را پیاده‌سازی نکند، باید کلاس فرزند به عنوان یک کلاس انتزاعی تعریف شود و همچنین متد آن نیز به صورت انتزاعی تعریف شود. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید.

```csharp
public interface ITest
{
    void TestMethod();
}

// Since Child does not implement the interface method, it must be abstract
public abstract class Child : ITest
{
    public abstract void TestMethod();
}
```

##### **چه زمانی از متد انتزاعی (Abstract Method) در سی شارپ استفاده کنیم؟**

متدهای انتزاعی معمولاً در جایی تعریف می‌شوند که انتظار می‌رود دو یا چند زیرکلاس نقش مشابهی را به شیوه‌ای متفاوت انجام دهند. همچنین می‌توانید همین کار را با استفاده از یک رابط نیز انجام دهید. اما اگر از یک کلاس انتزاعی استفاده کنیم، می‌توانیم برخی عملکردهای مشترک را ارائه دهیم که برای همه کلاس‌های فرزند یکسان خواهد بود و این با استفاده از رابط امکان‌پذیر نیست.

##### **چه نوع عضوی را می‌توانیم در یک کلاس انتزاعی تعریف کنیم؟**

ما می‌توانیم تمام اعضای استاتیک و غیر استاتیک شامل ویژگی‌ها، فیلدها، اندیس‌ها و همچنین متدهای انتزاعی را تعریف کنیم.

##### **آیا اعضای کلاس انتزاعی هنگام ایجاد یک شیء از کلاس فرعی ایجاد می‌شوند؟**

بله **،** اعضای غیر استاتیک آن هنگام ایجاد شیء زیرکلاس عینی آن، حافظه دریافت می‌کنند.

##### **چگونه می‌توانیم اعضای ثابت و غیر ثابت کلاس انتزاعی را اجرا کنیم؟**

اعضای استاتیک را می‌توان مستقیماً با استفاده از نام کلاس اجرا کرد و اعضای غیر استاتیک آن با استفاده از زیرکلاس یا شیء کلاس فرزند آن اجرا می‌شوند. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید.

```csharp
using System;

namespace AbstractClassesAndMethods
{
    class Program
    {
        static void Main(string[] args)
        {
            //Calling Abstract Class Static Method using Class Name
            AbsParent.StaticMethod();

            AbsChild absChild = new AbsChild();
            //Calling the Abstract Class Concrete Method using Child Class Object
            absChild.NonStaticMethod();

            //calling the Abstract Method using Child Class Object
            absChild.AbstractMethod();

            Console.ReadKey();
        }
    }

    public abstract class AbsParent
    {
        //Static Method
        public static void StaticMethod()
        {
            Console.WriteLine("Static Method");
        }

        //Non-Static Method
        public void NonStaticMethod()
        {
            Console.WriteLine("Non-Static Method");
        }

        //Abstract Method
        public abstract void AbstractMethod();
    }

    public class AbsChild : AbsParent
    {
        public override void AbstractMethod()
        {
            Console.WriteLine("AbstractMethod Implemented in Child Clas");
        }
    }
}
```

##### **آیا می‌توانیم یک متد انتزاعی (Abstract Method) را در سی شارپ (C#) به صورت استاتیک (Static) تعریف کنیم؟**

خیر، ما مجاز به اعلام یک متد انتزاعی به عنوان استاتیک نیستیم. این امر منجر به خطای زمان کامپایل می‌شود. اگر کامپایلر به ما اجازه دهد که آن را به عنوان استاتیک اعلام کنیم، می‌توان آن را مستقیماً با استفاده از نام کلاس فراخوانی کرد که نمی‌تواند توسط CLR در زمان اجرا اجرا شود. از این رو، برای محدود کردن فراخوانی متدهای انتزاعی، کامپایلر به ما اجازه نمی‌دهد که یک متد انتزاعی را به عنوان استاتیک اعلام کنیم. برای درک بهتر، لطفاً به تصویر زیر نگاهی بیندازید.

##### **آیا می‌توانیم یک کلاس Concrete را در C# به صورت Abstract تعریف کنیم؟**

بله، مجاز است. ما می‌توانیم یک کلاس انتزاعی را فقط با متدهای غیر انتزاعی تعریف کنیم. تعریف یک کلاس به صورت انتزاعی راهی برای جلوگیری از نمونه‌سازی کلاسی است که قرار است ابتدا از کلاسی دیگر ارث‌بری شود. برای اطمینان از اینکه اعضای غیر استاتیک کلاس ما فقط از طریق اشیاء زیرکلاس قابل دسترسی هستند، باید کلاس عینی را به صورت انتزاعی تعریف کنیم. برای درک بهتر، لطفاً به تصویر زیر نگاهی بیندازید.

```csharp
using System;

namespace AbstractClassesAndMethods
{
    class Program
    {
        static void Main(string[] args)
        {
            Example.Method1();

            //Compile Time Error: Cannot create an instance of the abstract class or interface 'Example'
            Example e = new Example();

            //Not possible
            e.Method2();
            Console.ReadKey();
        }
    }

    public abstract class Example
    {
        public static void Method1()
        {
            Console.WriteLine("Example Method1 method");
        }
        public void Method2()
        {
            Console.WriteLine("Example Method2 method");
        }
    }
}
```

##### **تفاوت بین متدهای انتزاعی و لغو متد در سی شارپ:**

مفهوم متدهای انتزاعی تقریباً شبیه به مفهوم override کردن متد در C# خواهد بود. فرض کنید، من یک کلاس به نام A دارم و در این کلاس، یک متد مجازی به نام Display به صورت زیر داریم.

```csharp
public class A
{
    public virtual void Display()
    {
    }
}
```

در اینجا، متد Display یک متد غیر انتزاعی یعنی متد مجازی است و اکنون، این متد مجازی می‌تواند در کلاس Child بازنویسی شود. بیایید یک کلاس فرزند به نام B ایجاد کنیم که از کلاس والد فوق یعنی A به ارث رسیده است. اکنون، کلاس فرزند می‌تواند متد Display کلاس A را به شرح زیر بازنویسی کند. نحوه بازنویسی متد با استفاده از اصلاحگر override و این، پیاده‌سازی مجدد نامیده می‌شود. این به این دلیل است که متد قبلاً در کلاس A پیاده‌سازی شده است و ما آن را در کلاس B دوباره پیاده‌سازی می‌کنیم.

```csharp
public class B : A
{
    public override void Display() //Optional
    {
        //Re-Implementation
    }
}
```

**نکته:** پیاده‌سازی مجدد یا بازنویسی متد مجازی کلاس والد در کلاس فرزند اختیاری است.

حال، بیایید همان مثال را با استفاده از کلاس انتزاعی و متد انتزاعی بررسی کنیم. کلاس A را به صورت زیر تغییر می‌دهیم. در اینجا، ما کلاس را به عنوان انتزاعی و متد را با حذف کلمه کلیدی virtual و همچنین بدنه متد، انتزاعی تعریف می‌کنیم.

```csharp
public abstract class A
{
    public abstract void Display();
}
```

حال، کلاس B از کلاس A ارث‌بری می‌کند و ما قبلاً در مورد اینکه کلاس فرزند باید پیاده‌سازی متدهای انتزاعی کلاس والد خود را فراهم کند، بحث کردیم. اینکه چگونه می‌توانیم پیاده‌سازی متد انتزاعی را در کلاس فرزند فراهم کنیم، با استفاده از همان اصلاح‌کننده override است، اما در اینجا override کردن متد اجباری است، نه اختیاری، و نکته دوم این است که این کار پیاده‌سازی مجدد نیست، بلکه به صورت زیر پیاده‌سازی می‌شود.

```csharp
public class B : A
{
    public override void Display() //Mandatory
    {
        //Implementation
    }
}
```

**نکته:** پیاده‌سازی در کلاس فرزند اجباری است زیرا کلاس والد پیاده‌سازی ارائه نمی‌دهد، بلکه در کلاس والد انتزاعی است.

بنابراین، مفهوم متدهای انتزاعی تقریباً شبیه به مفهوم override کردن متد است. در override کردن متد، اگر کلاس والد شامل متدهایی باشد که به صورت مجازی اعلان شده‌اند، می‌توان آن متدها را با استفاده از اصلاح‌کننده override تحت کلاس فرزند دوباره پیاده‌سازی کرد و پیاده‌سازی مجدد یا override کردن متد اختیاری است. از سوی دیگر، در مورد abstract، اگر کلاس والد شامل متدهای انتزاعی باشد، آن متدها باید در کلاس فرزند با استفاده از همان اصلاح‌کننده override پیاده‌سازی شوند و این کار اجباری است.
