## **تفاوت بین متغیر، مرجع و نمونه‌ای از یک کلاس در سی شارپ**

در این مقاله، قصد دارم تفاوت‌های بین **متغیرها، ارجاع‌ها و نمونه‌های یک کلاس در سی‌شارپ را** با مثال‌هایی مورد بحث قرار دهم. به عنوان یک برنامه‌نویس، درک متغیرها، ارجاع‌ها و نمونه‌های یک کلاس در سی‌شارپ برای ما بسیار مهم است.

##### **تفاوت بین متغیر، مرجع و نمونه‌ای از یک کلاس در سی شارپ**

1. **متغیر یک کلاس:** یک کپی از کلاس که مقداردهی اولیه نشده است.
2. **نمونه‌ای از یک کلاس:** یک کپی از کلاس که با استفاده از کلمه کلیدی new مقداردهی اولیه می‌شود و حافظه مخصوص به خود را دارد و هرگز با نمونه دیگری به اشتراک گذاشته نمی‌شود.
3. **ارجاع به یک کلاس:** یک کپی از کلاس که با استفاده از یک نمونه موجود مقداردهی اولیه می‌شود و ارجاع‌های یک کلاس هیچ تخصیص حافظه‌ای نخواهند داشت، آنها حافظه همان نمونه‌ای را که برای مقداردهی اولیه متغیر اختصاص داده شده است، به اشتراک می‌گذارند.

بیایید تعاریف فوق را با مثال‌هایی به طور مفصل درک کنیم.

##### **کلاس چیست؟**

به طور خلاصه، یک کلاس در سی شارپ یک نوع داده تعریف شده توسط کاربر است. ما می‌گوییم رشته یک نوع داده است. اما این نوع داده رشته‌ای به عنوان یک کلاس در سی شارپ ایجاد می‌شود. این یک کلاس از پیش تعریف شده است و همه کلاس‌های از پیش تعریف شده یا کلاس‌های تعریف شده توسط کاربر در سی شارپ نیز نوع داده نامیده می‌شوند.

نکته دومی که باید بدانید این است که ما هرگز نمی‌توانیم مستقیماً از یک کلاس استفاده کنیم. نحوه استفاده از یک کلاس به این معنی است که باید یک کپی از آن کلاس ایجاد کنیم. چرا باید یک کپی از کلاس ایجاد کنیم؟ بگذارید این را بفهمیم. می‌دانیم که int یک نوع داده در C# است. آیا می‌توانیم از نوع داده int به صورت زیر استفاده کنیم؟  
**عدد صحیح = ۱۰؛**

نه. این امکان ندارد. ما هرگز نمی‌توانیم از نوع داده int مانند موارد فوق استفاده کنیم. زیرا int فقط یک طرح برای داده‌های خاص شماست و تخصیص حافظه برای داده‌های شما ندارد. اگر می‌خواهید برای داده‌های خود تخصیص حافظه داشته باشید، باید به صورت زیر عمل کنید:  
عدد صحیح I = 10;

**منظور از «I» در اینجا چیست؟** در اینجا، «I» یک کپی از نوع داده int است. به همین ترتیب، عبارت زیر نیز نامعتبر است.  
**رشته = "سلام";**  
در اینجا، رشته یک کلاس است و کلاس یک نوع داده تعریف شده توسط کاربر است. شما نمی‌توانید مستقیماً از آن استفاده کنید. اگر می‌خواهید از رشته استفاده کنید، باید به صورت زیر از آن استفاده کنید.  
**رشته s = "سلام";**

**اینجا منظور از s چیست؟** در اینجا، s یک کپی از نوع داده رشته‌ای است.  
ببینید، برای ساختن خانه به نقشه نیاز داریم. آیا می‌توانیم بدون نقشه خانه بسازیم؟ نه. امکان‌پذیر نیست. بنابراین، هر خانه‌ای به نقشه نیاز دارد. همین موضوع اینجا هم صدق می‌کند. در اینجا، int و string چیزی جز نقشه خانه شما نیستند و i و s خانه‌های ساخته شده شما هستند. برای ساختن خانه، با یک معمار مشورت خواهید کرد. معمار نقشه را به شما می‌دهد؟ آیا می‌توانید در نقشه بمانید؟ نه، نمی‌توانید در نقشه زندگی کنید. بنابراین، اگر واقعاً می‌خواهید زندگی کنید، به خانه‌ای نیاز دارید که بر اساس نقشه ساخته شده باشد.

دقیقاً همین اتفاق اینجا هم می‌افتد. در اینجا، int نقشه و i خانه ساخته شده بر اساس نقشه است. بنابراین، اکنون، مقدار 10 چیزی جز شما نیست، شما در یک خانه ساخته شده زندگی می‌کنید که در مثال ما i است و خانه بر اساس نقشه یعنی int ساخته شده است. بنابراین، نکته‌ای که باید به خاطر داشته باشید این است که نوع داده همیشه نقشه خواهد بود و یک کپی از آن نوع داده، پیاده‌سازی آن نقشه است.

**نکته:** هر کلاسی، چه از پیش تعریف شده باشد و چه توسط کاربر تعریف شده باشد، یک نوع داده خواهد بود. برای استفاده از کلاس، کاری که باید انجام دهیم این است که یک کپی از آن کلاس ایجاد کنیم. تا زمانی که یک کپی از آن کلاس ایجاد نکنیم، حافظه اختصاص داده نخواهد شد. اگر می‌خواهید حافظه اختصاص داده شود، باید یک کپی از آن نوع ایجاد کنید.

##### **مثالی برای درک نحوه مصرف یک نوع در سی شارپ:**

لطفا به مثال زیر نگاهی بیندازید. در اینجا، من یک متغیر عدد صحیح به نام x با مقدار 10 ایجاد کرده‌ام و سپس درون متد Main مقدار x را در پنجره کنسول چاپ می‌کنم.

```csharp
using System;

namespace VariablesDemo
{
    internal class Example
    {
        int x = 10;
        static void Main(string[] args)
        {
            Console.WriteLine(x);
            Console.ReadKey();
        }
    }
}
```

حال، اگر سعی کنید کد بالا را اجرا کنید، با خطای زیر مواجه خواهید شد. در اینجا، وقتی سعی می‌کنیم مقدار x را چاپ کنیم، با خطا مواجه می‌شویم. دلیل این امر این است که x یک عضو نمونه یا به عبارت دیگر عضو غیر استاتیک است و شما نمی‌توانید مستقیماً از یک بلوک استاتیک به هیچ عضو غیر استاتیکی دسترسی پیدا کنید، و در اینجا متد Main یک بلوک استاتیک است.

![مثالی برای درک نحوه مصرف یک نوع در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/07/word-image-28868-1.png "مثالی برای درک نحوه مصرف یک نوع در سی شارپ")

حال، متد Main یک بلوک استاتیک است و x یک عضو نمونه است، بنابراین نمی‌توانید مستقیماً به x دسترسی داشته باشید. اگر می‌خواهید به x دسترسی داشته باشید، ابتدا باید یک نمونه از کلاس Example یا یک کپی از کلاس Example ایجاد کنید و فقط با استفاده از نمونه کلاس Example می‌توانید به مقدار x دسترسی داشته باشید، همانطور که در مثال زیر نشان داده شده است.

```csharp
using System;

namespace VariablesDemo
{
    internal class Example
    {
        int x = 10;
        static void Main(string[] args)
        {
            Example e = new Example();
            Console.WriteLine(e.x);
            Console.ReadKey();
        }
    }
}
```

بنابراین، در اینجا e یک نمونه از کلاس Example است، یا می‌توان گفت e یک کپی از کلاس Example است. و اکنون، حافظه برای آن نمونه e اختصاص داده شده است. یک نمونه از یک کلاس فقط با استفاده از کلمه کلیدی new ایجاد می‌شود.

بنابراین، نکته‌ای که باید به خاطر داشته باشید این است که ما فقط می‌توانیم مقدار x را با استفاده از نمونه کلاس یا کپی کلاس چاپ کنیم و نمونه فقط با استفاده از کلمه کلیدی new در سی شارپ ایجاد می‌شود.

##### **متغیرهای یک کلاس در سی شارپ:**

حال، بیایید بفهمیم متغیر یک کلاس در سی شارپ چیست. لطفاً به کد زیر توجه کنید.

**مثال ه؛**

در اینجا، ما e را مقداردهی اولیه نمی‌کنیم. بنابراین، متغیر یک کلاس چیست؟ متغیر یک کلاس به معنای یک کپی از کلاس است که مقداردهی اولیه نشده است. همچنین یک کپی بدون مقداردهی اولیه از یک کلاس در نظر گرفته می‌شود. برای درک بهتر، لطفاً به کد زیر نگاهی بیندازید.

```csharp
using System;

namespace VariablesDemo
{
    internal class Example
    {
        int x = 10;
        static void Main(string[] args)
        {
            //Variable
            //Uninitialized copy of class Example
            Example e;
            Console.WriteLine(e.x);
            Console.ReadKey();
        }
    }
}
```

حال، اگر سعی کنید کد بالا را اجرا کنید، خطای زمان کامپایل زیر را دریافت خواهید کرد، یعنی **استفاده از متغیر محلی بدون مقداردهی اولیه 'e'** . در اینجا، e یک نمونه نیست، همانطور که در پیام خطا می‌بینید، می‌گوید متغیر محلی e. بنابراین، در اینجا e یک متغیر محلی است. و شما سعی دارید با استفاده از متغیری که در زبان C# امکان‌پذیر نیست، به عضوی به نام x دسترسی پیدا کنید و از این رو خطای زیر را دریافت می‌کنید.

![متغیرهای یک کلاس در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/07/word-image-28868-2.png "متغیرهای یک کلاس در سی شارپ")

بنابراین، نکته‌ای که باید به خاطر داشته باشید این است که می‌توانید به اعضای غیر استاتیک یک کلاس فقط با استفاده از نمونه دسترسی پیدا کنید، نه با استفاده از متغیرها. یک نمونه از یک کلاس فقط با استفاده از کلمه کلیدی new ایجاد می‌شود. و اگر از کلمه کلیدی new استفاده نشود، به این معنی است که آن یک نمونه نیست، بلکه فقط یک متغیر است. حال، کد زیر را مشاهده کنید.

**مثال e؛** در اینجا e یک متغیر از کلاس است. مثال

**e = new Example();** حال، در اینجا، e یک نمونه از کلاس Example است. به محض اینکه از کلمه کلیدی new استفاده کنید، e به یک نمونه تبدیل می‌شود و اکنون، می‌توانید با استفاده از نمونه e، همانطور که در مثال زیر نشان داده شده است، به متغیر x دسترسی پیدا کنید.

```csharp
using System;

namespace VariablesDemo
{
    internal class Example
    {
        int x = 10;
        static void Main(string[] args)
        {
            Example e; //e is Variable of class Example
            e = new Example(); //e is instance of class Example
            Console.WriteLine(e.x);
            Console.ReadKey();
        }
    }
}
```

بنابراین، می‌توانیم با استفاده از دو روش زیر یک نمونه در سی‌شارپ ایجاد کنیم و هر دو یکسان خواهند بود. در حالت اول، اعلان و مقداردهی اولیه را در یک خط انجام داده‌ایم. و در حالت دوم، اعلان و مقداردهی اولیه را در دو خط جدا کرده‌ایم. ابتدا متغیر را اعلان می‌کنیم و سپس شیء را مقداردهی اولیه می‌کنیم. وقتی متغیر را مقداردهی اولیه می‌کنیم، متغیر به یک نمونه از کلاس تبدیل می‌شود و با استفاده از نمونه می‌توانیم فقط به اعضای غیر استاتیک کلاس دسترسی داشته باشیم.

##### **تفاوت بین متغیر و نمونه‌ای از یک کلاس در سی شارپ:**

متغیرهای یک کلاس چیزی جز کپی مقداردهی نشده از یک کلاس نیستند. آنها هیچ حافظه‌ای را اشغال نمی‌کنند. آنها مقدار null را نگه می‌دارند. و با استفاده از متغیرها، نمی‌توانیم عضو یک کلاس را مصرف کنیم. وقتی یک متغیر را با کلمه کلیدی new مقداردهی اولیه می‌کنیم، آن تبدیل به یک نمونه می‌شود. یک نمونه چیزی جز یک کپی از یک کلاس نیست. حافظه را اشغال می‌کند و با استفاده از نمونه‌های یک کلاس، می‌توانیم به اعضای غیر استاتیک آن کلاس دسترسی پیدا کنیم. برای درک بهتر تفاوت بین متغیر و نمونه در C#، لطفاً به نمودار زیر نگاهی بیندازید.

![تفاوت بین متغیر و نمونه‌ای از یک کلاس در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/07/word-image-28868-4.png "تفاوت بین متغیر و نمونه‌ای از یک کلاس در سی شارپ")

**نکته:** نکته‌ای که باید به خاطر داشته باشید این است که تا زمانی که متغیر را مقداردهی اولیه نکنید، تخصیص حافظه انجام نخواهد شد. به محض اینکه متغیر را با استفاده از کلمه کلیدی new مقداردهی اولیه کنید، تخصیص حافظه انجام می‌شود و فقط شما می‌توانید به اعضای غیر استاتیک کلاس دسترسی داشته باشید. و نمونه‌های (instance) فقط با استفاده از کلمه کلیدی new ایجاد می‌شوند.

وقتی ما یک متغیر را با استفاده از کلمه کلیدی new مقداردهی اولیه می‌کنیم، آن متغیر به یک نمونه تبدیل می‌شود. و می‌توانید متغیر را در همان خط (با استفاده از یک دستور واحد) تعریف و مقداردهی اولیه کنید، یا می‌توانید همانطور که در تصویر زیر نشان داده شده است، تعریف و مقداردهی اولیه را به دو دستور تقسیم کنید.

##### **ارجاع به یک کلاس در سی شارپ:**

حال، بیایید بفهمیم که ارجاع برای یک کلاس در سی شارپ چیست. لطفاً به کد زیر توجه کنید. در اینجا، من دو نمونه از کلاس Example یعنی e1 و e2 ایجاد می‌کنم. بنابراین، در اینجا، دو بار حافظه اختصاص داده خواهد شد. یکی برای e1 زیرا از کلمه کلیدی new استفاده می‌کنیم و دیگری برای e2 زیرا دوباره از کلمه کلیدی new برای مقداردهی اولیه نمونه‌ها استفاده می‌کنیم. بنابراین، در مثال‌های زیر، حافظه دو بار اختصاص داده می‌شود. و سپس مقدار x را با استفاده از هر دو نمونه چاپ می‌کنیم.

```csharp
using System;

namespace VariablesDemo
{
    internal class Example
    {
        int x = 10;
        static void Main(string[] args)
        {
            Example e1 = new Example(); //e1 is Instance of class Example
            Example e2 = new Example(); //e2 is Instance of class Example

            Console.WriteLine($"e1.x: {e1.x} and e2.x: {e2.x}");
            Console.ReadKey();
        }
    }
}
```

حالا، وقتی کد بالا را اجرا کنید، خروجی زیر را دریافت خواهید کرد.

![ارجاع به یک کلاس در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/07/word-image-28868-6.png "ارجاع به یک کلاس در سی شارپ")

در اینجا، ما دو کپی از متغیر x در حافظه داریم. بیایید این را ثابت کنیم. بیایید متغیر x را به ۵۰ از نمونه e1 تغییر دهیم و سپس دوباره مقدار x را با استفاده از هر دو نمونه، همانطور که در مثال زیر نشان داده شده است، چاپ کنیم.

```csharp
using System;

namespace VariablesDemo
{
    internal class Example
    {
        int x = 10;
        static void Main(string[] args)
        {
            Example e1 = new Example(); //e1 is Instance of class Example
            Example e2 = new Example(); //e2 is Instance of class Example

            Console.WriteLine($"e1.x: {e1.x} and e2.x: {e2.x}");

            e1.x = 50; //Modifying the x variable of e1 instance
            Console.WriteLine($"e1.x: {e1.x} and e2.x {e2.x}");

            Console.ReadKey();
        }
    }
}
```

همانطور که در خروجی بالا مشاهده می‌کنید، متغیر x مربوط به نمونه e1 فقط به 50 تغییر یافته است. این تغییر، متغیر x مربوط به نمونه e2 را تغییر نخواهد داد. حال، بیایید متغیر x مربوط به نمونه e2 را تغییر دهیم و سپس مقدار x هر دو نمونه را همانطور که در مثال زیر نشان داده شده است، دوباره چاپ کنیم.

```csharp
using System;

namespace VariablesDemo
{
    internal class Example
    {
        int x = 10;
        static void Main(string[] args)
        {
            Example e1 = new Example(); //e1 is Instance of class Example
            Example e2 = new Example(); //e2 is Instance of class Example

            Console.WriteLine($"e1.x: {e1.x} and e2.x: {e2.x}");

            e1.x = 50; //Modifying the x variable of e1 instance
            Console.WriteLine($"e1.x: {e1.x} and e2.x {e2.x}");

            e2.x = 150; //Modifying the x variable of e2 instance
            Console.WriteLine($"e1.x: {e1.x} and e2.x {e2.x}");

            Console.ReadKey();
        }
    }
}
```

**نکته:** نکته‌ای که باید بدانید این است که می‌توانید هر تعداد نمونه برای کلاس خود ایجاد کنید و برای هر نمونه، تخصیص حافظه جداگانه‌ای انجام خواهد شد. تخصیص حافظه انجام شده برای یک نمونه هرگز با نمونه‌های دیگر به اشتراک گذاشته نخواهد شد.

این زیبایی برنامه‌نویسی شی‌گرا است یا می‌توان گفت امنیت برنامه‌نویسی شی‌گرا است. هر نمونه منحصر به فرد است. تغییراتی که در یک نمونه ایجاد می‌شود هرگز در نمونه‌های دیگر منعکس نمی‌شود.

این‌ها همه در مورد نمونه‌هایی از یک کلاس بود. حالا، بیایید در مورد ارجاعات یک کلاس در سی‌شارپ صحبت کنیم. لطفاً به کد زیر نگاهی بیندازید. در اینجا، e1 یک نمونه از یک کلاس است اما e2 یک ارجاع از یک کلاس است. ببینید، e2 با کلمه کلیدی new مقداردهی اولیه نشده است و از این رو یک نمونه نیست. اما e2 با یک نمونه موجود مقداردهی اولیه شده است.

![ارجاعات یک کلاس در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2022/07/word-image-28868-10.png "ارجاعات یک کلاس در سی شارپ")

بنابراین، نکته‌ای که باید به خاطر داشته باشید این است که هر زمان یک کپی از کلاس با یک نمونه موجود مقداردهی اولیه شود، به آن ارجاع به یک کلاس گفته می‌شود. در اینجا، e2 با نمونه موجود e1 مقداردهی اولیه می‌شود و از این رو e2 یک ارجاع به کلاس خواهد بود. به عبارت ساده، e2 یک اشاره‌گر به e1 است. بنابراین، در اینجا e2 تخصیص حافظه جداگانه‌ای ندارد، بلکه به همان تخصیص حافظه‌ای که توسط نمونه e1 انجام می‌شود، اشاره خواهد کرد. این بدان معناست که هم e1 و هم e2 اکنون یک مکان حافظه را مصرف می‌کنند. بیایید این را ثابت کنیم.

در مثال زیر، e1 یک نمونه و e2 یک مرجع است که به نمونه e1 اشاره می‌کند. سپس ما با استفاده از نمونه e1 مقدار x را به 50 تغییر می‌دهیم و اگر هر دو به یک مکان حافظه اشاره کنند، تغییرات باید روی e2 نیز تأثیر بگذارند.

```csharp
using System;

namespace VariablesDemo
{
    internal class Example
    {
        int x = 10;
        static void Main(string[] args)
        {
            Example e1 = new Example(); //e1 is Instance of class Example
            Example e2 = e1; //e2 is Reference of class Example

            Console.WriteLine($"e1.x: {e1.x} and e2.x: {e2.x}");

            e1.x = 50; //Modifying the x variable of e1 instance
            Console.WriteLine($"e1.x: {e1.x} and e2.x {e2.x}");

            Console.ReadKey();
        }
    }
}
```

به عبارت ساده، ارجاع یک کلاس را می‌توان اشاره‌گری به نمونه نامید و هر تغییری که ما روی اعضا با استفاده از نمونه انجام می‌دهیم، زمانی که ما با استفاده از ارجاع‌ها به آن اعضا دسترسی پیدا می‌کنیم، منعکس خواهد شد و برعکس نیز صادق است.

**نکته:** با استفاده از متغیرها، نمی‌توانید به اعضای غیر استاتیک یک کلاس دسترسی پیدا کنید، اما با استفاده از هر دو نمونه یا ارجاع، می‌توانید به اعضای غیر استاتیک یک کلاس دسترسی پیدا کنید. دلیل این امر این است که ارجاع فقط با یک نمونه موجود مقداردهی اولیه می‌شود و متغیرها هرگز مقداردهی اولیه نمی‌شوند.

**نکته:** ارجاع‌ها حافظه‌ی اختصاصی خود را ندارند، بلکه به یک حافظه‌ی موجود اشاره می‌کنند. اما ارجاع‌ها مانند نمونه‌ها استفاده می‌شوند.
