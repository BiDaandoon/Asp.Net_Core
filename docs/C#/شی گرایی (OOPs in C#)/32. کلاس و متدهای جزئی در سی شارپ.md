## **کلاس و متدهای جزئی در سی شارپ به همراه مثال**

در این مقاله، قصد دارم در مورد **کلاس‌های جزئی و متدهای جزئی در سی‌شارپ** به همراه مثال صحبت کنم . در پایان این مقاله، شما خواهید فهمید که کلاس‌های جزئی و متدهای جزئی چیستند و چه زمانی و چگونه می‌توان از کلاس‌های جزئی و متدهای جزئی در سی‌شارپ به همراه مثال استفاده کرد.

##### **کلاس Partial در سی شارپ چیست؟**

کلاس جزئی (Partial Class) ویژگی جدیدی است که در C# 2.0 اضافه شده است و به ما امکان می‌دهد یک کلاس را در چندین فایل تعریف کنیم، یعنی می‌توانیم محتوای کلاس را به صورت فیزیکی به فایل‌های مختلف تقسیم کنیم، اما حتی از نظر فیزیکی هم آنها تقسیم شده‌اند، اما از نظر منطقی فقط یک واحد واحد هستند. کلاسی که کد آن را می‌توان در دو یا چند فایل نوشت، به عنوان کلاس جزئی (Partial Class) شناخته می‌شود. برای جزئی کردن هر کلاس، باید از کلمه کلیدی جزئی (Partial) استفاده کنیم.

طبق گفته مایکروسافت، می‌توان تعریف یک کلاس را بین دو یا چند فایل منبع تقسیم کرد. همچنین می‌توان تعریف یک ساختار یا یک رابط را بین دو یا چند فایل منبع تقسیم کرد که در مورد ویژگی‌های جدید C# 8 در مورد آن صحبت خواهیم کرد. هر فایل منبع شامل بخشی از نوع تعریف کلاس است و تمام قسمت‌ها هنگام کامپایل برنامه با هم ترکیب می‌شوند.

##### **درک کلاس Partial در سی شارپ با یک مثال:**

حال، بیایید با یک مثال، کلاس Partial را درک کنیم. ابتدا یک برنامه کنسول ایجاد کنید و سپس یک فایل کلاس با نام Employee.cs اضافه کنید. پس از اضافه کردن فایل کلاس، کد زیر را در آن کپی و جایگذاری کنید. در اینجا، لطفاً توجه داشته باشید که نام کلاس Employee است و در این کلاس، 4 ویژگی خودکار پیاده‌سازی شده یعنی FirstName، LastName، Gender و Salary را تعریف کرده‌ایم. در اینجا، همچنین 2 متد عمومی یعنی DisplayFullName و DisplayEmployeeDetails را تعریف کرده‌ایم.

```csharp
using System;

namespace PartialClassDemo
{
    public class Employee
    {
        public string FirstName { get; set; }
        public string LastName { get; set; }
        public string Gender { get; set; }
        public double Salary { get; set; }
        
        public void DisplayFullName()
        {
            Console.WriteLine($"Full Name is : {FirstName} {LastName}");
        }

        public void DisplayEmployeeDetails()
        {
            Console.WriteLine("Employee Details : ");
            Console.WriteLine($"First Name : {FirstName}");
            Console.WriteLine($"Last Name : {LastName}");
            Console.WriteLine($"Gender : {Gender}");
            Console.WriteLine($"Salary : {Salary}");
        }
    }
}
```

این یک کلاس Employee بسیار ساده است که دارای ۴ ویژگی عمومی خودکار پیاده‌سازی شده و ۲ متد عمومی است. حال، بیایید از کلاس فوق استفاده کنیم. لطفاً کلاس Program را که شامل متد Main است، مطابق شکل زیر تغییر دهید. در اینجا، ما یک نمونه از کلاس Employee ایجاد می‌کنیم و مقادیری را برای چهار ویژگی تنظیم می‌کنیم و سپس متدهای DisplayFullName و DisplayEmployeeDetails را فراخوانی می‌کنیم که قرار است نام کامل کارمند و اطلاعات کارمند را در پنجره کنسول نمایش دهند.

```csharp
using System;

namespace PartialClassDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Employee emp = new Employee
            {
                FirstName = "Pranaya",
                LastName = "Rout",
                Salary = 100000,
                Gender = "Male"
            };
            emp.DisplayFullName();
            emp.DisplayEmployeeDetails();
            Console.ReadKey();
        }
    }
}
```

##### **تقسیم تعریف کلاس فوق به دو فایل با استفاده از کلاس‌های جزئی در سی شارپ**

حالا کاری که قرار است انجام دهیم این است که تعریف کلاس Employee فوق را به دو فایل کلاس مختلف تقسیم می‌کنیم. یک فایل کلاس شامل هر ۴ ویژگی عمومیِ پیاده‌سازی خودکار (FirstName، LastName، Gender و Salary) خواهد بود و فایل کلاس دیگر شامل دو متد عمومیِ DisplayFullName و DisplayEmployeeDetails خواهد بود که درون کلاس Employee تعریف کرده‌ایم.

ابتدا، فایل کلاس Employee.cs را از پروژه حذف کنید. سپس باید دو فایل کلاس با نام‌های PartialEmployeeOne و PartialEmployeeTwo اضافه کنیم. فایل کلاس PartialEmployeeOne شامل هر ۴ ویژگی عمومی پیاده‌سازی خودکار (FirstName، LastName، Gender و Salary) و فایل کلاس PartialEmployeeTwo شامل دو متد عمومی DisplayFullName و DisplayEmployeeDetails خواهد بود. اگرچه نام فایل‌های کلاس متفاوت است، اما نام کلاس یکسان خواهد بود و در این حالت، ما نام کلاس را PartialEmployee در هر دو فایل کلاس ارائه می‌دهیم و همچنین با استفاده از کلمه کلیدی partial، کلاس را به صورت partial می‌سازیم.

###### **افزودن**   **PartialEmployeeOne.cs**

برای اضافه کردن فایل کلاس PartialEmployeeOne.cs، روی پروژه کلیک راست کرده و سپس یک فایل کلاس با نام PartialEmployeeOne.cs اضافه کنید و سپس کد زیر را کپی و در آن قرار دهید. توجه داشته باشید که در کد زیر، کلاس PartialEmployee با کلمه کلیدی partial مشخص شده است و فقط شامل 4 ویژگی عمومی مطابق با نیازهای ما است.

```csharp
namespace PartialClassDemo
{
    public partial class PartialEmployee
    {
        public string FirstName { get; set; }
        public string LastName { get; set; }
        public string Gender { get; set; }
        public double Salary { get; set; }
    }
}
```

**نکته:** در اینجا نام فایل کلاس PartialEmployeeOne.cs است، اما نام کلاس PartialEmployee است و کلاس با کلمه کلیدی partial علامت‌گذاری شده است که آن را به یک کلاس partial تبدیل می‌کند.

###### **افزودن**   **PartialEmployeeTwo.cs**

حالا بیایید فایل کلاس PartialEmployeeTwo.cs را به پروژه خود اضافه کنیم. برای انجام این کار، روی پروژه کلیک راست کرده و یک فایل کلاس با نام PartialEmployeeTwo.cs اضافه کنید و سپس کد زیر را کپی و در آن قرار دهید. توجه داشته باشید که در کد زیر، کلاس PartialEmployee نیز با کلمه کلیدی partial مشخص شده است و فقط شامل دو متد عمومی مطابق با نیازهای ما است. اما نکته‌ای که باید رعایت کنید این است که در اینجا، ما می‌توانیم به ویژگی‌هایی که در فایل کلاس PartialEmployeeOne.cs تعریف شده‌اند دسترسی داشته باشیم و این به دلیل کلاس partial امکان‌پذیر است.

```csharp
using System;

namespace PartialClassDemo

    {

        {
            Console.WriteLine($"Full Name is : {FirstName} {LastName}");

        {
            Console.WriteLine("Employee Details : ");
            Console.WriteLine($"First Name : {FirstName}");
            Console.WriteLine($"Last Name : {LastName}");
            Console.WriteLine($"Gender : {Gender}");
            Console.WriteLine($"Salary : {Salary}");
        }
    }

```

در اینجا نام فایل کلاس PartialEmployeeTwo.cs است، اما نام کلاس PartialEmployee است و ما این کلاس را با استفاده از کلمه کلیدی partial به صورت partial علامت گذاری کرده‌ایم. اگرچه تعریف کلاس PartialEmployee به چندین فایل کلاس تقسیم شده است، اما پس از کامپایل، آنها با هم گروه بندی شده و به یک کلاس واحد تبدیل می‌شوند.

##### **استفاده از کلاس Partial در سی شارپ:**

حال، ما قصد داریم از کلاس‌های Partial که تعریف آنها در دو فایل کلاس مختلف تقسیم شده است، استفاده کنیم. ما باید از کلاس Partial درست مانند یک کلاس معمولی استفاده کنیم. می‌توانیم یک نمونه ایجاد کنیم و همچنین می‌توانیم اعضا را با استفاده از آن نمونه فراخوانی کنیم. بنابراین، متد Main کلاس Program را مطابق شکل زیر تغییر دهید تا از کلاس PartialEmployee استفاده کند. در اینجا، می‌توانید ببینید که ما در حال ایجاد یک نمونه از کلاس PartialEmployee هستیم، 4 مقدار ویژگی را تنظیم می‌کنیم و سپس دو متد عمومی را فراخوانی می‌کنیم.

```csharp
using System;

namespace PartialClassDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            PartialEmployee emp = new PartialEmployee()
            {
                FirstName = "Pranaya",
                LastName = "Rout",
                Salary = 100000,
                Gender = "Male"
            };
            emp.DisplayFullName();
            emp.DisplayEmployeeDetails();
            Console.ReadKey();
        }
    }
}
```

##### **چه زمانی نیاز به استفاده از کلاس Partial در سی شارپ داریم؟**

چندین موقعیت وجود دارد که در آنها تقسیم تعریف کلاس مطلوب است

1. هنگام کار روی پروژه‌های بزرگ، تقسیم یک کلاس به فایل‌های جداگانه به چندین برنامه‌نویس اجازه می‌دهد تا به‌طور همزمان روی آن کار کنند.
2. هنگام کار با کد منبع تولید شده به صورت خودکار، کد می‌تواند بدون نیاز به ایجاد مجدد فایل منبع به کلاس اضافه شود. ویژوال استودیو از این رویکرد هنگام ایجاد فرم ویندوز، کد بسته‌بندی سرویس وب و غیره استفاده می‌کند.

در پروژه من، من از رویکرد Entity Framework Database First استفاده می‌کنم. در این حالت، Entity Framework مدل‌ها، یعنی کلاس‌ها را بر اساس پایگاه داده ایجاد می‌کند و کلاس‌ها را به صورت کلاس‌های جزئی ایجاد می‌کند. در مرحله بعد، می‌خواهم تغییراتی را با کلاس‌های جزئی تولید شده خودکار مانند اضافه کردن برخی از ویژگی‌های اضافی یا اضافه کردن برخی از ویژگی‌ها انجام دهم. اما اگر این تغییرات را با کلاس‌های جزئی تولید شده خودکار انجام دهم، هنگام به‌روزرسانی فایل EDMX، تغییرات من از بین می‌روند. بنابراین، کاری که من معمولاً انجام می‌دهم این است که یک کلاس جزئی ایجاد می‌کنم و در آن کلاس جزئی، تمام سفارشی‌سازی‌ها را انجام می‌دهم.

##### **قوانینی که باید هنگام کار با کلاس‌های Partial در سی شارپ رعایت کنید:**

**قانون ۱:** تمام بخش‌هایی که شامل تعریف کلاس هستند باید از کلمه کلیدی partial استفاده کنند. تمام بخش‌ها باید در زمان کامپایل در دسترس باشند تا نوع نهایی را تشکیل دهند. در غیر این صورت، با خطای کامپایل مواجه خواهیم شد که می‌گوید: « **اصلاح‌کننده جزئی وجود ندارد». یک تعریف جزئی دیگر از این نوع وجود دارد.** برای درک بهتر، لطفاً به کد مثال زیر نگاهی بیندازید. در اینجا، می‌توانیم ببینیم که در تعریف اول از کلمه کلیدی partial استفاده کرده‌ایم و اما در تعریف دوم کلاس، از کلمه کلیدی partial استفاده نکرده‌ایم و از این رو کامپایلر یک خطای کامپایل ایجاد می‌کند.

**قانون ۳:** اگر هر یک از بخش‌ها به صورت انتزاعی تعریف شوند، کل نوع به صورت انتزاعی در نظر گرفته می‌شود یا اگر هر یک از بخش‌ها به صورت مهر و موم شده تعریف شوند، کل نوع به صورت مهر و موم شده در نظر گرفته می‌شود یا اگر هر یک از بخش‌ها از یک کلاس ارث بری کنند، کل نوع از آن کلاس ارث بری می‌کند **.** در اینجا، می‌توانید ببینید که یکی از بخش‌هایی که به صورت انتزاعی تعریف کرده‌ایم، و از این رو کل کلاس انتزاعی می‌شود و همانطور که می‌دانیم نمی‌توانیم نمونه‌ای از یک کلاس انتزاعی ایجاد کنیم. و در اینجا وقتی سعی می‌کنیم نمونه‌ای از کلاس PartialEmployee که یک کلاس انتزاعی است ایجاد کنیم، کامپایلر یک خطای کامپایل ایجاد می‌کند.

**قانون ۴:** در مورد آن بحث کردیم، پشتیبانی نمی‌کند و [**C#.NET از ارث‌بری چندگانه کلاس که قبلاً در مقاله ارث‌بری چندگانه**](https://dotnettutorials.net/lesson/multiple-inheritance-csharp/) این به مشکل ابهام تبدیل شده است. این بدان معناست که بخش‌های مختلف کلاس Partial نباید کلاس‌های پایه متفاوتی را مشخص کنند. اگر کلاس‌های پایه متفاوتی را مشخص کنیم، با خطای کامپایل مواجه می‌شویم که می‌گوید **اعلان‌های Partial نباید کلاس‌های پایه متفاوتی را مشخص کنند** . برای درک بهتر، لطفاً به کد مثال زیر نگاهی بیندازید. در اینجا، می‌توانید ببینید که یک بخش از کلاس Partial از کلاس پایه Employee و بخش دیگر از کلاس Partial از کلاس پایه Customer ارث‌بری می‌کند. این بدان معناست که اکنون کلاس Partial دارای دو کلاس پایه است که این ارث‌بری چندگانه را با کلاس‌ها انجام می‌دهد که در C# امکان‌پذیر نیست و از این رو کامپایلر یک خطای کامپایل ایجاد می‌کند.

**قانون ۵:** اما با کلاس‌های Partial این امکان وجود دارد که بخش‌های مختلف کلاس Partial بتوانند رابط‌های پایه مختلفی را مشخص کنند و نوع نهایی باید و باید تمام متدهای رابط را پیاده‌سازی کند. در مثال زیر، PartialClass باید پیاده‌سازی هر دو متد رابط IEmployee و ICustomer را فراهم کند. این امر امکان‌پذیر است زیرا یک کلاس نمی‌تواند بیش از یک کلاس پایه داشته باشد، اما یک کلاس می‌تواند بیش از یک رابط پایه داشته باشد و به این ترتیب می‌توانیم وراثت چندگانه را در C# با رابط‌ها پیاده‌سازی کنیم.

```csharp
namespace PartialClassDemo
{
    public interface IEmployee
    {
        void EmployeeMethod();
    }
    public interface ICustomer
    {
        void CustomerMethod();
    }
    public partial class PartialClass : IEmployee
    {
        public void EmployeeMethod()
        {
            //Method Implementation
        }
    }
    public partial class PartialClass : ICustomer
    {
        public void CustomerMethod()
        {
            //Method Implementation
        }
    }
}
```

**نکته:** هر عضوی که در یک تعریف جزئی تعریف شود، برای همه بخش‌های دیگر کلاس جزئی در دسترس است. حال که کلاس‌های جزئی را درک کردیم، بیایید بیشتر پیش برویم و سعی کنیم متدهای جزئی را در سی شارپ درک کنیم.

##### **متدهای جزئی در سی شارپ چیست؟**

یک کلاس جزئی ممکن است شامل یک متد جزئی باشد. یک بخش از کلاس شامل امضای متد است. پیاده‌سازی متد جزئی می‌تواند در همان بخش یا بخش‌های دیگر کلاس جزئی تعریف شود. اگر پیاده‌سازی ارائه نشود، آنگاه متد و تمام فراخوانی‌های متد جزئی توسط کامپایلر در زمان کامپایل حذف می‌شوند.

بیایید با یک مثال متدهای Partial را درک کنیم. یک برنامه کنسول ایجاد کنید. یک فایل کلاس با نام PartialClassOne.cs به پروژه اضافه کنید و سپس کد زیر را در آن کپی و جایگذاری کنید.

```csharp
using System;

namespace PartialClassDemo
{
    partial class PartialClass
    {
        // Declaration of the partial method.
        partial void PartialMethod();

        // A public method calling the partial method
        public void PublicMethod()
        {
            Console.WriteLine("Public Method Invoked");
            PartialMethod();
        }
    }
}
```

همانطور که در کد بالا مشاهده می‌کنید، ما PartialMethod() را با استفاده از کلمه کلیدی partial ایجاد کرده‌ایم که آن را به یک متد partial تبدیل می‌کند و توجه داشته باشید که این متد partial بدنه خود را ندارد و فقط شامل امضای متد است. پیاده‌سازی این Partial را می‌توانید در تعریف این کلاس ارائه دهید یا می‌توانید پیاده‌سازی را در سایر بخش‌های این کلاس partial ارائه دهید. اما مهمترین نکته‌ای که باید به خاطر داشته باشید این است که پیاده‌سازی یک متد partial اختیاری است. اگر پیاده‌سازی را ارائه ندهیم، متد و تمام فراخوانی‌های متد partial به سادگی توسط کامپایلر در زمان کامپایل حذف می‌شوند. حال، اگر برنامه را بسازید و اگر کد IL را با استفاده از ابزار ILDASM مشاهده کنید، خواهید دید که متد Partial از کلاس Partial حذف شده است و همچنین فراخوانی متد partial از متد public نیز همانطور که در تصویر زیر نشان داده شده است، حذف شده است.

![متدهای جزئی در سی شارپ چیست؟](https://dotnettutorials.net/wp-content/uploads/2022/09/word-image-29911-8.png "متدهای جزئی در سی شارپ چیست؟")

در این مثال، متد PartialMethod() جزئی درون متد PublicMethod() فراخوانی می‌شود، اما پیاده‌سازی متد جزئی را نداریم. برای اثبات این موضوع، کد زیر را در متد Main() از کلاس Program کپی و جایگذاری کنید. در اینجا، ما یک نمونه از کلاس Partial ایجاد کرده و متد عمومی را فراخوانی می‌کنیم.

```csharp
using System;

namespace PartialClassDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            PartialClass SC = new PartialClass();
            SC.PublicMethod();
            Console.ReadKey();
        }
    }
}
```

**خروجی:** **متد عمومی فراخوانی شد**

وقتی برنامه‌ی فوق را اجرا می‌کنیم، توجه داشته باشید که با وجود نداشتن پیاده‌سازی برای PartialMethod() جزئی، خطای کامپایلر دریافت نمی‌کنیم. از آنجایی که پیاده‌سازی متد جزئی وجود ندارد، کامپایلر امضا و تمام فراخوانی‌های متد را حذف می‌کند.

پیاده‌سازی متدهای جزئی همانطور که بحث شد، می‌تواند در همان بخش یا در بخش‌های دیگر کلاس جزئی ارائه شود. حال، یک فایل کلاس با نام PartialClassTwo.cs اضافه می‌کنیم و سپس کد زیر را در آن کپی و جایگذاری می‌کنیم. پیاده‌سازی متد جزئی در اینجا ارائه شده است.

```csharp
using System;

namespace PartialClassDemo
{
    partial class PartialClass
    {
        // Partial method implemented
        partial void PartialMethod()
        {
            Console.WriteLine("Partial PartialMethod Invoked");
        }
    }
}
```

حالا، دوباره برنامه کنسول را اجرا کنید و لطفاً خروجی را مشاهده کنید. این بار، پیام‌های مربوط به متد جزئی و متد عمومی، مطابق شکل زیر، روی کنسول چاپ می‌شوند.

##### **قوانینی که باید هنگام کار با متدهای جزئی در سی شارپ رعایت کنید:**

هنگام کار با متدهای جزئی در سی شارپ، باید از برخی قوانین و مقررات پیروی کنیم. بیایید همه این قوانین و مقررات را یک به یک با مثال درک کنیم.

**قانون ۱:** متدهای Partial در سی شارپ به طور پیش‌فرض private هستند و می‌توانیم از هر مشخص‌کننده دسترسی صریحاً استفاده کنیم، و اگر سعی کنیم از هر مشخص‌کننده دسترسی صریحاً مانند public، private، protected و غیره استفاده کنیم، کامپایلری دریافت خواهیم کرد که اعلام می‌کند **یک متد Partial نمی‌تواند اصلاح‌کننده‌های دسترسی یا اصلاح‌کننده‌های virtual، abstract، override، new، sealed یا extern داشته باشد .** برای درک بهتر، لطفاً به کد مثال زیر نگاهی بیندازید. در اینجا، ما به طور صریح سعی داریم از یک اصلاح‌کننده دسترسی خصوصی با متد Partial استفاده کنیم، و از این رو کامپایلر خطا می‌دهد.

![یک متد جزئی نمی‌تواند دارای اصلاح‌کننده‌های دسترسی یا اصلاح‌کننده‌های مجازی، انتزاعی، override، جدید، مهر و موم شده یا خارجی باشد.](https://dotnettutorials.net/wp-content/uploads/2022/09/word-image-29911-10.png "یک متد جزئی نمی‌تواند دارای اصلاح‌کننده‌های دسترسی یا اصلاح‌کننده‌های مجازی، انتزاعی، override، جدید، مهر و موم شده یا خارجی باشد.")

**قانون ۲:** اعلان و پیاده‌سازی متد جزئی نباید همزمان باشد. اعلان آن باید در یک مکان باشد و پیاده‌سازی آن باید در مکانی متفاوت، یا در همان بخش یا در بخش‌های دیگر کلاس جزئی، باشد. اگر اعلان و پیاده‌سازی را همزمان قرار دهیم، با خطای کامپایلر مواجه می‌شویم که می‌گوید: « **هیچ اعلان تعریفی برای پیاده‌سازی اعلان متد جزئی 'PartialDemo.PartialClass.partialMethod()' یافت نشد»** . برای درک بهتر، لطفاً به کد مثال زیر نگاهی بیندازید. در اینجا، ما متد جزئی را همزمان اعلان و پیاده‌سازی می‌کنیم و از این رو با خطای کامپایلر مواجه می‌شویم.

![هیچ تعریفی برای پیاده‌سازی تعریف متد جزئی 'PartialDemo.PartialClass.partialMethod()' یافت نشد.](https://dotnettutorials.net/wp-content/uploads/2022/09/word-image-29911-11.png "هیچ تعریفی برای پیاده‌سازی تعریف متد جزئی 'PartialDemo.PartialClass.partialMethod()' یافت نشد.")

**قانون ۳:** نوع بازگشتی یک متد جزئی باید void باشد. اگر سعی کنیم از هر نوع بازگشتی دیگری مانند int، string، bool و غیره استفاده کنیم، با خطای کامپایلر مواجه خواهیم شد که می‌گوید **متدهای جزئی باید نوع بازگشتی void داشته باشند.** برای درک بهتر، لطفاً به کد مثال زیر نگاهی بیندازید. در اینجا، ما سعی داریم نوع بازگشتی را int قرار دهیم و از این رو کامپایلر خطا می‌دهد.

![کلاس‌ها و متدهای جزئی در سی شارپ به همراه مثال](https://dotnettutorials.net/wp-content/uploads/2022/09/word-image-29911-12.png "کلاس‌ها و متدهای جزئی در سی شارپ به همراه مثال")

**قانون ۴:** یک متد جزئی باید و باید درون یک کلاس جزئی یا ساختار جزئی تعریف شود. یک کلاس یا ساختار غیر جزئی نمی‌تواند شامل متدهای جزئی باشد. اگر سعی کنیم یک متد جزئی را تحت یک کلاس یا ساختار غیر جزئی تعریف کنیم، در زمان کامپایل با خطایی مواجه خواهیم شد که می‌گوید: **یک متد جزئی باید درون یک کلاس جزئی یا ساختار جزئی تعریف شود** . برای درک بهتر، لطفاً به کد زیر نگاهی بیندازید. در اینجا، ما سعی داریم یک متد جزئی را تحت یک کلاس غیر جزئی تعریف کنیم و از این رو کامپایلر خطا می‌دهد.

![کلاس و متد جزئی در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2018/08/Partial-Class-and-Partial-Method-in-C.jpg "کلاس و متد جزئی در سی شارپ")

**قانون ۵:** امضای اعلان متد جزئی باید با امضای پیاده‌سازی مطابقت داشته باشد، در غیر این صورت با خطای کامپایل مواجه خواهیم شد. برای درک بهتر، لطفاً به کد زیر نگاهی بیندازید. در اینجا، در بخش اول کلاس جزئی، متد جزئی را با یک پارامتر عدد صحیح تعریف کرده‌ایم و در بخش دوم کلاس جزئی که پیاده‌سازی را ارائه می‌دهیم، پارامتر عدد صحیح را لحاظ نکرده‌ایم و از این رو کامپایلر خطا می‌دهد.

![امضای اعلان متد جزئی باید با امضای پیاده‌سازی مطابقت داشته باشد.](https://dotnettutorials.net/wp-content/uploads/2022/09/word-image-29911-14.png "امضای اعلان متد جزئی باید با امضای پیاده‌سازی مطابقت داشته باشد.")

**قانون ۶:** یک متد جزئی فقط یک بار می‌تواند در همان بخش یا در بخش‌های دیگر کلاس جزئی پیاده‌سازی شود. اگر سعی کنیم یک متد جزئی را بیش از یک بار پیاده‌سازی کنیم، با خطای کامپایل مواجه خواهیم شد که می‌گوید: **یک متد جزئی ممکن است چندین اعلان پیاده‌سازی نداشته باشد .** برای درک بهتر، لطفاً به کد مثال زیر نگاهی بیندازید. در اینجا، در بخش اول، متد جزئی را تعریف کرده‌ایم و در بخش دوم و در بخشی که پیاده‌سازی را برای متد جزئی ارائه می‌دهیم، و از آنجایی که دو پیاده‌سازی ارائه می‌دهیم، با خطای کامپایل مواجه می‌شویم. دلیل این امر این است که در یک کلاس، نمی‌توانیم دو متد را با نام و پارامترهای یکسان تعریف کنیم، یعنی override کردن متد در یک کلاس واحد امکان‌پذیر نیست.
