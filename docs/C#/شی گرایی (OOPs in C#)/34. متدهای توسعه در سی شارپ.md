## **متدهای توسعه در سی شارپ به همراه مثال**

در این مقاله، قصد دارم در مورد **متدهای توسعه‌یافته (Extension Methods) در سی‌شارپ** با مثال صحبت کنم. در پایان این مقاله، شما متوجه خواهید شد که متدهای توسعه‌یافته (Extension Methods) در سی‌شارپ دقیقاً چه هستند و چه زمانی و چگونه می‌توان از این متدهای توسعه‌یافته در سی‌شارپ استفاده کرد.

##### **متدهای توسعه (Extension Methods) در سی شارپ چیستند؟**

این یک ویژگی جدید است که در C# 3.0 اضافه شده است و به ما امکان می‌دهد بدون ویرایش کد منبع کلاس، متدهای جدیدی را به آن اضافه کنیم، یعنی اگر یک کلاس از مجموعه‌ای از اعضا تشکیل شده باشد و در آینده بخواهید متدهای جدیدی به آن کلاس اضافه کنید، می‌توانید آن متدها را بدون ایجاد هیچ تغییری در کد منبع کلاس اضافه کنید.

متدهای توسعه می‌توانند به عنوان رویکردی برای گسترش قابلیت‌های یک کلاس در آینده مورد استفاده قرار گیرند، اگر کد منبع کلاس در دسترس نباشد یا ما هیچ مجوزی برای ایجاد تغییر در کلاس نداشته باشیم.

قبل از متدهای توسعه، وراثت رویکردی بود که برای گسترش قابلیت‌های یک کلاس استفاده می‌شد، یعنی اگر بخواهیم اعضای جدیدی را به یک کلاس موجود بدون ایجاد تغییر در کلاس اضافه کنیم، یک کلاس فرزند برای آن کلاس موجود تعریف می‌کنیم و سپس اعضای جدید را به کلاس فرزند اضافه می‌کنیم.

در مورد یک متد الحاقی، ما عملکرد یک کلاس موجود را گسترش خواهیم داد. در این حالت، ما یک کلاس جدید ایجاد می‌کنیم و با استفاده از آن کلاس جدید، عملکرد یک کلاس موجود را گسترش می‌دهیم.

هر دوی این رویکردها می‌توانند برای گسترش قابلیت‌های یک کلاس موجود مورد استفاده قرار گیرند، در حالی که در مورد وراثت، متدهای تعریف شده در کلاس‌های قدیمی و جدید را با استفاده از شیء کلاس جدید فراخوانی می‌کنیم، در حالی که در مورد متدهای توسعه، متدهای قدیمی و جدید را با استفاده از شیء کلاس قدیمی فراخوانی می‌کنیم.

**نکته:** گاهی اوقات وراثت امکان‌پذیر نیست. این بدان معناست که اگر کلاس با استفاده از اصلاحگر sealed تعریف شود، نمی‌توانیم یک کلاس مشتق شده از sealed ایجاد کنیم. اما می‌توانیم با استفاده از Extension Method، عملکرد کلاس sealed را گسترش دهیم.

##### **مثال برای درک متدهای توسعه در سی شارپ:**

بیایید با یک مثال، متدهای توسعه (Extension Methods) در سی شارپ را درک کنیم. ابتدا یک برنامه کنسول ایجاد کنید و سپس یک فایل کلاس با نام OldClass.cs اضافه کنید و سپس کد زیر را در آن کپی و جایگذاری کنید. همانطور که در کد زیر مشاهده می‌کنید، در اینجا OldClass را با یک عضو داده یعنی x و دو متد یعنی Test1 و Test2 ایجاد کرده‌ایم.

```csharp
using System;

namespace ExtensionMethods
{
    public class OldClass
    {
        public int x = 100;
        public void Test1()
        {
            Console.WriteLine("Method one: " + this.x);
        }
        public void Test2()
        {
            Console.WriteLine("Method two: " + this.x);
        }
    }
}
```

حالا نیاز ما اضافه کردن سه متد جدید به کلاس OldClass است. به دلایلی، فرض می‌کنیم کد منبع در دسترس ما نیست یا به منبع دسترسی برای ویرایش نداریم. این بدان معناست که نمی‌توانیم کد منبع OldClass را تغییر دهیم. پس چگونه می‌توانیم قابلیت OldClass را گسترش دهیم، یعنی چگونه می‌توانیم سه متد به OldClass اضافه کنیم و چگونه می‌توانیم آن متدهای جدید را با استفاده از شیء OldClass فراخوانی کنیم. اینجاست که متدهای الحاقی وارد عمل می‌شوند.

بنابراین، می‌توانیم با کمک متدهای افزونه، قابلیت‌های OldClass را گسترش دهیم. بیایید یک کلاس جدید با نام NewClass.cs ایجاد کنیم و سپس کد زیر را در آن کپی و جایگذاری کنیم.

```csharp
using System;

namespace ExtensionMethods
{
    public static class NewClass
    {
        public static void Test3(this OldClass O)
        {
            Console.WriteLine("Method Three");
        }
        public static void Test4(this OldClass O, int x)
        {
            Console.WriteLine("Method Four: " + x);
        }
        public static void Test5(this OldClass O)
        {
            Console.WriteLine("Method Five:" + O.x);
        }
    }
}
```

ابتدا برنامه را آزمایش می‌کنیم و می‌بینیم که آیا می‌توانیم با استفاده از شیء OldClass به متدهای فوق دسترسی پیدا کنیم یا خیر، و سپس متدهای الحاقی یعنی کدی که در کلاس فوق نوشته شده است را درک خواهیم کرد. حال برای آزمایش اینکه آیا متدها با استفاده از اشیاء کلاس قدیمی قابل دسترسی هستند یا خیر، متد اصلی کلاس Program را به صورت زیر تغییر می‌دهیم. در اینجا، می‌توانید ببینید که ما در حال ایجاد یک نمونه از OldClass هستیم و سپس متدهایی را که درون OldClass تعریف شده‌اند و همچنین متدهایی را که درون NewClass تعریف شده‌اند، فراخوانی می‌کنیم و کامپایلر نیز خطایی نمی‌دهد.

```csharp
using System;

namespace ExtensionMethods
{
    class Program
    {
        static void Main(string[] args)
        {
            OldClass obj = new OldClass();
            obj.Test1();
            obj.Test2();
            //Calling Extension Methods
            obj.Test3();
            obj.Test4(10);
            obj.Test5();
            Console.ReadLine();
        }
    }
}
```

حالا، برنامه را اجرا کنید و ببینید که آیا همه چیز طبق انتظار کار می‌کند یا خیر، خروجی زیر را نمایش می‌دهد.

##### **نکاتی که هنگام کار با متدهای Extension در سی شارپ باید به خاطر داشته باشید:**

هنگام کار با متد Extension در سی شارپ، باید نکات زیر را به خاطر داشته باشیم.

1. متدهای افزونه فقط زیر کلاس استاتیک تعریف شوند. اگر NewClass ما را بررسی کنید، خواهید دید که NewClass یک کلاس استاتیک است.
2. قبلاً بحث کردیم که کلاس استاتیک در سی شارپ فقط شامل اعضای استاتیک است. از آنجایی که یک متد الحاقی تحت یک کلاس استاتیک تعریف می‌شود، به این معنی است که متد الحاقی باید به عنوان یک متد استاتیک ایجاد شود، در حالی که به محض اینکه متد به کلاس دیگری متصل شود، به متد غیر استاتیک تبدیل می‌شود. حال، اگر متدهای موجود در NewClass را بررسی کنید، خواهید دید که هر سه متد فقط به صورت استاتیک تعریف شده‌اند.
3. اولین پارامتر یک متد الحاقی، پارامتر اتصال (binding parameter) است که باید نام کلاسی باشد که متد باید به آن متصل شود و پارامتر اتصال باید با پیشوند this شروع شود. از آنجایی که در اینجا ما این متدهای الحاقی را برای گسترش قابلیت‌های OldClass ایجاد می‌کنیم، می‌توانید بررسی کنید که اولین پارامتر همه این متدها OldClass خواهد بود که با کلمه کلیدی this نیز پیشوند شده است.
4. یک متد الحاقی می‌تواند فقط یک پارامتر اتصال داشته باشد و آن باید در وهله اول در لیست پارامترها تعریف شود.
5. در صورت نیاز، می‌توان یک متد الحاقی با پارامترهای معمولی تعریف کرد که از مکان دوم لیست پارامترها شروع می‌شوند. اگر متد Test3 را بررسی کنید، پارامتر دوم را از نوع int ارسال کرده‌ایم و هنگام فراخوانی این متد، باید یک مقدار صحیح نیز ارسال کنیم.

##### **مثال بلادرنگ متد افزونه در سی شارپ:**

بیایید یک سناریوی بلادرنگ را بررسی کنیم که در آن می‌توانیم از متد الحاقی در سی‌شارپ استفاده کنیم. همانطور که می‌دانیم، رشته یک کلاس داخلی است که توسط چارچوب دات‌نت ارائه شده است. این بدان معناست که کد منبع این کلاس در دسترس ما نیست و از این رو می‌توانیم کد منبع کلاس رشته را تغییر دهیم. همچنین، رشته یک کلاس مهر و موم شده در سی‌شارپ است و از این رو نمی‌توانیم یک کلاس مشتق شده از کلاس رشته ایجاد کنیم.

حالا نیاز ما این است که یک متد به کلاس String اضافه کنیم، مثلاً GetWordCount() و آن متد تعداد کلمات موجود در یک رشته را برمی‌گرداند و ما باید این متد را همانطور که در تصویر زیر نشان داده شده است، فراخوانی کنیم. یعنی با استفاده از شیء رشته، باید آن متد را فراخوانی کنیم.

![مثال بلادرنگ از متدهای افزونه سی شارپ](https://dotnettutorials.net/wp-content/uploads/2018/09/C-Extension-Methods-Real-time-Example.png "مثال بلادرنگ از متدهای افزونه سی شارپ")

چگونه می‌توانیم به این هدف دست یابیم؟ می‌توانیم با استفاده از Extension Methods در C# به الزام فوق دست یابیم. ابتدا یک فایل کلاس با نام **StringExtension.cs** **ایجاد کنید و سپس کد زیر را در آن کپی و جایگذاری کنید. همانطور که می‌بینید، در اینجا کلاس را به صورت استاتیک ایجاد کرده‌ایم و از این رو GetWordCount را نیز به صورت استاتیک قرار** داده‌ایم و اولین پارامتر را به عنوان نام کلاس رشته ارائه می‌دهیم تا بتوانیم این متد را روی شیء کلاس String فراخوانی کنیم. در اینجا، رشته را بر اساس فاصله تقسیم می‌کنیم و سپس هر کلمه را در یک آرایه رشته‌ای ذخیره می‌کنیم و سپس LINQ Count را فراخوانی می‌کنیم که تعداد عناصر موجود در آرایه رشته‌ای را برمی‌گرداند و آن تعداد را از متد برمی‌گردانیم. اما اگر رشته خالی یا تهی باشد، در این صورت، تعداد را 0 برمی‌گردانیم.

```csharp
using System.Linq;

namespace ExtensionMethods
{
    public static class StringExtension
    {
        public static int GetWordCount(this string inputstring)
        {
            if (!string.IsNullOrEmpty(inputstring))
            {
                //Split the string by a space
                string[] strArray = inputstring.Split(' ');
                return strArray.Count();
            }
            else
            {
                return 0;
            }
        }
    }
}
```

پس از ایجاد متد الحاقی برای کلاس رشته، می‌توانید از آن متد الحاقی روی شیء String استفاده کنید. حال، متد Main کلاس Program را طوری تغییر می‌دهیم که از متد الحاقی رشته برای دریافت تعداد کلمات، همانطور که در کد زیر نشان داده شده است، استفاده کند.

```csharp
using System;

namespace ExtensionMethods
{
    class Program
    {
        static void Main(string[] args)
        {
            string myWord = "Welcome to Dotnet Tutorials Extension Methods Article";
            int wordCount = myWord.GetWordCount();
            Console.WriteLine("string : " + myWord);
            Console.WriteLine("Count : " + wordCount);
            Console.Read();
        }
    }
}
```

حالا برنامه را اجرا کنید و باید خروجی مورد انتظار را همانطور که در تصویر زیر نشان داده شده است، دریافت کنید.

![مثال بلادرنگ از متدهای افزونه در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2018/09/Real-time-Example-of-Extension-Methods-in-C.png "مثال بلادرنگ از متدهای افزونه در سی شارپ")

**نکته:** متدهای توسعه‌یافته در سی‌شارپ به ما این امکان را می‌دهند که بدون ایجاد یک نوع مشتق‌شده جدید، و بدون تغییر نوع اصلی، متدهایی را به یک نوع موجود اضافه کنیم. متدهای توسعه‌یافته، متدهای استاتیک هستند، اما طوری فراخوانی می‌شوند که انگار متدهای نمونه روی نوع توسعه‌یافته هستند.

##### **LINQ به طور موثر از متدهای Extension استفاده می‌کند:**

رایج‌ترین متدهای افزونه، عملگرهای پرس‌وجوی استاندارد LINQ هستند که قابلیت پرس‌وجو را به انواع موجود System.Collections.IEnumerable و System.Collections.Generic.IEnumerable\<T> اضافه می‌کنند. وقتی شروع به بحث در مورد مفهوم LINQ کنیم، درک بهتری از متدهای افزونه خواهید داشت. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید. در اینجا، ما از متد Where در LINQ برای فیلتر کردن مجموعه استفاده می‌کنیم.

```csharp
using System;
using System.Collections.Generic;
using System.Linq;

namespace SealedDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            List<int> intList = new List<int>()
            {
                10, 20, 30, 40, 50
            };

            //If you go to the definition of Where method, then you will see that
            //it is implemented as an extension method
            var List1 = intList.Where(x => x > 20).ToList();

            Console.ReadKey();
        }
    }
}
```

در اینجا، اگر به تعریف متد Where بروید، خواهید دید که همانطور که در تصویر زیر نشان داده شده است، به عنوان یک متد الحاقی پیاده‌سازی شده است.
