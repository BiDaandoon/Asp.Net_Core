در این مقاله، کپسوله‌سازی در سی‌شارپ را با مثال‌هایی مورد بحث قرار خواهم داد. کپسوله‌سازی یکی از اصول اساسی برنامه‌نویسی شی‌گرا است. بسیاری از دانشجویان و برنامه‌نویسان، به ویژه مبتدیان، درک اصل کپسوله‌سازی را کمی دشوار می‌دانند. بنابراین، در این مقاله، سعی می‌کنم آن را تا حد امکان ساده کنم تا بتوانید به راحتی آن را درک کنید. به عنوان بخشی از این مقاله، نکات زیر را با جزئیات خواهید آموخت.

6. **پیاده‌سازی کپسوله‌سازی در سی‌شارپ با استفاده از ویژگی‌ها**

##### **اصل کپسوله سازی در سی شارپ چیست؟**

طبق MSDN، **کپسوله‌سازی (Encapsulation) وضعیت داخلی و عملکرد یک شیء را پنهان می‌کند و فقط دسترسی به آن را از طریق مجموعه‌ای عمومی از توابع امکان‌پذیر می‌سازد** . بیایید تعریف بالا را به صورت زیر ساده کنیم:

فرآیند اتصال یا گروه‌بندی **حالت (یعنی اعضای داده)** و **رفتار (یعنی توابع عضو)** با هم در یک واحد واحد (یعنی **کلاس، رابط، ساختار و غیره** ) در C# کپسوله‌سازی نامیده می‌شود. اصل کپسوله‌سازی تضمین می‌کند که حالت و رفتار یک واحد (یعنی **کلاس، رابط، ساختار و غیره** ) قابل دسترسی نباشد **) مستقیماً از واحدهای دیگر (یعنی کلاس، رابط، ساختار و غیره** .

![کپسوله سازی در سی شارپ](https://dotnettutorials.net/wp-content/uploads/2018/08/Encapsulation-in-C.jpg "کپسوله سازی در سی شارپ")

اصل کپسوله‌سازی در سی‌شارپ بسیار شبیه به کپسول است. همانطور که یک کپسول داروی خود را درون خود قرار می‌دهد، به همان روش در سی‌شارپ، اصل کپسوله‌سازی حالت (متغیرها) و رفتار (متدها) را در یک واحد واحد به نام کلاس، enum، رابط و غیره قرار می‌دهد. بنابراین، می‌توانید کپسوله‌سازی را به عنوان یک پوشش یا لایه در نظر بگیرید که حالت‌ها و رفتارهای مرتبط را در یک واحد واحد به هم متصل می‌کند.

##### **مثالی از کپسوله‌سازی در دنیای واقعی:**

همانطور که قبلاً بحث کردیم، یکی از نمونه‌های واقعی کپسوله‌سازی، کپسول است، زیرا کپسول تمام مواد دارویی خود را درون خود جای می‌دهد. به همین ترتیب، کپسوله‌سازی C#، یعنی واحدها (کلاس، رابط، enums، structs و غیره) تمام اعضای داده و توابع عضو خود را درون خود محصور می‌کند.

یک مثال دیگر از کپسوله‌سازی در دنیای واقعی می‌تواند کیف مدرسه یا محل کار شما باشد. این کیف حاوی چیزهای مختلفی مانند خودکار، مداد، دفترچه یادداشت و غیره است. برای برداشتن هر چیزی، باید آن کیف را باز کنید. به طور مشابه، در C#، یک واحد کپسوله‌سازی شامل داده‌ها و رفتارهای درون خود است و برای دسترسی به آنها، به یک شیء از آن واحد نیاز دارید.

##### **مثال برای درک کپسوله‌سازی در سی‌شارپ:**

هر کلاس، رابط، ساختار، enum و غیره که ایجاد می‌کنیم، نمونه‌ای از کپسوله‌سازی است، بنابراین بیایید کلاسی به نام Bank به صورت زیر ایجاد کنیم تا کپسوله‌سازی را درک کنیم:

```csharp
namespace EncapsulationDemo
{
    class Bank
    {
        public long AccountNumber;
        public string Name;
        public int Balance;

        public void GetBalance()
        {
        }
        public void WithdrawAmount()
        {
        }
        public void Deposit()
        {
        }
    }
}
```

در اینجا، کلاس Bank نمونه‌ای از کپسوله‌سازی است. متغیرها (AccountNumber، Name و Balance) و متدهای (GetBalance، WithdrawAmount و Deposit) کلاس در یک واحد واحد، که کلاس Bank است، محدود شده‌اند. در اینجا، کپسوله‌سازی، جزئیات پیاده‌سازی کلاس Bank را به آن متصل کرده و آن را از سایر کلاس‌ها پنهان می‌کند. اگر کلاس‌های دیگر بخواهند به این جزئیات دسترسی داشته باشند، باید شیء کلاس Bank را ایجاد کنند تا به داده‌ها و رفتار آن دسترسی پیدا کنند، همانطور که در کد زیر نشان داده شده است.

```csharp
namespace EncapsulationDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            Bank bank = new Bank();
            bank.AccountNumber = 12345678;
            bank.Name = "Pranata";
            bank.GetBalance();
            bank.WithdrawAmount();
        }
    }
}
```

به طور مشابه، اگر یک رابط، ساختار یا enum ایجاد کنید، این نیز نمونه‌ای از کپسوله‌سازی است. بزرگترین مزیت کپسوله‌سازی، پنهان‌سازی داده‌ها است. این بدان معناست که با استفاده از کپسوله‌سازی، می‌توانیم به پنهان‌سازی داده‌ها دست یابیم. بیایید بیشتر پیش برویم و پنهان‌سازی داده‌ها را در C# با مثال‌هایی درک کنیم.

##### **چیست ؟ پنهان سازی داده در سی شارپ**

پنهان‌سازی داده‌ها یا پنهان‌سازی اطلاعات فرآیندی است که در آن داده‌های داخلی را از دنیای خارج پنهان می‌کنیم. هدف از پنهان‌سازی داده‌ها، محافظت از داده‌ها در برابر سوءاستفاده توسط دنیای خارج است. پنهان‌سازی داده‌ها همچنین به عنوان کپسوله‌سازی داده‌ها شناخته می‌شود. بدون اصل کپسوله‌سازی، نمی‌توانیم به پنهان‌سازی داده‌ها دست یابیم.

به عبارت ساده، می‌توانیم بگوییم که فرآیند تعریف یک کلاس با پنهان کردن اعضای داده داخلی آن از خارج از کلاس و دسترسی به آن اعضای داده داخلی فقط از طریق متدهای عمومی (متدهای setter و getter) یا ویژگی‌هایی با اعتبارسنجی مناسب، کپسوله‌سازی (Encapsulation) نامیده می‌شود.

**نکته:** کپسوله‌سازی داده‌ها، پنهان‌سازی داده‌ها نیز نامیده می‌شود، زیرا با استفاده از این اصل، می‌توانیم داده‌های داخلی را از دید خارج از کلاس پنهان کنیم.

##### **چگونه می‌توانیم پنهان‌سازی داده‌ها یا کپسوله‌سازی داده‌ها را در سی‌شارپ پیاده‌سازی کنیم؟**

در سی شارپ، کپسوله سازی داده (Data Encapsulation) پیاده سازی شده است.

1. با تعریف متغیرها به صورت خصوصی (برای محدود کردن دسترسی مستقیم به آنها از خارج از کلاس)
2. با تعریف یک جفت متد یا ویژگی عمومی setter و getter برای دسترسی به متغیرهای خصوصی از خارج از کلاس.

ما متغیرها را به صورت خصوصی (private) تعریف می‌کنیم تا دسترسی مستقیم به آنها از خارج از کلاس متوقف شود. متدهای عمومی setter و getter یا ویژگی‌های عمومی برای دسترسی به متغیرهای خصوصی از خارج از کلاس با اعتبارسنجی‌های مناسب استفاده می‌شوند. اگر دسترسی مستقیم به متغیرهای یک کلاس را فراهم کنیم، نمی‌توانیم داده‌ها را قبل از ذخیره آنها در متغیر یا هنگام بازیابی داده‌های ذخیره شده در متغیر، اعتبارسنجی کنیم. بنابراین نکته‌ای که باید به خاطر داشته باشید این است که با پیاده‌سازی کپسوله‌سازی داده‌ها یا پنهان‌سازی داده‌ها در سی‌شارپ، ما از داده‌ها محافظت یا آنها را ایمن می‌کنیم.

بنابراین، کپسوله‌سازی داده‌ها یا پنهان‌سازی داده‌ها در سی‌شارپ با استفاده از **مشخصه‌های دسترسی**(Access Specifiers) پیاده‌سازی می‌شود . یک مشخصه دسترسی، دامنه و قابلیت مشاهده عضو کلاس را تعریف می‌کند و ما قبلاً در مقاله قبلی خود در مورد انواع مختلف مشخصه‌های دسترسی پشتیبانی شده در سی‌شارپ بحث کرده‌ایم. سی‌شارپ از شش مشخصه دسترسی زیر پشتیبانی می‌کند:

1. **public** : اعضای public می‌توانند توسط هر کد دیگری در همان اسمبلی یا اسمبلی دیگری که به آن ارجاع می‌دهد، قابل دسترسی باشند.
2. **private** : اعضای private فقط از طریق کد همان کلاس قابل دسترسی هستند.
3. **protected** : اعضای protected در سی شارپ هم در داخل همان کلاس و هم در کلاس‌هایی که از آن کلاس مشتق شده‌اند، در دسترس هستند.
4. **داخلی** : اعضای داخلی می‌توانند توسط هر کدی در همان اسمبلی قابل دسترسی باشند اما از اسمبلی دیگر قابل دسترسی نیستند.
5. **protected internal** : اعضای داخلی protected می‌توانند توسط هر کدی در اسمبلی که در آن تعریف شده‌اند یا از درون یک کلاس مشتق شده در اسمبلی دیگر، قابل دسترسی باشند.
6. **private protected:** اعضای private protected می‌توانند توسط انواع مشتق شده از کلاسی که در اسمبلی حاوی آن تعریف شده است، قابل دسترسی باشند.

##### **پیاده‌سازی کپسوله‌سازی داده‌ها یا پنهان‌سازی داده‌ها در سی‌شارپ با استفاده از متدهای Setter و Getter:**

بیایید مثالی برای درک کپسوله‌سازی در سی‌شارپ بزنیم. در مثال زیر، متغیر balance را در کلاس Bank به صورت private تعریف می‌کنیم و از این رو، نمی‌توان مستقیماً از خارج از کلاس Bank به آن دسترسی داشت. برای دسترسی به متغیر balance از خارج از کلاس Bank، دو متد public به نام‌های GetBalance و SetBalance را ارائه کرده‌ایم. متد GetBalance (که getter نیز نامیده می‌شود) برای دریافت مقدار ذخیره شده در متغیر balance از خارج از کلاس bank استفاده می‌شود و متد SetBalance (که Setter نیز نامیده می‌شود) برای تنظیم مقدار در متغیر balance از خارج از کلاس bank استفاده می‌شود. در داخل کلاس Bank، می‌توانید مستقیماً به متغیرهای private دسترسی داشته باشید، اما نمی‌توانید مستقیماً از خارج از کلاس Bank به آنها دسترسی داشته باشید.

```csharp
using System;

namespace EncapsulationDemo
{
    public class Bank
    {
        //Hiding class data by declaring the variable as private
        private double balance;

        //Creating public Setter and Getter methods

        //Public Getter Method
        //This method is used to return the data stored in the balance variable
        public double GetBalance()
        {
            //add validation logic if needed
            return balance;
        }

        //Public Setter Method
        //This method is used to stored the data  in the balance variable
        public void SetBalance(double balance)
        {
            // add validation logic to check whether data is correct or not
            this.balance = balance;
        }
    }
    class Program
    {
        public static void Main()
        {
            Bank bank = new Bank();
            //You cannot access the Private Variable
            //bank.balance; //Compile Time Error

            //You can access the private variable via public setter and getter methods
            bank.SetBalance(500);
            Console.WriteLine(bank.GetBalance());
            Console.ReadKey();
        }
    }
}
```

##### **مزایای ارائه دسترسی به متغیر از طریق متدهای Setter و Getter در سی شارپ چیست؟**

اگر دسترسی به متغیر را از طریق متدهای Setter و Getter در سی شارپ فراهم کنیم، می‌توانیم داده‌های داده شده توسط کاربر را قبل از ذخیره مقدار در متغیر، اعتبارسنجی کنیم. برای مثال، در برنامه بالا، اگر نمی‌خواهید مقدار -VE را در متغیر balance ذخیره کنید، قبل از ذخیره آن در متغیر، مقدار را بررسی و اعتبارسنجی می‌کنید. بنابراین، می‌توانیم مقدار داده شده را قبل از ذخیره آن در متغیر balance اعتبارسنجی کنیم. اگر دسترسی مستقیم به متغیر balance را فراهم کنیم، اعتبارسنجی مقدار مقدار داده شده قبل از ذخیره آن در متغیر balance غیرممکن است.

بنابراین، دلیل اصلی استفاده از پنهان‌سازی داده‌ها، امنیت است. از آنجایی که ما از مشخص‌کننده‌های دسترسی خصوصی برای متغیرهای خود استفاده می‌کنیم، می‌توانیم اطلاعات حیاتی را در چنین متغیرهایی ذخیره کنیم که فقط در داخل کلاس قابل مشاهده خواهند بود. هیچ کس دیگری نمی‌تواند مستقیماً به آنها دسترسی داشته باشد. همچنین می‌توانیم در صورت نیاز، اعتبارسنجی‌هایی را در متدهای setter و getter اعمال کنیم. این کار همچنین امنیت را افزایش می‌دهد تا هیچ کس نتواند داده‌های غیرقانونی را برای سوءاستفاده تنظیم کند.

##### **اگر در طراحی یک کلاس، اصل کپسوله‌سازی در سی‌شارپ را رعایت نکنیم، چه مشکلی پیش می‌آید؟**

اگر در طراحی کلاس از اصل کپسوله‌سازی در سی‌شارپ پیروی نکنیم، نمی‌توانیم داده‌های داده شده توسط کاربر را مطابق با الزامات تجاری خود اعتبارسنجی کنیم، زیرا مدیریت تغییرات آینده بسیار دشوار است. بیایید این موضوع را با یک مثال درک کنیم. فرض کنید در الزامات اولیه پروژه، مشتری اشاره نکرده است که برنامه نباید اجازه ذخیره عدد منفی را بدهد. بنابراین، ما دسترسی مستقیم به متغیر را از خارج از کلاس فراهم می‌کنیم و اکنون، کاربر می‌تواند هر مقداری را در آن ذخیره کند، همانطور که در مثال زیر نشان داده شده است. در اینجا، می‌توانید ببینید که ما مستقیماً از خارج از کلاس بانک به متغیر Amount دسترسی پیدا می‌کنیم و مقادیر مثبت و منفی را در آن قرار می‌دهیم.

```csharp
using System;

namespace EncapsulationDemo
{
    public class Bank
    {
        public int Amount;
    }
    class Program
    {
        public static void Main()
        {
            Bank bank = new Bank();
            //We can access the Amount Variable directly

            //Setting positive amount
            bank.Amount = 50;
            Console.WriteLine(bank.Amount);

            //Setting negative amount
            bank.Amount = -150;
            Console.WriteLine(bank.Amount);

            Console.ReadKey();
        }
    }
}
```

همین. همانطور که انتظار می‌رفت، کار می‌کند. بعداً، در آینده، کلاینت می‌خواهد که برنامه مقدار منفی را مجاز نداند. سپس، باید مقادیر داده شده توسط کاربر را قبل از ذخیره آنها در متغیر Amount اعتبارسنجی کنیم. از این رو، باید برنامه را با پیروی از اصل کپسوله‌سازی به شرح زیر توسعه دهیم:

```csharp
using System;

namespace EncapsulationDemo
{
    public class Bank
    {
        private int Amount;
        public int GetAmount()
        {
            return Amount;
        }
        public void SetAmount(int Amount)
        {
            if (Amount > 0)
            {
                this.Amount = Amount;
            }
            else
            {
                throw new Exception("Please Pass a Positive Value");
            }
        }
    }
    class Program
    {
        public static void Main()
        {
            try
            {
                Bank bank = new Bank();
                //We cannot access the Amount Variable directly
                //bank.Amount = 50; //Compile Time Error
                //Console.WriteLine(bank.Amount); //Compile Time Error

                //Setting Positive Value
                bank.SetAmount(10);
                Console.WriteLine(bank.GetAmount());

                //Setting Negative Value
                bank.SetAmount(-150);
                Console.WriteLine(bank.GetAmount());
            }
            catch(Exception ex)
            {
                Console.WriteLine(ex.Message);
            }
           
            Console.ReadKey();
        }
    }
}
```

##### **پیاده‌سازی کپسوله‌سازی داده‌ها یا پنهان‌سازی داده‌ها در سی‌شارپ با استفاده از ویژگی‌ها:**

ویژگی‌ها (Properties) یک ویژگی جدید زبان برنامه‌نویسی هستند که در سی‌شارپ معرفی شده‌اند. ویژگی‌ها در سی‌شارپ با خواندن و نوشتن مقادیر در یک فیلد یا متغیر از یک کلاس، به محافظت از آن کمک می‌کنند. رویکرد اول، یعنی خودِ setter و getter، خوب است، اما کپسوله‌سازی داده‌ها در سی‌شارپ با ویژگی‌ها می‌تواند بسیار روان‌تر انجام شود.

بیایید با یک مثال نحوه پیاده‌سازی کپسوله‌سازی داده‌ها یا پنهان‌سازی داده‌ها در سی‌شارپ را با استفاده از ویژگی‌ها درک کنیم . در مثال زیر، درون کلاس Bank، متغیر \_Amount را به عنوان private علامت‌گذاری کرده‌ایم تا دسترسی مستقیم از خارج از کلاس Bank محدود شود. ما با اعلام متغیر \_Amount به عنوان public، ویژگی Amount را برای دسترسی به آن در معرض نمایش قرار داده‌ایم. اکنون، از خارج از کلاس Bank، می‌توانیم از طریق ویژگی public Amount به متغیر private \_Amount دسترسی پیدا کنیم.

```csharp
using System;

namespace EncapsulationDemo
{
    public class Bank
    {
        private double _Amount;
        public double Amount
        {
            get
            {
                return _Amount;
            }
            set
            {
                // Validate the value before storing it in the _Amount variable
                if (value < 0)
                {
                    throw new Exception("Please Pass a Positive Value");
                }
                else
                {
                    _Amount = value;
                }
            }
        }
    }
    class Program
    {
        public static void Main()
        {
            try
            {
                Bank bank = new Bank();
                //We cannot access the _Amount Variable directly
                //bank._Amount = 50; //Compile Time Error
                //Console.WriteLine(bank._Amount); //Compile Time Error

                //Setting Positive Value using public Amount Property
                bank.Amount= 10;

                //Setting the Value using public Amount Property
                Console.WriteLine(bank.Amount);
                
                //Setting Negative Value
                bank.Amount = -150;
                Console.WriteLine(bank.Amount);
            }
            catch (Exception ex)
            {
                Console.WriteLine(ex.Message);
            }

            Console.ReadKey();
        }
    }
}
```

##### **مزایای کپسوله‌سازی در سی‌شارپ:**

1. **حفاظت از داده‌ها:** می‌توانید داده‌ها را قبل از ذخیره در متغیر، اعتبارسنجی کنید.
2. **دستیابی به پنهان‌سازی داده‌ها:** کاربر هیچ اطلاعی از پیاده‌سازی داخلی کلاس نخواهد داشت.
3. **امنیت:** اصل کپسوله‌سازی به ایمن‌سازی کد ما کمک می‌کند، زیرا تضمین می‌کند که واحدهای دیگر (کلاس‌ها، رابط‌ها و غیره) نمی‌توانند مستقیماً به داده‌ها دسترسی داشته باشند.
4. **انعطاف‌پذیری:** اصل کپسوله‌سازی در سی‌شارپ، کد ما را انعطاف‌پذیرتر می‌کند و به برنامه‌نویس اجازه می‌دهد تا به راحتی کد را تغییر دهد یا به‌روزرسانی کند.
5. **کنترل:** اصل کپسوله‌سازی کنترل بیشتری بر داده‌های ذخیره شده در متغیرها می‌دهد. برای مثال، می‌توانیم داده‌ها را با اعتبارسنجی اینکه آیا داده‌ها به اندازه کافی برای ذخیره در متغیر مناسب هستند یا خیر، کنترل کنیم.

کپسوله‌سازی یکی از چهار اصل اساسی برنامه‌نویسی شی‌گرا (OOP) است. این به دسته‌بندی داده‌ها (وضعیت) و روش‌ها (رفتارها) که روی داده‌ها عمل می‌کنند در یک واحد واحد و محدود کردن دسترسی به برخی از اجزای شیء اشاره دارد.
