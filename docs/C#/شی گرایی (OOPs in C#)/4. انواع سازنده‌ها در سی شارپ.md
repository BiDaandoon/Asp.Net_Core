## **انواع سازنده‌ها در سی شارپ به همراه مثال**

در این مقاله، قصد دارم **انواع سازنده‌ها در سی‌شارپ را** با مثال‌هایی مورد بحث قرار دهم.

##### **انواع سازنده‌ها در سی شارپ**

پنج نوع سازنده در سی شارپ وجود دارد که به شرح زیر هستند:

1. **سازنده پیش‌فرض یا بدون پارامتر**
2. **سازنده پارامتری**
3. **سازنده کپی**
4. **سازنده استاتیک**
5. **سازنده خصوصی**

بیایید هر یک از این سازنده‌ها را با جزئیات و مثال‌هایی بررسی کنیم.

##### **سازنده‌های پیش‌فرض یا بدون پارامتر در سی شارپ**

اگر یک متد سازنده هیچ پارامتری نگیرد، آن را سازنده پیش‌فرض یا بدون پارامتر می‌نامیم. این سازنده‌ها می‌توانند توسط یک برنامه‌نویس به صورت صریح تعریف شوند یا در غیر این صورت به صورت ضمنی تعریف می‌شوند، مشروط بر اینکه هیچ سازنده صریحی در زیر کلاس وجود نداشته باشد. بنابراین، سازنده‌های پیش‌فرض یا بدون پارامتر دوباره به دو نوع طبقه‌بندی می‌شوند. آنها به شرح زیر هستند:

1. **سازنده پیش‌فرض تعریف‌شده توسط سیستم**
2. **سازنده پیش‌فرض تعریف‌شده توسط کاربر**

##### **سازنده پیش‌فرض تعریف‌شده توسط سیستم یا سازنده بدون پارامتر در سی‌شارپ چیست؟**

به عنوان یک برنامه‌نویس، اگر هیچ سازنده‌ای به طور صریح در برنامه خود تعریف نشده باشد، کامپایلر به طور پیش‌فرض در زمان کامپایل یک سازنده ارائه می‌دهد. به آن سازنده، سازنده پیش‌فرض گفته می‌شود و سازنده پیش‌فرض بدون پارامتر است. سازنده پیش‌فرض، مقادیر پیش‌فرض را به اعضای داده (متغیرهای غیر استاتیک) اختصاص می‌دهد. از آنجایی که این سازنده توسط سیستم ایجاد می‌شود، به آن سازنده پیش‌فرض تعریف‌شده توسط سیستم نیز می‌گویند.

##### **مثالی برای درک سازنده ضمنی یا پیش‌فرض تعریف‌شده توسط سیستم در سی‌شارپ:**

در مثال زیر، درون کلاس Employee، چند متغیر ایجاد کرده‌ایم. و سپس از متد Main، یک نمونه از کلاس Employee ایجاد کرده و سپس مقادیر متغیرها را چاپ کرده‌ایم.

```csharp
using System;

namespace ConstructorDemo
{
    class Employee
    {
        public int Id, Age;
        public string Address, Name;
        public bool IsPermanent;
    }
    class Test
    {
        static void Main(string[] args)
        {
            Employee e1 = new Employee();
            Console.WriteLine("Employee Id is:  " + e1.Id);
            Console.WriteLine("Employee Name is:  " + e1.Name);
            Console.WriteLine("Employee Age is:  " + e1.Age);
            Console.WriteLine("Employee Address is:  " + e1.Address);
            Console.WriteLine("Is Employee Permanent:  " + e1.IsPermanent);
            Console.ReadKey();
        }
    }
}
```

بنابراین، وقتی کد بالا را اجرا می‌کنید، خواهید دید که مقادیر پیش‌فرض بر اساس نوع متغیر، همانطور که در تصویر زیر نشان داده شده است، روی کنسول چاپ می‌شوند. برای عددی (در مثال ما Id و Age)، مقدار پیش‌فرض ۰ است. برای رشته (در مثال ما Address و Name) یا شیء، مقدار پیش‌فرض null خواهد بود (در این حالت چیزی روی کنسول چاپ نمی‌شود) و برای نوع Boolean (در مثال ما IsPermanent)، مقدار پیش‌فرض false است.

![انواع سازنده‌ها در سی شارپ به همراه مثال](https://dotnettutorials.net/wp-content/uploads/2022/06/Types-of-Constructors-in-C-with-Examples.jpg "انواع سازنده‌ها در سی شارپ به همراه مثال")

در مثال ما، ما این مقادیر پیش‌فرض را مشخص نکرده‌ایم. پس چه کسی و چه زمانی این مقادیر پیش‌فرض را ارائه داده است؟ این مقادیر پیش‌فرض توسط سازنده پیش‌فرض بر اساس نوع داده متغیر ارائه می‌شوند و کامپایلر سازنده پیش‌فرض را در زمان کامپایل ارائه می‌دهد. بنابراین، کلاس Employee پس از کامپایل به صورت زیر تبدیل می‌شود.

![مثالی برای درک سازنده ضمنی یا پیش‌فرض تعریف‌شده توسط سیستم در سی‌شارپ](https://dotnettutorials.net/wp-content/uploads/2022/06/word-image-27825-2.png "مثالی برای درک سازنده ضمنی یا پیش‌فرض تعریف‌شده توسط سیستم در سی‌شارپ")

همانطور که می‌بینید، سازنده ضمنی که همیشه عمومی و بدون پارامتر است، متغیرها را با مقادیر پیش‌فرض مقداردهی اولیه می‌کند و این مقادیر پیش‌فرض را در خروجی مشاهده می‌کنید. از آنجایی که این سازنده ضمنی متغیرها را با مقادیر پیش‌فرض مقداردهی اولیه می‌کند، ما آن را سازنده پیش‌فرض تعریف‌شده توسط سیستم نیز می‌نامیم.

**نکته:** نکته‌ای که باید در نظر داشته باشید این است که کامپایلر فقط در صورتی سازنده پیش‌فرض را ارائه می‌دهد که به عنوان یک برنامه‌نویس، هیچ سازنده‌ای را به صراحت تعریف نکرده باشیم.

##### **چه زمانی باید سازنده را به صورت صریح ارائه دهیم؟**

اگر بخواهیم در زمان ایجاد شیء، منطق سفارشی را اجرا کنیم، آن منطق می‌تواند منطق مقداردهی اولیه شیء یا منطق مفید دیگری باشد، آنگاه به عنوان یک توسعه‌دهنده، باید سازنده را به صراحت در C# ارائه دهیم.

##### **سازنده پیش‌فرض تعریف‌شده توسط کاربر در سی‌شارپ چیست؟**

سازنده‌ای که توسط کاربر و بدون هیچ پارامتری تعریف می‌شود، سازنده پیش‌فرض تعریف‌شده توسط کاربر نامیده می‌شود. این سازنده هیچ آرگومانی نمی‌پذیرد، اما به عنوان بخشی از بدنه سازنده، می‌توانید منطق خود را بنویسید.

##### **مثالی برای درک سازنده پیش‌فرض تعریف‌شده توسط کاربر در سی‌شارپ**

در مثال زیر، درون کلاس Employee، یک سازنده‌ی بدون پارامتر عمومی ایجاد کرده‌ایم که برای مقداردهی اولیه متغیرها با برخی مقادیر پیش‌فرض hard-coded استفاده می‌شود. و سپس از متد Main، یک نمونه از کلاس Employee ایجاد کرده و متد Display را فراخوانی می‌کنیم.

```csharp
using System;

namespace ConstructorDemo
{
    class Employee
    {
        public int Id, Age;
        public string Address, Name;
        public bool IsPermanent;

        //User Defined Default Constructor
        public Employee()
        {
            Id = 100;
            Age = 30;
            Address = "Bhubaneswar";
            Name = "Anurag";
            IsPermanent = true;
        }

        public void Display()
        {
            Console.WriteLine("Employee Id is:  " + Id);
            Console.WriteLine("Employee Age is:  " + Age);
            Console.WriteLine("Employee Address is:  " + Address);
            Console.WriteLine("Employee Name is:  " + Name);
            Console.WriteLine("Is Employee Permanent:  " + IsPermanent);
        }
    }

    class Program
    {
        static void Main(string[] args)
        {
            Employee e1 = new Employee();
            e1.Display();

            Console.ReadKey();
        }
    }
}
```

اشکال سازنده پیش‌فرض تعریف‌شده توسط کاربر در بالا این است که هر نمونه (یعنی شیء) از کلاس با مجموعه مقادیر یکسانی مقداردهی اولیه (اختصاص داده) می‌شود. این بدان معناست که نمی‌توان هر نمونه از کلاس را با مقادیر مختلف مقداردهی اولیه کرد. برای درک بهتر، لطفاً متد Main را به صورت زیر تغییر دهید و خروجی را مشاهده کنید.

```csharp
class Program
{

    {
        Employee e1 = new Employee();
        e1.Display();

        Employee e2 = new Employee();
        Console.WriteLine();
        e2.Display();

        Console.ReadKey();
    }
}
```

##### **چه زمانی باید یک سازنده پارامتردار را در یک کلاس تعریف کنیم؟**

اگر بخواهیم شیء را به صورت پویا با مقادیر داده شده توسط کاربر مقداردهی اولیه کنیم یا اگر بخواهیم هر نمونه از یک کلاس را با مجموعه ای متفاوت از مقادیر مقداردهی اولیه کنیم، باید از سازنده پارامتری در C# استفاده کنیم. مزیت این روش این است که می‌توانیم هر نمونه را با مقادیر مختلف مقداردهی اولیه کنیم.

##### **سازنده پارامتری در سی شارپ چیست؟**

اگر یک متد سازنده با پارامترها تعریف شود، آن را در سی شارپ، سازنده پارامتری می‌نامیم و این سازنده‌ها فقط توسط برنامه‌نویسان تعریف می‌شوند اما هرگز نمی‌توانند به طور ضمنی تعریف شوند. بنابراین، به عبارت ساده، می‌توانیم بگوییم که سازنده‌ای که توسط توسعه‌دهنده با پارامترها داده می‌شود، در سی شارپ، سازنده پارامتری نامیده می‌شود.

بیایید با مثال‌هایی، سازنده پارامتری را در سی‌شارپ درک کنیم. لطفاً به کد زیر نگاهی بیندازید. در اینجا، ما کلاسی به نام ParameterizedConstructor داریم و این کلاس یک سازنده دارد که یک پارامتر عدد صحیح می‌گیرد. از آنجایی که این سازنده یک پارامتر می‌گیرد، آن را سازنده پارامتری می‌نامیم. و در این سازنده، مقدار i را چاپ می‌کنیم.

##### **چه زمانی باید از سازنده پارامتری در سی شارپ استفاده کنیم؟**

با کمک یک سازنده پارامتری، می‌توانیم هر نمونه از کلاس را با مجموعه‌ای متفاوت از مقادیر مقداردهی اولیه کنیم. این بدان معناست که با استفاده از سازنده پارامتری می‌توانیم مجموعه‌ای متفاوت از مقادیر را در نمونه‌های مختلف ایجاد شده در کلاس ذخیره کنیم.

بگذارید این را با یک مثال درک کنیم. لطفاً به کلاس زیر نگاهی بیندازید. این همان کلاسی است که در مثال قبلی با کمی تغییر روی آن کار کردیم. اکنون، در کلاس، متغیری به نام x تعریف کرده‌ام. به یاد داشته باشید، این متغیر x فقط با مقدار پیش‌فرض یعنی 0 مقداردهی اولیه شده است. چرا مقدار پیش‌فرض؟ زیرا ما مقداری اختصاص نداده‌ایم و وقتی مقداری اختصاص ندهیم، سازنده مسئولیت اختصاص مقدار را بر عهده می‌گیرد. و مقدار x برابر با 0 خواهد بود. سپس متدی به نام Display ایجاد کرده‌ایم که در آن مقدار x را چاپ می‌کنیم.

![چه زمانی باید از سازنده پارامتری در سی شارپ استفاده کنیم؟](https://dotnettutorials.net/wp-content/uploads/2022/06/word-image-27825-10.png "چه زمانی باید از سازنده پارامتری در سی شارپ استفاده کنیم؟")

حال، بیایید این متد Display را با استفاده از دو نمونه مختلف فراخوانی کنیم. کد کامل مثال در زیر آمده است.

```csharp
using System;

namespace ConstructorDemo
{
    class Program
    {
        static void Main(string[] args)
        {
            ParameterizedConstructor obj1 = new ParameterizedConstructor(10);
            obj1.Display();

            ParameterizedConstructor obj2 = new ParameterizedConstructor(20);
            obj2.Display();

            Console.ReadKey();
        }
    }

    public class ParameterizedConstructor
    {
        int x;
        public ParameterizedConstructor(int i)
        {
            Console.WriteLine($"Parameterized Constructor is Called : {i}");
        }
        public void Display()
        {
            Console.WriteLine($"Value of X = {x}");
        }
    }
}
```

```csharp
            ParameterizedConstructor obj1 = new ParameterizedConstructor(10);
            obj1.Display();

    }

    public class ParameterizedConstructor
```

```csharp
using System;

namespace ConstructorDemo
{

    {
        static void Main(string[] args)
        {
            CopyConstructor obj1 = new CopyConstructor(10);
```

```csharp
            CopyConstructor obj2 = new CopyConstructor(obj1);
            obj2.Display();

            Console.ReadKey();

    public class CopyConstructor
    {
        int x;

        public CopyConstructor(int i)

            x = i;

        {
            x = obj.x;
        }

        public void Display()
        {
            Console.WriteLine($"Value of X = {x}");
        }
    }
}
```

##### **نکاتی که باید هنگام کار با سازنده‌های استاتیک در سی شارپ به خاطر داشته باشید:**

###### **نکته ۱:**

اگر یک کلاس شامل متغیرهای استاتیک باشد، فقط سازنده‌های استاتیک ضمنی وارد عمل می‌شوند، در غیر این صورت باید آنها را به طور صریح تعریف کنیم. از سوی دیگر، سازنده‌های غیر استاتیک به طور ضمنی در هر کلاسی (به جز کلاس استاتیک) تعریف می‌شوند، مشروط بر اینکه هیچ سازنده‌ای را به طور صریح تعریف نکرده باشیم.

###### **نکته ۲:**

سازنده‌های استاتیک مسئول مقداردهی اولیه متغیرهای استاتیک هستند و این سازنده‌ها هرگز به طور صریح فراخوانی نمی‌شوند. آنها به طور ضمنی فراخوانی می‌شوند و علاوه بر این، این سازنده‌ها اولین سازنده‌هایی هستند که در هر کلاسی اجرا می‌شوند. برای درک بهتر، لطفاً به مثال زیر نگاهی بیندازید. در اینجا، ما یک سازنده استاتیک تعریف کرده‌ایم و لطفاً توجه داشته باشید که از متد Main، سازنده استاتیک را فراخوانی نمی‌کنیم.

```csharp
using System;

namespace ConstructorDemo
{
    public class StaticConstructor
    {
        static StaticConstructor()
        {
            Console.WriteLine("Static Constructor Executed!");
        }
        static void Main(string[] args)
        {
            Console.WriteLine("Main Method Exceution Started...");
            Console.ReadKey();
        }
    }
}
```

حالا، وقتی کد بالا را اجرا می‌کنید، سازنده‌ی Static ابتدا اجرا می‌شود و سپس متد main. و این خروجی زیر را اثبات می‌کند.

سازنده‌های غیراستاتیک هرگز به طور ضمنی فراخوانی نمی‌شوند، آنها همیشه به طور صریح فراخوانی می‌شوند در حالی که سازنده‌های استاتیک هرگز به طور صریح فراخوانی نمی‌شوند، آنها همیشه به طور ضمنی فراخوانی می‌شوند. اجرا چگونه اتفاق می‌افتد؟ ببینید، متد Main نقطه شروع اجرا است و در این مورد تفاوتی ندارد. اجرای برنامه از متد Main شروع می‌شود اما قبل از اجرای هر دستوری در داخل متد Main، ابتدا سازنده Static اجرا می‌شود و پس از اتمام اجرای سازنده Static، اجرای متد Main ادامه می‌یابد. بنابراین، سازنده Static اولین بلوک کد در یک کلاس است که اجرا می‌شود.

###### **نکته ۳:**

سازنده‌های استاتیک را نمی‌توان پارامتردهی کرد، بنابراین بارگذاری بیش از حد سازنده‌های استاتیک در سی‌شارپ امکان‌پذیر نیست. حال، سوال این است که چرا نمی‌توانیم سازنده استاتیک را پارامتردهی کنیم؟ پاسخ ساده است. سازنده‌های استاتیک به صورت ضمنی اجرا می‌شوند و از این رو ما هرگز فرصتی برای ارسال مقدار نداریم. و از آنجایی که محدودکننده استاتیک اولین بلوکی است که در یک کلاس اجرا می‌شود، بنابراین فرصتی برای ارسال مقدار وجود ندارد.

##### **سازنده خصوصی در سی شارپ چیست؟**

در سی شارپ، امکان ایجاد یک سازنده به صورت خصوصی (private) نیز وجود دارد. سازنده‌ای که دسترسی به آن خصوصی است، به عنوان سازنده خصوصی (private constructor) شناخته می‌شود. وقتی یک کلاس شامل یک سازنده خصوصی باشد، نمی‌توانیم برای کلاس خارج از کلاس، شیء ایجاد کنیم. بنابراین، از سازنده‌های خصوصی برای ایجاد یک شیء برای کلاس درون همان کلاس استفاده می‌شود. به طور کلی، از سازنده‌های خصوصی در مفهوم Remoting استفاده می‌شود.

##### **مثال برای درک سازنده خصوصی در سی شارپ**

```csharp
using System;

namespace ConstructorDemo
{
    class Program
    {
        private Program()
        {
            Console.WriteLine("This is private constructor");
        }
        static void Main(string[] args)
        {
            Program p = new Program();
            Console.WriteLine("Main method");
            Console.ReadKey();
        }
    }
}
```

##### **نکاتی که باید در مورد سازنده خصوصی در سی شارپ به خاطر داشته باشید:**

1. با استفاده از سازنده خصوصی در سی شارپ می‌توانیم الگوی طراحی سینگلتون را پیاده‌سازی کنیم.
2. وقتی کلاس فقط شامل اعضای استاتیک باشد، باید از سازنده خصوصی در سی شارپ استفاده کنیم.
3. با استفاده از سازنده خصوصی نمی‌توان از خارج از کلاس، نمونه‌ای ایجاد کرد.
