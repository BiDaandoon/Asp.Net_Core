
# **بهترین روش‌های استاندارد کدنویسی:**

استانداردهای کدنویسی برای حفظ کیفیت، خوانایی و قابلیت نگهداری نرم‌افزار حیاتی هستند. در ادامه برخی از بهترین روش‌های استاندارد کدنویسی که معمولاً هنگام نوشتن کد برای پروژه‌هایم رعایت می‌کنم آورده شده است:

- یکپارچگی (Consistency)
- خوانایی (Readability)
- کامنت‌گذاری و مستندسازی (Commenting and Documentation)
- بهترین روش‌های کنترل نسخه (Version Control Best Practices)
- مدیریت خطا (Error Handling)
- پرهیز از بهینه‌سازی زودهنگام (Avoid Premature Optimization)
- تست (Testing)
- روش‌های امنیتی (Security Practices)
- بازآرایی کد (Refactoring)
- بازبینی کد (Code Reviews)
- استفاده از چارچوب‌ها و استانداردها (Use Frameworks and Standards)
- یکپارچه‌سازی مداوم/تحویل مداوم (CI/CD)

## **یکپارچگی (Consistency):**

قراردادهای نام‌گذاری یکسان، ساختارهای فایل و سبک‌های کدنویسی، پایگاه کد را برای خواندن و فهم آسان‌تر می‌کند.

- هدف: اطمینان از این‌که هر کسی در تیم یا خارج از آن که کد را می‌خواند بتواند پروژه را سریع بفهمد و در آن پیمایش کند.
- پیاده‌سازی: برای مثال، می‌توانید بین camelCase، PascalCase و snake_case یکی را انتخاب کنید و در کل پروژه به آن پایبند بمانید.

مثال: در یک پروژهٔ دات‌نت، به‌طور یکسان از PascalCase برای متدها و ویژگی‌های عمومی و از camelCase برای متغیرهای محلی و پارامترها استفاده کنید.

```csharp
public class User
{
    public string FirstName { get; set; }
    private void UpdateUserDetails(string userName)
    {
        string updatedName = userName;
    }
}
```

## **خوانایی (Readability):**

نوشتن کدی که خواندنش آسان باشد، به اندازهٔ کدی که کار می‌کند اهمیت دارد. این شامل استفاده از نام‌های متغیر و تابع بامعنا، پرهیز از عبارات بیش‌ازحد پیچیده، و افزودن فاصله‌گذاری و تورفتگی که به جدا شدن بصری بلوک‌های کد کمک می‌کند، می‌شود.

- هدف: کد را برای فهم آسان‌تر می‌کند و زمان لازم برای آشنایی دیگران با آن را کاهش می‌دهد.
- پیاده‌سازی: مسائل پیچیده را به توابع کوچک‌تر و قابل‌مدیریت بشکنید. از نام‌های واضح و توصیفی برای متغیرها و توابع استفاده کنید.

مثال: در سی‌شارپ از اعضای با بدنهٔ عبارتی (expression-bodied members) برای متدها یا ویژگی‌های ساده استفاده کنید تا خوانایی افزایش یابد.

```csharp
public class Calculator
{
    // استفاده از اعضای با بدنهٔ عبارتی برای توابع ساده
    public int Add(int x, int y) => x + y;
    public int Multiply(int x, int y) => x * y;
}
```

## **کامنت‌گذاری و مستندسازی (Commenting and Documentation):**

کامنت‌ها و مستندات باید توضیح دهند «چرا» کاری انجام شده، نه «چگونه». کامنت‌های خوب کد را تکرار نمی‌کنند، بلکه هدف آن را روشن می‌سازند. مستندات باید شامل دستورالعمل‌های راه‌اندازی، جزئیات استفاده، و توضیح هدف و سازوکار کد باشد.

- هدف: به نگهداری بلندمدت کد کمک می‌کند، به‌ویژه زمانی که توسعه‌دهندگان اصلی دیگر در دسترس نباشند.
- پیاده‌سازی: کامنت‌هایی بنویسید که ویژگی‌های غیرآشکار یا دلایل انتخاب‌های خاص کدنویسی را توضیح دهند. یک فایل README به‌روز و سایر فایل‌های مستندات که یک نمای کلی از پروژه ارائه می‌دهند را نگهداری کنید.

مثال: در سی‌شارپ از کامنت‌های مستندات XML برای متدها استفاده کنید تا هدف، پارامترها و نوع بازگشتی آن‌ها را توضیح دهید.

```csharp
/// <summary>
/// یک ماشین‌حساب با عملیات پایه‌ای حسابی را نمایش می‌دهد.
/// </summary>
public class Calculator
{
    /// <summary>
    /// دو عدد صحیح را جمع کرده و نتیجه را برمی‌گرداند.
    /// </summary>
    /// <param name="x">عدد اول.</param>
    /// <param name="y">عدد دوم.</param>
    /// <returns>مجموع x و y.</returns>
    public int Add(int x, int y)
    {
        return x + y;
    }
}
```

## **بهترین روش‌های کنترل نسخه (Version Control Best Practices):**

از سیستم‌های کنترل نسخه مانند Git و TFS به‌طور مؤثر استفاده کنید. این شامل پیام‌های commit بامعنا، ادغام مکرر و نگه‌داشتن شاخه‌ها در حداقل هدفمند است.

- هدف: ثبت سابقهٔ تمام تغییرات، امکان بازگشت به نسخه‌های پیشین و پشتیبانی از محیط‌های کاری مشارکتی.
- پیاده‌سازی: از شاخه‌ها برای ویژگی‌های خاص یا رفع باگ استفاده کنید، تغییرات را مرتباً ادغام کنید تا از تداخل جلوگیری شود و پیام‌های commit واضح و توصیفی بنویسید.

مثال: راه‌حل‌های دات‌نت را در پروژه‌هایی با وابستگی‌های مشخص سازمان‌دهی کنید و تغییرات را با پیام‌های بامعنا ثبت کنید.

```
git commit -m "Refactor UserService to improve dependency injection."
```

## **مدیریت خطا (Error Handling):**

به‌جای نادیده گرفتن خطاها، آن‌ها را به‌درستی مدیریت کنید. پیام‌های خطای بامعنا ارائه دهید و اطمینان حاصل کنید که شکست‌ها باعث آسیب بیشتر به‌صورت پنهان نشوند.

- هدف: با مدیریت موقعیت‌های غیرمنتظره، قابلیت اطمینان و کاربردپذیری نرم‌افزار را بهبود می‌بخشد.
- پیاده‌سازی: از بلوک‌های try-catch در جایی که ممکن است استثنا رخ دهد استفاده کنید و استثناهای سفارشی بامعنا برای مدیریت مؤثر سناریوهای مختلف خطا ایجاد کنید.

مثال: در سی‌شارپ از فیلتر کردن استثنا برای مدیریت دقیق‌تر خطا استفاده کنید.

```csharp
try
{
    ProcessFile(filePath);
}
catch (IOException e) when (e.Message.Contains("permission"))
{
    Console.WriteLine("هنگام پردازش فایل، خطای دسترسی رخ داد.");
}
catch (IOException e)
{
    Console.WriteLine("خطای ورودی/خروجی رخ داد.");
}
```

## **پرهیز از بهینه‌سازی زودهنگام (Avoid Premature Optimization):**

تنها پس از آن‌که مشخص شد گلوگاه‌های عملکرد کجا هستند، کد را بهینه‌سازی کنید و حتی در آن صورت نیز تأثیر بهینه‌سازی‌ها را اندازه‌گیری کنید.

- هدف: تلاش توسعه را بر ایجاد کد کاربردی و تمیز متمرکز می‌کند، نه بهینه‌سازی بخش‌های بالقوه نامربوط سیستم.
- پیاده‌سازی: قبل از انجام بهینه‌سازی، برنامه را پروفایل کنید تا گلوگاه‌ها را شناسایی کنید. اطمینان حاصل کنید که خوانایی و قابلیت نگهداری به‌خاطر افزایش اندک عملکرد قربانی نشود.

مثال: در سی‌شارپ با یک کوئری LINQ ساده شروع کنید و در صورت بروز مشکلات عملکردی بعداً آن را بهینه کنید.

```csharp
var result = context.Users.Where(u => u.Age > 30).ToList();
```

## **تست (Testing):**

تست خودکار باید بخشی جدایی‌ناپذیر از فرآیند توسعه باشد. این شامل تست‌های واحد، تست‌های یکپارچگی و احتمالاً تست‌های عملکردی و انتها به انتها، بسته به اندازه و ماهیت پروژه می‌شود.

- هدف: اطمینان از این‌که کد همان‌طور که انتظار می‌رود کار می‌کند و تغییرات آینده، عملکرد موجود را خراب نمی‌کنند.
- پیاده‌سازی: مجموعهٔ جامعی از تست‌ها (واحد، یکپارچگی و سیستم) تهیه کنید. این تست‌ها را به‌طور منظم با استفاده از چارچوب‌های اتوماسیون تست اجرا کنید.

مثال: در یک پروژهٔ دات‌نت از NUnit برای نوشتن تست‌های واحد استفاده کنید. نوشتن تست‌های واحد برای کلاس Calculator

```csharp
using NUnit.Framework;
[TestFixture]
public class CalculatorTests
{
    private Calculator _calculator;
    [SetUp]
    public void SetUp()
    {
        _calculator = new Calculator();
    }
    [Test]
    public void Add_WhenCalled_ReturnSumOfArguments()
    {
        var result = _calculator.Add(1, 2);
        Assert.AreEqual(3, result);
    }
}
```

## **روش‌های امنیتی (Security Practices):**

همیشه پیامدهای امنیتی کد خود را در نظر بگیرید. با پیروی از بهترین روش‌ها و دستورالعمل‌های امنیتی، می‌توانید از آسیب‌پذیری‌های رایج مانند تزریق SQL، XSS و CSRF در برنامه‌های وب جلوگیری کنید.

- هدف: از برنامه در برابر آسیب‌پذیری‌ها و حملات محافظت می‌کند که برای حفظ اعتماد کاربران و تطابق با مقررات حیاتی است.
- پیاده‌سازی: شیوه‌های کدنویسی امن را دنبال کنید، مانند اعتبارسنجی ورودی‌ها، استفاده از دستورات آماده برای کوئری‌های SQL و پیروی از اصل کمترین امتیاز.

مثال: پرس‌وجوی امن داده‌ها با استفاده از Entity Framework

```csharp
public class UserRepository : IUserRepository
{
    private readonly MyAppContext _context;
    public UserRepository(MyAppContext context)
    {
        _context = context;
    }
    public async Task<User> GetUserByUsernameAsync(string username)
    {
        return await _context.Users
            .FirstOrDefaultAsync(u => u.Username == username);
    }
}
```

## **بازآرایی کد (Refactoring):**

کد را به‌طور منظم بازآرایی کنید تا ساختار داخلی آن را بدون تغییر رفتار بیرونی بهبود بخشید. این کمک می‌کند پایگاه کد تمیز و سازگار بماند.

- هدف: پایگاه کد را تمیز و سازگار نگه می‌دارد که به‌روزرسانی‌ها و بهبودها را ساده می‌کند.
- پیاده‌سازی: به‌طور منظم کد را بررسی و بازسازی کنید تا طراحی و مشکلات عملکردی بهبود یابد. از ابزارهای بازآرایی که معمولاً در IDEهای مدرن تعبیه شده‌اند استفاده کنید تا اطمینان حاصل شود تغییرات، عملکرد را تغییر نمی‌دهند.

مثال: کد تکراری را در یک برنامهٔ دات‌نت به یک متد یا سرویس مشترک بازآرایی کنید.

```csharp
public void LogOrder(Order order)
{
    LogInfo("Order processed", order.Id);
}
public void LogError(string message)
{
    LogInfo("Error", message);
}
private void LogInfo(string type, string message)
{
    logger.Log($"{type}: {message}");
}
```

## **بازبینی کد (Code Reviews):**

بازبینی‌های همتا از کد، همکاری را تشویق می‌کند، کیفیت کد را افزایش می‌دهد و دانش را در سراسر تیم به اشتراک می‌گذارد.

- هدف: کیفیت کلی کد را از طریق نظارت همتا افزایش می‌دهد و اشتراک‌گذاری دانش تیم را تشویق می‌کند.
- پیاده‌سازی: برای ادغام کد از درخواست‌های pull استفاده کنید، جلسات منظم بازبینی کد برگزار کنید و بازخورد سازنده را تشویق کنید.

مثال: در پروژه‌های دات‌نت از درخواست‌های pull در Azure DevOps یا GitHub برای تسهیل بازبینی کد استفاده کنید و اطمینان حاصل کنید که هر تغییر توسط عضو دیگری از تیم بازبینی می‌شود.

## **استفاده از چارچوب‌ها و استانداردها (Use Frameworks and Standards):**

هر جا قابل‌اجرا است، از چارچوب‌ها، کتابخانه‌ها و استانداردهای موجود استفاده کنید. آن‌ها اغلب مسائل رایج را به روشی آزمایش‌شده و کارآمد حل می‌کنند.

- هدف: با استفاده از راه‌حل‌های آزمایش‌شده توسط جامعه و کتابخانه‌های رایج، در زمان صرفه‌جویی می‌کند و خطاها را کاهش می‌دهد.
- پیاده‌سازی: چارچوب‌های با مستندات خوب را انتخاب کنید که با نیازهای پروژه متناسب باشند و از قراردادها و الگوهای توصیه‌شده توسط آن چارچوب‌ها پیروی کنید.

مثال: برای برنامه‌های وب از ASP.NET Core استفاده کنید تا از طراحی چارچوب مقاوم، ماژولار و قابل تست آن بهره ببرید.

## **یکپارچه‌سازی مداوم/تحویل مداوم (CI/CD):**

خطوط لولهٔ CI/CD را برای خودکارسازی فرآیندهای تست و استقرار بگنجانید تا اطمینان حاصل شود که کد می‌تواند در هر زمان به‌طور قابل‌اطمینان منتشر شود.

- هدف: فرآیند یکپارچه‌سازی کد جدید در مخزن اصلی و استقرار در محیط عملیاتی را ساده می‌کند و وظایف دستی و خطای انسانی را به حداقل می‌رساند.
- پیاده‌سازی: خطوط لولهٔ خودکار برای ساخت، تست و استقرار برنامه‌هایتان تنظیم کنید. این خطوط لوله را با استفاده از ابزارهایی مانند Jenkins، CircleCI یا GitHub Actions مدیریت کنید.

لطفاً بهترین روش‌های استاندارد کدنویسی فوق را در پروژهٔ خود دنبال کنید و ببینید چگونه می‌توانند کیفیت، خوانایی و قابلیت نگهداری نرم‌افزار را بهبود بخشند. امیدوارم از این مقالهٔ بهترین روش‌های استاندارد کدنویسی لذت ببرید.
