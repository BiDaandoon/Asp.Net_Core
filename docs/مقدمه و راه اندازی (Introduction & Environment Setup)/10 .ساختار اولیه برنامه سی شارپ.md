##### **بخش فضای نام را وارد کنید:**

این بخش شامل دستورات ورودی است که برای وارد کردن BCL (کتابخانه‌های کلاس پایه) و همچنین در صورت نیاز، فضاهای نام تعریف شده توسط کاربر استفاده می‌شوند. این مشابه دستورات ورودی در زبان برنامه‌نویسی C است. فرض کنید می‌خواهید از برخی کلاس‌ها و رابط‌ها در کد خود استفاده کنید، باید فضای نامی را که این کلاس‌ها و رابط‌ها از آنجا تعریف شده‌اند، وارد کنید. به عنوان مثال، اگر قصد دارید از کلاس Console در کد خود استفاده کنید، باید فضای نام System را وارد کنید زیرا کلاس Console متعلق به فضای نام System است.  
**نحو: با استفاده از NamespaceName;**  
**مثال: با استفاده از System;**

اگر فضای نام مورد نظر عضوی از فضای نام دیگری باشد، باید فضای نام‌های والد و فرزند را که با نقطه از هم جدا شده‌اند، به صورت زیر مشخص کنیم:  
**با استفاده از System.Data؛**  
**با استفاده از System.IO؛**

**نکته:** یک فضای نام، ظرفی است که شامل گروهی از کلاس‌ها و رابط‌های مرتبط است، همچنین یک فضای نام می‌تواند شامل فضاهای نام دیگری نیز باشد.

##### **بخش اعلان فضای نام:**

در اینجا یک فضای نام تعریف شده توسط کاربر تعریف شده است. در برنامه‌های .NET، تمام کلاس‌ها و رابط‌ها یا هر نوع مربوط به پروژه باید درون یک فضای نام تعریف شوند. به طور کلی، ما تمام کلاس‌های مرتبط را تحت یک فضای نام قرار می‌دهیم و در یک پروژه، می‌توانیم چندین فضای نام ایجاد کنیم.  
**نحو: فضای نام NamespaceName {}**  
**مثال: فضای نام MyFirstProject {}**

معمولاً نام فضای نام با نام پروژه یکسان است، اما اجباری نیست، می‌توانید هر نام دلخواهی را به فضای نام بدهید.

##### **بخش اعلان کلاس:**

برای هر برنامه دسکتاپ در .NET، به یک فایل کلاس راه‌اندازی نیاز داریم. به عنوان مثال، برای هر برنامه دسکتاپ .NET مانند Console و Windows، باید یک کلاس Start-Up وجود داشته باشد که متد Main را داشته باشد و اجرای برنامه از آنجا شروع شود. وقتی با استفاده از Visual Studio یک برنامه Console ایجاد می‌کنیم، Visual Studio به طور پیش‌فرض فایل کلاس Start-Up را با نام Program.cs ایجاد می‌کند که دارای کلاسی با نام Program خواهد بود که شامل متد Main است. یک کلاس راه‌اندازی چیزی نیست جز کلاسی که شامل یک متد Main() است که اجرای برنامه از آنجا شروع می‌شود.

Syntax:  
class ClassName  
{  
}

Example:  
class Program  
{  
}

**نکته:** شما می‌توانید این رفتار پیش‌فرض را تغییر دهید. می‌توانید کلاس خودتان را ایجاد کنید و همچنین می‌توانید با اضافه کردن متد Main، آن کلاس را به عنوان کلاس Start-Up تعریف کنید. در مقالات بعدی، به تفصیل در مورد این موضوع صحبت خواهیم کرد.

##### **بخش متد ()Main:**

متد main() نقطه ورود یا نقطه شروع برنامه برای شروع اجرای آن است. وقتی برنامه شروع به اجرا می‌کند، متد main اولین بلوک برنامه خواهد بود که اجرا می‌شود. متد Main شامل منطق اصلی برنامه است.

##### **از چی استفاده میشه؟**

استفاده کردن یک کلمه کلیدی است. با استفاده از این کلمه کلیدی، می‌توانیم به .NET BCL در برنامه‌های C# اشاره کنیم، یعنی فضاهای نام BCL را نیز اضافه کنیم، همچنین می‌توانیم فضاهای نام تعریف شده توسط کاربر را نیز اضافه کنیم که در ادامه این دوره در مورد آنها بحث خواهیم کرد. جدا از وارد کردن فضای نام، کاربردهای دیگری از دستورات using وجود دارد که در ادامه این دوره در مورد آنها نیز بحث خواهیم کرد. فعلاً همین کافی است.

**نکته:** در دات‌نت، کتابخانه‌های کلاس پایه به مجموعه‌ای از فضاهای نام تقسیم می‌شوند. هر فضای نام شامل مجموعه‌ای از کلاس‌ها و زیرفضاهای نام از پیش تعریف شده است. فضای نام شامل فضای نام دیگری به نام زیرفضاهای نام است.

1. این فریمورک ویژگی‌های رابط کاربری گرافیکی (GUI) را ارائه می‌دهد. زبان‌های برنامه‌نویسی قدیمی‌تر مانند C و C++ از ویژگی‌های رابط کاربری گرافیکی (GUI) پشتیبانی نمی‌کنند، اما C#.NET ویژگی‌های کامل رابط کاربری گرافیکی (GUI) را ارائه می‌دهد. تمام ویژگی‌های رابط کاربری گرافیکی از این فریمورک گرفته می‌شوند.
2. ما می‌توانیم به هر پایگاه داده‌ای متصل شویم و عملیات را انجام دهیم. با استفاده از فناوری‌های ADO.NET و Entity Framework، می‌توانیم عملیات پایگاه داده را با هر پایگاه داده‌ای انجام دهیم. ADO.NET و Entity Framework همچنین بخشی از چارچوب .NET هستند.
3. این چارچوب همچنین به ما در توسعه برنامه‌های مبتنی بر وب کمک می‌کند. با استفاده از فناوری ASP.NET می‌توانیم برنامه‌های مبتنی بر وب توسعه دهیم. ASP.NET به تنهایی نمی‌تواند برنامه‌های وب را توسعه دهد؛ این زبان به پشتیبانی زبان نیاز دارد. بنابراین، در اینجا می‌توانیم از C# به عنوان زبان برنامه‌نویسی استفاده کنیم. ASP.NET نیز بخشی از این چارچوب است.
