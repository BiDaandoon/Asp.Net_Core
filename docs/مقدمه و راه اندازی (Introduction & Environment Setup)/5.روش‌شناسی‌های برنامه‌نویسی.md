
در این مقاله قصد دارم روش‌شناسی‌های برنامه‌نویسی را به طور کامل بررسی کنم. لطفاً ابتدا مقالهٔ قبلی ما را مطالعه کنید که در آن انواع مختلف برنامه‌ها را به تفصیل شرح داده‌ایم. در پایان این مقاله، عمدتاً خواهید فهمید که برنامه‌نویسی یکپارچه (Monolithic) چیست، برنامه‌نویسی ماژولار/رویه‌ای (Modular/Procedural) چیست و برنامه‌نویسی شیءگرا (Object-Oriented) چیست.

**روش‌شناسی‌های برنامه‌نویسی**

پارادایم‌های برنامه‌نویسی، سبک‌ها/روش‌های نوشتن کد در یک برنامه هستند. بسته به نیازها، سبک‌های برنامه‌نویسی گوناگونی به کار گرفته شده‌اند. از آغاز پیدایش زبان‌های برنامه‌نویسی، سبک‌های متفاوتی استفاده شده است و ما در این مقاله به بررسی چند روش شناخته‌شده خواهیم پرداخت.

- برنامه‌نویسی یکپارچه (Monolithic)
- برنامه‌نویسی ماژولار/رویه‌ای (Modular/Procedural)
- برنامه‌نویسی شیءگرا (Object-Oriented)

در عصر امروز، ما از سبک برنامه‌نویسی شیءگرا پیروی می‌کنیم، اما در آغاز از رویکرد یکپارچه استفاده می‌کردیم. پس بیایید ببینیم چه تغییراتی به‌تدریج میان آن‌ها رخ داده است.

**برنامه‌نویسی یکپارچه (Monolithic Programming):**

در برنامه‌نویسی یکپارچه، کل کد برنامه در یک بلوک واحد و بدون هیچ‌گونه تقسیم کد به توابع قرار دارد. برنامه‌نویسی یکپارچه به این صورت تعریف می‌شود که کل برنامه در یک تابع واحد (یعنی تابع اصلی) نوشته شود. یک فرد می‌تواند این سبک برنامه‌نویسی را به‌تنهایی بنویسد و نگهداری کند، درست همان‌طور که یک هنرمند می‌تواند یک نقاشی را خلق کند. برای مثال: در این مثال، یک کد نمونه به سبک برنامه‌نویسی یکپارچه نوشته‌ایم.

```csharp
static void Main(string[] args)
{
    int i, n;
    bool IsPrime = true;
    Console.WriteLine("یک عدد صحیح مثبت وارد کنید");
    n = Convert.ToInt32(Console.ReadLine());
    // 0 و 1 عدد اول نیستند
    if(n == 0 || n==1)
    {
        IsPrime = false;
    }
    else
    {
        for(i=2; i <= n/2; ++i)
        {
            if(n % i == 0)
            {
                IsPrime = false;
                break;
            }
        }
    }
    if(IsPrime)
    {
        Console.WriteLine(n + " یک عدد اول است");
    }
    else
    {
        Console.WriteLine(n + " یک عدد اول نیست");
    }
}
```

**مزایا**

- زبان برنامه‌نویسی و نحوهٔ کارکرد آن را تعریف می‌کند.
- کدنویسی در پارادایم یکپارچه آسان‌تر و سریع‌تر است.
- سادگی کد بزرگ‌ترین مزیت آن است.

**معایب سبک برنامه‌نویسی یکپارچه:**

- برنامه بسیار بزرگ و پیچیده به نظر می‌رسد.
- اشکال‌زدایی، تست و نگهداری برنامه بسیار دشوار است.
- در هر لحظه فقط یک نفر می‌تواند برنامه را بنویسد. در نتیجه، توسعهٔ پروژه ممکن است زمان زیادی ببرد. کد قابل تقسیم به بخش‌ها نیست؛ بنابراین توسعه دشوار می‌شود.
- در برنامه‌نویسی یکپارچه، داده‌ها و دستورالعمل‌ها در یک بلوک واحد قرار دارند و از این رو کد شلوغ و طولانی می‌شود.
- افزونگی یا تکرار کد مشابه در چندین جا یک عیب بزرگ است.
- استفادهٔ مجدد از کد ممکن نیست.

**برنامه‌نویسی ماژولار/رویه‌ای (Modular/Procedural Programming):**

در این سبک برنامه‌نویسی، کد در بلوک‌های جداگانه نوشته می‌شود و نه در یک بلوک واحد، به‌طوری‌که می‌توان از آن‌ها در برنامه هرگاه نیاز باشد استفاده کرد، بدون این‌که لازم باشد کل قطعه کد دوباره نوشته شود. در اینجا، مجموعه‌ای مشخص از کد به شکل توابع یا ماژول‌هایی وجود دارد که می‌توان هر جا لازم شد از آن‌ها استفاده کرد. از این رو به آن برنامه‌نویسی ماژولار گفته می‌شود. محبوب‌ترین زبان برنامه‌نویسی در برنامه‌نویسی ماژولار، زبان C است.

برنامه‌نویسی ماژولار به تقسیم یک برنامه به تعدادی بخش‌های کاربردی اشاره دارد. اگر برنامهٔ اصلی به زیربرنامه‌هایی تقسیم شود، می‌توانیم هر زیرماژول را جداگانه کدنویسی کنیم و سپس آن‌ها را در یک واحد یکپارچه ادغام کنیم. ماژول‌های مجزای این چنینی، توابع نامیده می‌شوند. برای مثال: در این مثال، یک کد نمونه به سبک ماژولار/رویه‌ای نوشته‌ایم.

```csharp
static void Main(string[] args)
{
    int n;
    Console.WriteLine("یک عدد صحیح مثبت وارد کنید");
    n = Convert.ToInt32(Console.ReadLine());
    
    bool isPrime = IsPrime(n);
    if (isPrime)
    {
        Console.WriteLine(n + " یک عدد اول است");
    }
    else
    {
        Console.WriteLine(n + " یک عدد اول نیست");
    }
}
private static bool IsPrime(int n)
{
    bool IsPrime = true;
    // 0 و 1 عدد اول نیستند
    if (n == 0 || n == 1)
    {
        IsPrime = false;
    }
    else
    {
        for (int i = 2; i <= n / 2; ++i)
        {
            if (n % i == 0)
            {
                IsPrime = false;
                break;
            }
        }
    }
    return IsPrime;
}
```

**مزایای برنامه‌نویسی ماژولار:**

- با تقسیم یک برنامه به برنامه‌های کوچک‌تر برای اجرای وظایف گوناگون، توسعهٔ آسان نرم‌افزار و برنامه را ممکن می‌سازد.
- خوانایی کد را افزایش می‌دهد و فهم کد را آسان‌تر می‌کند.
- قابلیت استفادهٔ مجدد از کد را بالا می‌برد، به طوری که یک قطعه کد مشخص می‌تواند چندین بار در برنامه هرگاه نیاز باشد استفاده شود.
- می‌توان از آن برای نوشتن برنامه‌های بزرگ‌تر و پیچیده‌تر استفاده کرد.
- تیمی از برنامه‌نویسان می‌توانند هم‌زمان کار کنند و بنابراین پروژه‌های بزرگ به‌راحتی انجام می‌شوند.
- فهم برنامه آسان است.
- اشکال‌زدایی و نگهداری آسان می‌شود.

مثال: فرض کنید یک رویداد در دانشگاه برگزار می‌شود که شامل کارهای زیادی مانند تدارکات است؛ بنابراین می‌توانید کارها را بین تیم‌های مختلف تقسیم کنید و رویداد به‌راحتی مدیریت می‌شود. به طور مشابه، کل کار (کد) بین توابع مختلف تقسیم می‌شود و برنامه به‌راحتی اجرا می‌گردد.

اهمیت تابع اصلی: تابع اصلی، مدیر کل کد برنامه است و مدیریت می‌کند که کدام بلوک کد چه زمانی اجرا شود. این شبیه به مدیر یک فروشگاه است که تمام کارهای انجام‌شده توسط افراد مختلف در کسب‌وکار را مدیریت می‌کند.

**کلاس‌ها در برنامه‌نویسی شیءگرا**

- کلاس – کلاس روشی برای نوشتن کد سازمان‌یافته و ساختاریافته است که در آن داده‌ها و توابع مرتبط با آن‌ها با یکدیگر درون کلاس محصور می‌شوند.
- می‌توان اشیایی از نوع کلاس ساخت و از آن‌ها برای انجام وظایف گوناگون استفاده کرد.
- روش برنامه‌نویسی با استفاده از کلاس‌ها، روش برنامه‌نویسی شیءگرا نامیده می‌شود.

**برنامه‌نویسی شیءگرا (Object-Oriented Programming):**

در برنامه‌نویسی شیءگرا، کد به شکل کلاس‌هایی نوشته می‌شود که شامل اعضای داده و توابع عضو مختلف هستند؛ داده‌ها و توابع مرتبط با آن‌ها با هم گروه‌بندی می‌شوند. همچنین می‌توان با استفاده از نوع کلاس، اشیاء ساخت.

می‌دانیم که در رویکرد ماژولار، داده‌ها و توابع به صورت موجودیت‌های جداگانه هستند؛ یعنی توابع می‌توانند از داده‌های بیرونی استفاده کنند. اما در سبک شیءگرا، می‌توانیم داده و تابع را با هم به عنوان یک موجودیت واحد داشته باشیم. این موجودیت، کلاس نامیده می‌شود و ما چندین نمونه از یک کلاس به نام اشیاء می‌سازیم. برای مثال: در این مثال، یک کد نمونه به سبک شیءگرا نوشته‌ایم.

```csharp
// Car یک کلاس است
class Car
{
    string Name;
    int Speed;
    int Weight;
    void Break()
    {
    }
    void SlowDown()
    {
    }
    static void Main(string[] args)
    {
        // ford یک شیء است
        Car ford = new Car();
    }
}
```

**مزایای برنامه‌نویسی شیءگرا:**

- می‌توانیم برنامه‌ها را از ماژول‌های کاری استانداردی که با یکدیگر ارتباط برقرار می‌کنند بسازیم، به جای این‌که مجبور باشیم نوشتن کد را از صفر شروع کنیم، که این امر منجر به صرفه‌جویی در زمان توسعه و بهره‌وری بالاتر می‌شود.
- زبان شیءگرا امکان شکستن برنامه را به مسائل کوچک (هر بار یک شیء) فراهم می‌کند که به‌راحتی قابل حل هستند.
- سیستم‌های شیءگرا به‌راحتی از سیستم‌های کوچک به بزرگ ارتقا می‌یابند.
- فناوری جدید نوید بهره‌وری بیشتر برنامه‌نویس، کیفیت بهتر نرم‌افزار و هزینهٔ نگهداری کمتر را می‌دهد.
- در برنامه‌نویسی شیءگرا، کد در مقیاس وسیعی توزیع می‌شود که سطح بالاتری از انتزاع را نشان می‌دهد.
- کلاس شامل داده‌ها و توابع است، بنابراین پیچیدگی درون کلاس قرار دارد و برنامه‌نویس با کل کلاس سروکار دارد.
- قابلیت استفادهٔ مجدد از کلاس در داخل کد و خارج از کد یک مزیت بزرگ است.

**تفاوت پارادایم‌ها**

پارادایم‌ها تنها روشی برای نوشتن و سازمان‌دهی کد هستند، اما منطق و عملکرد برنامه ثابت می‌ماند.

مثال: برای شخصی که یک هتل را اداره می‌کند دشوار است که تمام کارها را به‌تنهایی انجام دهد، مثل آشپزی، سرو غذا و شستن ظرف‌ها، در حالی که اگر کارها بین افراد تقسیم شود، وظیفه آسان می‌شود. می‌توانیم مشاهده کنیم که وظیفه ثابت می‌ماند، اما وقتی توسط افراد مختلف به‌طور جداگانه انجام شود، کل کار به‌راحتی انجام می‌شود. به طور مشابه، وقتی کد بین توابع یا کلاس‌ها تقسیم می‌شود، اجرای آن آسان می‌گردد.
